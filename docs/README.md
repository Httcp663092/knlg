# 知识库首页

这里是 `knlg` 的知识库入口。它采用轻量 Markdown 结构，直接在 GitHub 上即可浏览，也方便以后升级为 MkDocs、VitePress 或其他静态站点。

## 核心入口

- [知识地图](knowledge-map.md)：从主题角度查看知识库内容。
- [记录规范](conventions.md)：统一标题、标签、状态和更新方式。
- [临时收集箱](inbox/README.md)：暂存还没有整理的想法和资料。
- [长期领域](areas/README.md)：持续维护的主题和能力。
- [项目沉淀](projects/README.md)：围绕具体项目、目标和交付物的记录。
- [资料库](resources/README.md)：外部资料、链接、书籍、课程和工具。
- [归档](archive/README.md)：已经过期但仍可参考的内容。
- [模板](templates/README.md)：新建笔记时优先复制这里的模板。

## 推荐工作流

1. 快速记录：先写入 `inbox/`，不追求完整。
2. 初步整理：补充来源、标签、状态和下一步动作。
3. 分类沉淀：移动到 `areas/`、`projects/` 或 `resources/`。
4. 定期维护：更新索引，合并重复内容，归档过期内容。

## 笔记状态

- `draft`：草稿，还没有整理完成。
- `active`：正在使用或持续维护。
- `stable`：内容比较稳定，可作为参考。
- `archived`：已经归档，不再主动维护。
