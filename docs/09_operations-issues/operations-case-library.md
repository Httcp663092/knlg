这页用于沉淀跨模块、可复用的问题处理案例。它和 FAQ 的区别是：FAQ 适合快速判断方向，案例库要把现象、影响、排查、根因、处理、预防和关联知识都写完整。

## 使用前先做的四个确认

| 确认项 | 为什么重要 | 示例 |
| --- | --- | --- |
| 范围 | 判断是个别数据问题还是系统性问题 | 只有一个客户余额不对，还是所有客户都不对 |
| 时间 | 判断是否和上线、补丁、接口批次、月结有关 | 6 月 1 日后新增单据才异常 |
| 状态 | 金蝶很多结果受审核、过账、结账、核销状态影响 | 单据已审核但凭证未生成 |
| 口径 | 报表差异往往不是金额错，而是过滤条件不同 | 业务日期、记账日期、会计期间不同 |

## 案例一：应收明细和总账客户余额不一致

### 现象

财务在月末对账时发现：应收款管理模块中某客户余额为 120,000 元，但总账辅助核算明细账中同一客户余额为 100,000 元，差异 20,000 元。客户维度、币别和期间看起来一致。

### 影响

- 资产负债表应收账款可能不准确。
- 客户账龄和催收口径不一致。
- 月结无法确认，审计追溯困难。

### 排查顺序

1. 确认对账期间：应收模块按业务日期、到期日还是会计期间查询，总账按哪个期间查询。
2. 确认币别和汇率：是否存在外币单据、本位币金额和原币金额混查。
3. 确认单据状态：应收单、收款单、退款单是否审核，是否全部参与对账。
4. 确认凭证状态：业务单据是否生成凭证，凭证是否审核和过账。
5. 查询凭证生成情况：是否有生成失败、重复生成、手工删除凭证、业务凭证断链。
6. 检查手工总账凭证：是否有人直接录入应收账款科目并挂了客户维度。
7. 检查核销：收款单是否已经核销，是否存在部分核销、反核销后未重新生成凭证。

### 常见根因

| 根因 | 判断方法 | 处理方式 |
| --- | --- | --- |
| 业务单据未生成凭证 | 凭证生成情况查询有失败记录 | 修正凭证模板或数据后重新生成 |
| 凭证已生成但未过账 | 总账余额表按过账凭证查询金额少 | 审核并过账凭证 |
| 手工凭证直接记入应收科目 | 总账明细有来源为空或非应收来源 | 冲销手工凭证，改由业务单据生成 |
| 维度带错客户 | 凭证客户维度和应收单客户不一致 | 调整凭证模板或红冲重做 |
| 查询口径不一致 | 应收按业务日期，总账按期间 | 统一口径重新对账 |

### 推荐处理

1. 先导出应收对账明细、总账辅助明细和凭证生成情况，保留月结证据。
2. 如果是未生成凭证，优先修正凭证模板或单据基础资料，再批量生成。
3. 如果是手工凭证导致，使用红字或反向凭证调整，避免直接改库。
4. 如果是维度错误，先确认是否允许反审核反过账，再按公司制度处理。
5. 处理后重新生成应收与总账对账表，归档差异前后截图或导出表。

### 预防

- 应收账款科目设置受控或通过制度限制手工凭证。
- 月末固定跑“业务单据未生成凭证”清单。
- 凭证模板新增或修改后做客户、部门、项目维度回归测试。

## 案例二：采购入库已经完成，但应付暂估或存货凭证没有生成

### 现象

仓库已经审核采购入库单，供应商送货也已经入库，但财务生成凭证时没有带出这张入库单，或者凭证生成失败。

### 影响

- 存货账和总账不一致。
- 暂估应付没有确认。
- 月末成本无法准确结转。

### 排查顺序

1. 检查采购入库单是否审核，业务日期是否在当前核算期间。
2. 检查物料是否启用存货核算，物料属性、计价方法、库存组织是否完整。
3. 检查是否完成存货核算或入库成本确认。
4. 检查供应商、部门、物料、仓库等维度是否满足凭证模板条件。
5. 检查凭证模板是否覆盖采购入库暂估场景。
6. 检查是否有红字入库、退货、跨组织采购或费用分摊未完成。

### 常见根因

| 根因 | 现象 | 处理 |
| --- | --- | --- |
| 单据期间不在当前期间 | 查询当前期间没有单据 | 按业务日期和会计期间重新确认 |
| 物料未纳入存货核算 | 入库有数量无金额 | 完善物料和核算范围，重新核算 |
| 入库成本为 0 或未核算 | 凭证模板取不到成本金额 | 完成采购成本确认或暂估成本 |
| 凭证模板条件不覆盖 | 同类单据部分能生成，部分不能 | 增补模板条件和科目取数规则 |
| 基础资料缺失维度 | 报错提示科目或维度为空 | 补充部门、供应商、物料分类、成本项目 |

### 推荐处理

1. 不要直接手工补总账凭证作为第一选择，先确认业务链路是否完整。
2. 如果必须临时手工补账，要标记来源和后续冲回规则，防止正式凭证生成后重复。
3. 修复模板后，用一张单据测试，再批量处理。
4. 月结前建立“入库未生成凭证”和“零成本入库”检查。

## 案例三：库存数量正确，但存货余额和总账不一致

### 现象

仓库库存台账数量正确，库存金额看起来也有数据，但总账存货科目余额与存货核算模块余额不一致。

### 影响

- 资产负债表存货金额不可信。
- 成本结转可能偏差。
- 财务和仓库互相认为对方数据不对。

### 排查顺序

1. 先确认对账口径：组织、库存组织、货主、仓库、物料、批号、期间。
2. 区分数量差异和金额差异：数量一致但金额不一致，重点看核算和凭证。
3. 检查是否所有出入库单都已审核并参与存货核算。
4. 检查是否存在异常成本：零成本、负成本、暂估未冲回、费用分摊未完成。
5. 检查存货凭证是否全部生成、审核和过账。
6. 检查总账是否存在手工记入存货科目的凭证。

### 典型根因

- 上月有未生成凭证的出入库单，差异滚到本月。
- 存货核算已完成，但凭证未过账。
- 采购费用分摊或暂估冲回在存货模块已处理，总账未同步。
- 手工凭证直接调整存货科目，没有对应库存业务。
- 报表按不同库存组织或货主查询。

### 推荐处理

1. 先按期间导出存货收发存汇总、存货核算明细和总账科目明细。
2. 把差异拆成期初差异、本期借方差异、本期贷方差异和期末差异。
3. 如果期初就不一致，回溯到首次差异期间，不要只调当前月。
4. 如果是手工总账凭证，优先做业务调整或反向凭证规范处理。
5. 处理后把对账表纳入月结证据包。

## 案例四：MRP 没有生成采购建议或生产建议

### 现象

销售订单或预测需求已经录入，计划员运行 MRP 后，系统没有生成预期的采购申请、计划订单或生产建议。

### 影响

- 采购计划延迟。
- 生产备料不及时。
- 计划员改用手工 Excel，系统计划能力失效。

### 排查顺序

1. 确认需求来源是否被 MRP 纳入：销售订单、预测单、独立需求、生产订单缺料。
2. 检查需求日期是否落在计划运算范围内。
3. 检查物料计划属性：制造件、采购件、委外件、计划策略、批量规则、提前期。
4. 检查 BOM 是否审核、生效、版本正确，子项是否参与计划。
5. 检查库存、在途、在制、已分配量是否抵消了需求。
6. 检查组织、仓库、计划范围是否一致。
7. 检查是否存在最小批量、固定批量、订货策略导致建议数量被调整。

### 常见根因

| 根因 | 处理 |
| --- | --- |
| BOM 未审核或未生效 | 审核并确认生效日期、用途、版本 |
| 物料计划属性错误 | 维护计划策略、供给方式、提前期 |
| 需求日期不在计划范围 | 调整运算范围或需求日期 |
| 库存或在途足够 | 查看供需明细，确认系统抵消逻辑 |
| 组织或仓库范围不一致 | 统一计划组织、库存组织、仓库范围 |

### 推荐处理

1. 使用供需明细或计划运算日志，不要只看最后有没有建议单。
2. 对一个物料单独跑计划，缩小范围验证。
3. 把需求、库存、在途、在制、BOM、提前期放到一张检查表里。
4. 对关键物料建立计划参数维护流程，变更后必须测试。

- [MRP 与计划管理详细教程](../03_manufacturing/mrp-planning-guide.md)
- [工程数据与 BOM 详细教程](../03_manufacturing/engineering-data-bom-guide.md)

## 案例五：WebAPI 调用成功，但金蝶生成了重复单据

### 现象

外部系统调用接口后返回成功，但由于网络超时、重试机制或外部系统重复推送，金蝶出现两张业务含义相同的单据。

### 影响

- 库存、应收、应付或生产数据重复。
- 财务凭证可能重复生成。
- 对账时外部系统和金蝶数量、金额不一致。

### 排查顺序

1. 检查外部系统请求日志：请求时间、业务主键、请求体、响应结果。
2. 检查金蝶接口日志或集成日志：是否收到多次请求。
3. 检查接口是否设置外部单号、幂等键或唯一性校验。
4. 检查保存接口和提交审核接口是否分开调用，是否保存成功后提交失败又重复保存。
5. 检查二开插件是否在保存后又触发下推或复制。

### 常见根因

- 外部系统超时后重试，但金蝶第一次已经保存成功。
- 接口没有按外部业务主键做幂等控制。
- 外部单号字段没有唯一约束或没有纳入查询校验。
- 保存、提交、审核分段调用时状态处理不完整。

### 推荐处理

1. 先冻结后续推送，避免重复扩大。
2. 按外部业务主键找出重复单据，确认哪张进入后续流程。
3. 未审核且未下游引用的单据可以按流程作废或删除；已生成下游单据的要按业务红冲或反向流程处理。
4. 接口层增加幂等：同一外部业务主键再次推送时返回已有单据编号，不重复创建。
5. 建立接口对账表：外部单号、金蝶单号、状态、金额、最后更新时间。

## 案例六：用户能看到菜单，但打开单据没有数据

### 现象

用户已经分配菜单权限，能进入对应单据列表，但列表为空，或者只能看到部分组织、部分部门的数据。

### 影响

- 用户误以为数据丢失。
- 业务无法及时处理。
- 权限排查容易在菜单权限上反复打转。

### 排查顺序

1. 确认用户、角色、岗位是否正确。
2. 检查菜单权限、按钮权限、字段权限是否完整。
3. 检查数据权限：组织、部门、业务范围、创建人、负责人、仓库、客户供应商范围。
4. 检查用户当前登录组织和默认组织。
5. 检查列表过滤方案是否默认带了条件。
6. 检查单据是否在该用户可见状态，例如草稿、已提交、已审核、已关闭。

### 常见根因

| 根因 | 特征 | 处理 |
| --- | --- | --- |
| 只有菜单权限，没有数据权限 | 能进菜单但无数据 | 补充组织或业务数据权限 |
| 登录组织不对 | 换组织后能看到 | 设置用户组织或提醒切换 |
| 列表默认过滤条件 | 清空过滤后有数据 | 调整过滤方案 |
| 角色继承不完整 | 同岗位其他人正常 | 对比角色和权限差异 |
| 数据隔离规则生效 | 只能看本人或本部门 | 确认内控规则后调整 |

### 推荐处理

1. 用一个正常用户和异常用户做权限对比。
2. 不要直接给管理员权限解决，要定位最小缺失权限。
3. 权限变更要记录申请人、审批人、角色、范围和生效时间。
4. 对财务、付款、价格、成本、工资等敏感数据要保留职责分离。

## 案例七：补丁或二开发布后凭证模板异常

### 现象

发布补丁或二开后，原来可以正常生成凭证的业务单据开始报错，例如科目为空、核算维度为空、金额为 0、摘要不对或凭证重复。

### 影响

- 月结被阻断。
- 财务凭证质量下降。
- 业务和开发互相难以定位责任。

### 排查顺序

1. 确认异常开始时间和发布窗口是否吻合。
2. 对比发布前后单据字段、基础资料字段、凭证模板取数字段是否变化。
3. 检查新增字段是否影响单据转换、保存插件、审核插件、凭证模板条件。
4. 用一张发布前正常单据和一张发布后异常单据对比字段值。
5. 检查插件日志和凭证生成日志。
6. 确认是否清理缓存、重启服务或重新发布元数据。

### 常见根因

- 二开修改字段标识或字段类型，凭证模板还按旧字段取数。
- 新增分录或单据转换后，凭证模板条件没有覆盖新业务类型。
- 插件在保存或审核时重算金额，导致凭证取数时金额为空或为 0。
- 发布后缓存未刷新，元数据版本不一致。

### 推荐处理

1. 先回归标准流程，确认是全量异常还是某个业务类型异常。
2. 如果影响月结，先评估是否回滚发布或临时禁用相关插件。
3. 修改凭证模板后用历史单据、新单据、红字单据、跨组织单据分别测试。
4. 发布清单中固定加入“凭证生成回归测试”。

## 案例八：报表查询越来越慢

### 现象

财务报表、库存报表或自定义报表以前几秒能出结果，现在需要几十秒甚至超时。用户反馈集中在月末或多人同时查询时。

### 影响

- 月结效率下降。
- 用户重复点击导致系统压力更大。
- 可能被误判为系统宕机。

### 排查顺序

1. 确认慢的是所有报表还是某一张报表。
2. 确认慢的是所有用户还是某些用户、某些组织、某些过滤条件。
3. 检查查询期间是否过长，是否一次查多年明细。
4. 检查自定义报表是否存在跨大表无条件联查、函数过滤、模糊查询。
5. 检查服务器资源、数据库连接、慢 SQL、锁等待。
6. 检查是否最近导入大量历史数据、发布补丁或改过索引。

### 常见根因

- 用户用明细报表查多年数据，没有按期间、组织、物料缩小范围。
- 自定义报表 SQL 没有使用有效过滤条件。
- 取数口径包含未必要的大字段、附件或日志表。
- 月末并发生成凭证、核算、报表，资源竞争。
- 数据量增长后原有索引不适合新查询。

### 推荐处理

1. 先给用户一个临时查询口径：缩小期间、组织、物料、客户或供应商范围。
2. 对自定义报表记录慢 SQL、参数和执行时间，不只描述“很慢”。
3. 把报表分为日常查询、月结查询、审计追溯三类，避免一个报表承担全部场景。
4. 对大报表增加汇总表、异步任务或定时报表输出。
5. 性能优化前先备份 SQL 和口径，优化后要核对金额和数量是否一致。
