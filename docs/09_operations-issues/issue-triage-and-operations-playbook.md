这篇用于把金蝶问题排查变成标准动作。遇到问题时，不要先猜原因，要先收集证据、固定口径、缩小范围，再决定是配置、数据、权限、接口、二开还是环境问题。

## 一、问题处理总流程

```text
接收问题
  -> 确认影响范围
  -> 收集证据
  -> 复现问题
  -> 分类定位
  -> 给出临时方案
  -> 根因分析
  -> 修复验证
  -> 预防沉淀
```

## 二、必须收集的信息

| 信息 | 示例 |
| --- | --- |
| 用户 | 张三、财务经理 |
| 模块 | 总账、应收、采购、库存 |
| 单据编号 | XSCKD0001 |
| 组织 | A 公司 |
| 账簿/期间 | 主账簿、2026-06 |
| 时间 | 问题发生时间 |
| 操作步骤 | 点了哪个按钮 |
| 报错原文 | 完整错误信息 |
| 截图 | 报错截图和查询条件 |
| 影响范围 | 单个用户还是所有用户 |
| 是否可复现 | 必现、偶发、不可复现 |

## 三、问题分类

| 类型 | 特征 | 处理方向 |
| --- | --- | --- |
| 权限问题 | 用户看不到菜单、不能审核 | 查角色、按钮、组织权限 |
| 配置问题 | 模板、参数、流程不命中 | 查配置和适用范围 |
| 数据问题 | 单据状态、金额、关联异常 | 查单据、上下游、日志 |
| 接口问题 | 外部同步失败 | 查请求、响应、幂等、日志 |
| 二开问题 | 标准功能正常，扩展逻辑异常 | 查插件日志和发布记录 |
| 环境问题 | 多用户大面积异常 | 查服务、数据库、网络 |
| 性能问题 | 查询慢、保存慢 | 查数据量、SQL、服务器资源 |

## 四、排查方法

### 先判断范围

问四个问题：

1. 只有一个用户有问题，还是所有用户都有问题。
2. 只有一个单据有问题，还是同类单据都有问题。
3. 只有生产环境有问题，还是测试环境也有问题。
4. 是今天突然出现，还是一直存在。

### 再判断变化

最近是否发生过：

- 发布补丁。
- 修改权限。
- 修改参数。
- 修改凭证模板。
- 修改工作流。
- 导入数据。
- 接口切换。
- 二开上线。

很多问题的根因都在“最近变化”里。

## 五、常见问题处理路径

### 用户看不到菜单

1. 查用户是否启用。
2. 查角色是否分配。
3. 查菜单权限。
4. 查组织权限。
5. 查是否有许可或模块授权。
6. 用同角色用户对比。

### 单据不能审核

1. 查单据状态。
2. 查当前用户是否是审批人。
3. 查审批流是否到达当前节点。
4. 查必填字段是否完整。
5. 查插件或校验规则报错。
6. 查是否有下游单据限制。

### 凭证生成失败

1. 查源单是否审核。
2. 查期间是否开放。
3. 查凭证模板是否启用。
4. 查科目来源。
5. 查辅助核算。
6. 查金额字段。
7. 查生成日志。

### 接口同步失败

1. 查外部单号。
2. 查请求报文。
3. 查响应报文。
4. 查接口账号权限。
5. 查主数据映射。
6. 查幂等记录。
7. 查是否需要重试或人工处理。

### 报表数据不对

1. 固定查询条件。
2. 查源数据。
3. 查凭证是否过账。
4. 查报表公式。
5. 查权限和组织范围。
6. 和标准账表交叉验证。

## 六、临时方案和根因修复

临时方案用于恢复业务，但不能替代根因修复。

| 临时方案 | 风险 |
| --- | --- |
| 手工补录 | 可能破坏源单追溯 |
| 手工凭证 | 可能导致子模块和总账不一致 |
| 临时放开权限 | 可能造成越权 |
| 手工导入数据 | 可能重复或错误 |
| 暂停接口 | 可能积压业务 |

每个临时方案都要有：

- 生效时间。
- 失效时间。
- 负责人。
- 风险说明。
- 后续根因修复计划。

## 七、问题台账

问题台账字段：

| 字段 | 说明 |
| --- | --- |
| 编号 | ISSUE-202606-001 |
| 标题 | 应收单无法生成凭证 |
| 模块 | 应收 |
| 优先级 | P0/P1/P2/P3 |
| 状态 | 新建、处理中、待验证、已关闭 |
| 负责人 | 顾问或运维 |
| 发现时间 | 时间 |
| 影响范围 | 用户、组织、模块 |
| 根因 | 配置、权限、数据、接口、二开 |
| 解决方案 | 处理动作 |
| 预防措施 | 如何避免复发 |

## 八、优先级定义

| 优先级 | 标准 | 响应 |
| --- | --- | --- |
| P0 | 生产大面积不可用、月结/付款阻塞 | 立即处理 |
| P1 | 核心流程不可用，有较大业务影响 | 当日处理 |
| P2 | 部分用户或非核心流程异常 | 排期处理 |
| P3 | 咨询、优化、低影响问题 | 计划处理 |

## 九、关闭问题的标准

问题关闭前要满足：

- 根因明确。
- 解决方案已执行。
- 用户或关键用户验证通过。
- 影响数据已修复或有说明。
- 临时权限已回收。
- 相关文档已更新。
- 问题台账已补完整。

## 十、知识沉淀

每处理一个典型问题，要沉淀到：

- [常见问题库](../10_common-questions/README.md)：短问题和快速判断。
- 本目录：完整问题处理过程。
- 对应模块文档：稳定的业务规则和操作方法。
- SQL 区：可复用查询。
- 实施区：可复用风险和验收点。
