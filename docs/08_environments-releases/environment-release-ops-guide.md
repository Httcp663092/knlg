这篇用于记录金蝶环境、版本、补丁和发布运维的标准做法。环境管理做不好，开发、配置、数据和财务月结都会被影响。

## 一、环境分层

| 环境 | 用途 | 管理要求 |
| --- | --- | --- |
| DEV | 开发和单元测试 | 可频繁变更，但要记录 |
| TEST | 顾问测试和联调 | 模拟生产配置 |
| UAT | 用户验收 | 数据和权限尽量接近生产 |
| PROD | 正式生产 | 严格变更审批 |
| DR | 灾备或备份 | 定期演练恢复 |

每个环境都要记录：

- 访问地址。
- 产品版本。
- 补丁号。
- 数据库类型。
- 中间件版本。
- 服务器信息。
- 管理员账号保管方式。
- 最近一次备份时间。

## 二、版本和补丁记录

版本记录模板：

| 字段 | 内容 |
| --- | --- |
| 环境 | PROD、UAT、TEST |
| 产品 | 金蝶云·星空、金蝶云·苍穹 |
| 当前版本 | 具体版本号 |
| 补丁号 | 补丁包编号 |
| 发布时间 | 计划发布时间 |
| 发布人 | 运维或顾问 |
| 影响模块 | 财务、供应链、制造、接口 |
| 回归结果 | 通过或失败 |
| 回滚方案 | 是否可回滚 |

## 三、发布流程

```text
发布申请
  -> 影响评估
  -> 测试环境验证
  -> UAT 验收
  -> 生产备份
  -> 生产发布
  -> 验证
  -> 观察
  -> 归档
```

### 发布前检查

- 是否避开月结、盘点、工资、税务申报关键窗口。
- 是否有完整发布包。
- 是否有数据库备份。
- 是否有配置备份。
- 是否有回滚方案。
- 是否通知关键用户。
- 是否准备验证用例。
- 是否评估接口影响。

### 发布后验证

至少验证：

- 登录和菜单。
- 关键单据新增、保存、审核。
- 凭证生成。
- 报表查询。
- 接口调用。
- 权限。
- 日志。
- 性能。

财务相关发布还要验证：

- 总账凭证录入、审核、过账。
- 应收应付核销。
- 固定资产折旧。
- 报表公式。
- 月结检查。

## 四、回滚方案

回滚不是一句“恢复备份”。要提前确认：

- 数据库能否恢复。
- 代码包能否回退。
- 配置能否回退。
- 回滚会丢失哪些业务数据。
- 回滚窗口多长。
- 谁决策回滚。
- 回滚后如何通知用户。

如果发布后已经产生大量新业务数据，直接恢复备份风险很高，要评估用补丁修复还是回滚。

## 五、账号和权限运维

账号管理要求：

- 离职用户及时禁用。
- 管理员账号不要多人共用。
- 接口账号单独建立。
- 接口账号权限最小化。
- 定期审查高权限用户。
- 生产环境不使用个人开发账号。

权限变更记录：

- 申请人。
- 用户。
- 角色。
- 组织范围。
- 变更原因。
- 审批人。
- 生效时间。
- 回收时间。

## 六、备份和恢复

备份要回答：

- 备份什么。
- 什么时候备份。
- 存在哪里。
- 保留多久。
- 谁能恢复。
- 多久能恢复。
- 是否演练过。

建议：

- 生产发布前必须备份。
- 月结前后保留关键备份。
- 定期做恢复演练。
- 备份文件权限要控制。

## 七、日志和监控

要关注：

- 应用日志。
- 数据库日志。
- 接口日志。
- 定时任务日志。
- 登录和权限日志。
- 业务操作日志。
- 服务器 CPU、内存、磁盘、网络。

常见告警：

- 接口失败率升高。
- 定时任务失败。
- 数据库连接异常。
- 磁盘空间不足。
- 登录异常。
- 报表查询变慢。
- 月结任务失败。

## 八、环境刷新

测试环境经常需要从生产刷新数据。

刷新前：

- 确认刷新范围。
- 备份测试环境原数据。
- 脱敏敏感信息。
- 调整接口地址，避免测试环境调用生产接口。
- 调整定时任务，避免误发消息。
- 通知测试用户。

刷新后：

- 检查登录。
- 检查基础功能。
- 检查接口地址。
- 检查用户权限。
- 检查测试数据是否可用。

## 九、发布风险清单

| 风险 | 预防 |
| --- | --- |
| 发布影响月结 | 避开月结窗口，提前财务回归 |
| 接口地址配错 | 发布后检查环境变量和接口配置 |
| 权限丢失 | 发布前导出权限矩阵 |
| 配置被覆盖 | 发布前备份配置 |
| 补丁引入新问题 | UAT 回归测试 |
| 无法回滚 | 发布前演练回滚路径 |
