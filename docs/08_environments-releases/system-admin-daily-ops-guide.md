系统管理员是金蝶系统长期稳定运行的第一责任角色之一。管理员不只是建账号、改密码，还要维护组织、用户、角色、权限、参数、接口账号、环境信息、发布记录和日常巡检。很多上线后的“系统问题”，本质是账号、权限、基础资料或变更管理不规范。

## 一、管理员职责

| 职责 | 说明 |
| --- | --- |
| 用户管理 | 新增、禁用、调岗、密码重置 |
| 角色权限 | 角色维护、权限矩阵、组织范围 |
| 基础配置 | 编码规则、参数、打印、审批流 |
| 环境信息 | 访问地址、版本、补丁、管理员账号 |
| 接口账号 | 外部系统访问账号、权限、密钥 |
| 发布配合 | 发布前备份、发布后验证 |
| 日志排查 | 登录、操作、接口、后台任务 |
| 安全审计 | 高权限用户、离职账号、异常登录 |

## 二、账号生命周期

### 新员工开户

开户步骤：

1. 收到账号申请。
2. 确认人员、部门、岗位、组织。
3. 匹配标准角色。
4. 分配组织范围和数据范围。
5. 设置初始密码或登录方式。
6. 要求用户首次登录修改密码。
7. 用用户账号测试菜单和关键操作。
8. 记录开户信息。

开户申请至少包含：

- 姓名。
- 部门。
- 岗位。
- 所属组织。
- 需要模块。
- 参考同岗位用户。
- 生效日期。
- 审批人。

### 调岗

调岗不能只加新权限，必须先评估旧权限是否要回收。

处理：

1. 导出原角色和组织范围。
2. 收回不再需要的权限。
3. 分配新岗位角色。
4. 更新组织和数据范围。
5. 测试新岗位关键操作。
6. 保留变更记录。

### 离职

离职处理要及时：

- 禁用用户。
- 回收角色。
- 关闭移动端或外部访问。
- 交接待办、审批和业务单据。
- 禁用或更换该用户持有的接口密钥。
- 保留历史操作记录，不要删除历史用户导致追溯困难。

## 三、角色和权限维护

### 角色设计原则

- 角色按岗位设计，不按个人设计。
- 高危权限单独拆角色。
- 临时权限设置有效期。
- 生产环境管理员账号不参与日常业务。
- 权限变更必须有申请和审批。

### 高危权限清单

| 权限 | 风险 |
| --- | --- |
| 反审核 | 可能修改已生效业务 |
| 反过账 | 影响总账余额 |
| 反结账 | 影响已关账期间 |
| 删除单据 | 破坏追溯 |
| 修改基础资料关键字段 | 影响历史业务展示 |
| 修改凭证模板 | 影响财务入账 |
| 修改审批流 | 影响内控 |
| 数据导入 | 可能批量污染数据 |
| 管理员权限 | 可以越权操作 |

### 权限月度复核

每月检查：

- 离职用户是否禁用。
- 长期未登录用户是否需要保留。
- 管理员账号是否多人共用。
- 高权限角色是否合理。
- 临时权限是否到期。
- 接口账号权限是否过宽。
- 财务关键权限是否有审批记录。

## 四、组织和基础资料维护

组织、仓库、部门、物料、客户、供应商等资料变更要谨慎。

### 维护规则

- 新增资料要按编码规则。
- 关键资料要审核后使用。
- 不建议随意删除已被业务单据引用的资料。
- 停用前确认无未结业务。
- 修改名称、分类、税率、单位等字段要评估影响。

### 物料变更案例

如果要修改物料基本单位：

- 检查是否已有库存。
- 检查是否有采购、销售、生产单据。
- 检查 BOM 是否引用。
- 检查计价和成本是否受影响。
- 必要时新建物料替代，旧物料停用。

## 五、参数和审批流维护

系统参数影响范围大，不能随意改。

### 参数变更流程

```text
提出变更
  -> 说明原因
  -> 评估影响模块
  -> 测试环境验证
  -> 关键用户确认
  -> 生产变更
  -> 变更后验证
  -> 记录归档
```

### 审批流变更

审批流变更要关注：

- 未完成审批任务如何处理。
- 新规则从什么时候生效。
- 是否影响金额阈值。
- 是否影响部门、岗位、组织。
- 是否需要通知用户。

## 六、接口账号管理

接口账号不能用个人账号。

### 接口账号要求

- 单独命名，例如 `api_oms_sync`。
- 最小权限。
- 只开放需要的组织和功能。
- 密钥或密码专人保管。
- 定期轮换。
- 禁止用于前台人工操作。
- 记录调用系统和联系人。

### 接口账号台账

| 字段 | 示例 |
| --- | --- |
| 账号 | api_wms |
| 外部系统 | WMS |
| 用途 | 出入库同步 |
| 权限 | 库存单据新增和查询 |
| 组织范围 | A 工厂 |
| 负责人 | IT 张三 |
| 密钥到期 | 2026-12-31 |
| 最近复核 | 2026-06-30 |

## 七、日常巡检

### 每日巡检

- 登录是否正常。
- 关键接口是否成功。
- 后台任务是否失败。
- 生产环境磁盘空间是否充足。
- 昨日是否有 P0/P1 问题未关闭。
- 是否有异常登录或高频失败登录。

### 每周巡检

- 备份是否成功。
- 接口失败日志是否清理。
- 高权限用户是否变化。
- 待处理工作流是否大量积压。
- 数据清理或归档任务是否执行。
- 报表或任务性能是否异常。

### 每月巡检

- 权限复核。
- 管理员账号复核。
- 版本和补丁记录更新。
- 月结相关任务复盘。
- 备份恢复演练计划。
- 知识库问题沉淀。

## 八、发布配合

管理员参与发布时要做：

### 发布前

- 确认发布窗口。
- 通知用户。
- 备份数据库和配置。
- 导出关键权限和参数。
- 停止或暂停必要定时任务。
- 确认回滚方案。

### 发布后

- 验证登录。
- 验证关键菜单和权限。
- 验证接口。
- 验证后台任务。
- 验证财务凭证和报表。
- 更新版本台账。

## 九、实际案例

### 案例：离职用户仍能通过接口写入数据

现象：

- 某接口持续写入采购入库。
- 负责人已经离职。
- 外部系统仍使用该员工账号调用接口。

风险：

- 历史追溯显示离职人员操作。
- 密码无法有效管理。
- 权限无法按系统用途控制。

处理：

1. 新建专用接口账号。
2. 只授予采购入库所需权限。
3. 外部系统切换账号。
4. 禁用离职员工账号。
5. 更新接口账号台账。

## 十、管理员检查清单

- [ ] 新增用户有申请和审批。
- [ ] 调岗用户已回收旧权限。
- [ ] 离职用户已禁用。
- [ ] 管理员账号不多人共用。
- [ ] 接口账号单独维护。
- [ ] 高危权限有审批。
- [ ] 参数变更有测试记录。
- [ ] 审批流变更有影响评估。
- [ ] 每日接口和任务巡检完成。
- [ ] 版本、补丁、发布记录完整。
