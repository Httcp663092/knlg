备份、恢复和监控是金蝶生产环境运维的底线能力。没有可恢复的备份，发布和数据修复都缺少安全网；没有监控，接口失败、磁盘不足、后台任务异常、报表慢查询往往会在业务已经受影响后才被发现。

## 一、备份策略

### 备份对象

| 对象 | 说明 |
| --- | --- |
| 数据库 | 业务数据、配置数据、日志数据 |
| 应用配置 | 环境变量、连接配置、参数文件 |
| 附件文件 | 单据附件、影像、上传文件 |
| 二开包 | 插件、服务、脚本、发布包 |
| 报表和打印模板 | 自定义报表、套打模板 |
| 接口配置 | 地址、账号、映射、调度 |

### 备份频率

| 场景 | 建议 |
| --- | --- |
| 生产数据库 | 每日至少一次全备或按企业策略 |
| 关键发布前 | 发布前强制备份 |
| 月结前后 | 保留月结前、月结后备份 |
| 大批量导入前 | 导入前备份相关数据 |
| 数据修复前 | 修复前备份影响范围 |

### 备份保留

保留策略要考虑：

- 最近 7 天快速恢复。
- 最近 1-3 个月月度备份。
- 年结或审计节点长期备份。
- 备份介质和权限控制。
- 异地或灾备要求。

## 二、备份检查

备份成功不代表可恢复。必须检查：

- 备份任务是否成功。
- 备份文件大小是否合理。
- 备份文件是否可访问。
- 备份文件是否加密或权限受控。
- 是否有恢复演练记录。

### 备份台账

| 字段 | 示例 |
| --- | --- |
| 环境 | PROD |
| 对象 | 数据库 |
| 备份时间 | 2026-06-23 02:00 |
| 类型 | 全量 |
| 文件位置 | 备份服务器路径 |
| 大小 | 500GB |
| 结果 | 成功 |
| 校验人 | 运维 |
| 恢复演练 | 2026-06-15 已验证 |

## 三、恢复演练

恢复演练至少回答：

- 多久能恢复。
- 恢复到哪个时间点。
- 哪些数据会丢失。
- 应用配置是否也能恢复。
- 附件是否能恢复。
- 接口和任务是否会误连生产。

### 演练步骤

```text
选择备份
  -> 准备恢复环境
  -> 恢复数据库
  -> 恢复附件和配置
  -> 调整环境地址和接口
  -> 启动应用
  -> 验证登录和关键业务
  -> 输出演练报告
```

### 验证内容

- 用户可以登录。
- 关键菜单可访问。
- 单据可查询。
- 附件可打开。
- 报表可查询。
- 接口地址已切换到测试或关闭。
- 后台任务不会误执行生产动作。

## 四、监控体系

### 基础设施监控

| 指标 | 风险 |
| --- | --- |
| CPU | 持续高会导致操作慢 |
| 内存 | 不足会导致服务不稳定 |
| 磁盘空间 | 满盘会导致写入失败 |
| 网络 | 接口和用户访问异常 |
| 数据库连接数 | 连接耗尽导致登录或查询失败 |

### 应用监控

- 登录失败。
- 页面访问异常。
- 后台任务失败。
- 流程审批积压。
- 接口失败率。
- 报表查询耗时。
- 大批量导入任务。
- 异常日志数量。

### 业务监控

- 未审核单据积压。
- 接口待处理消息积压。
- 月结异常项。
- 凭证生成失败。
- 库存负数。
- 成本计算异常。
- 银企对账失败。

## 五、告警分级

| 级别 | 示例 | 响应 |
| --- | --- | --- |
| P0 | 生产无法登录、数据库不可用 | 立即处理 |
| P1 | 关键接口失败、付款或月结阻塞 | 当日处理 |
| P2 | 报表慢、部分任务失败 | 排期处理 |
| P3 | 日志增长、低风险提醒 | 计划优化 |

告警要进入问题台账，不能只停留在聊天记录。

## 六、性能巡检

### 常见性能问题

| 现象 | 可能原因 |
| --- | --- |
| 登录慢 | 认证服务、网络、数据库连接 |
| 单据保存慢 | 插件逻辑、校验规则、数据库锁 |
| 报表查询慢 | 数据量大、过滤条件少、SQL 未优化 |
| 审批慢 | 工作流任务积压、消息服务异常 |
| 接口慢 | 外部系统慢、网络、幂等检查 |
| 月结慢 | 数据量大、成本计算复杂、后台任务冲突 |

### 巡检步骤

1. 确认问题时间段。
2. 判断是否全员、单用户、单模块。
3. 查看服务器资源。
4. 查看数据库连接和慢查询。
5. 查看应用日志。
6. 查看最近发布和配置变更。
7. 复现并记录耗时。
8. 优化后对比耗时。

## 七、发布和备份联动

任何生产发布前都要做：

- 数据库备份。
- 配置备份。
- 发布包留档。
- 回滚说明。
- 验证用例。

发布后要观察：

- 关键接口是否正常。
- 错误日志是否上升。
- 报表和后台任务是否变慢。
- 财务关键流程是否正常。

## 八、数据修复和备份联动

生产数据修复前必须：

- 明确修复范围。
- 查询修复前数据。
- 备份影响表或数据库。
- 在测试环境演练。
- 客户审批。
- 生产执行后导出修复后数据。

修复记录建议：

| 字段 | 说明 |
| --- | --- |
| 问题编号 | ISSUE-001 |
| 修复对象 | 单据、表、字段 |
| 修复前 | 原始值或备份文件 |
| 修复后 | 新值 |
| 执行人 | 运维或顾问 |
| 审批人 | 客户负责人 |
| 验证人 | 关键用户 |

## 九、实际案例

### 案例：月结当天磁盘满导致凭证生成失败

现象：

- 财务生成凭证时报错。
- 多个用户反馈系统保存慢。
- 服务器磁盘使用率 99%。

处理：

1. 停止非必要后台任务。
2. 清理可确认的临时文件和过期日志。
3. 扩容磁盘。
4. 重新执行凭证生成。
5. 验证总账凭证和业务单据。

预防：

- 设置磁盘 80%、90%、95% 告警。
- 日志定期归档。
- 月结前做资源巡检。

## 十、巡检清单

### 每日

- [ ] 备份任务是否成功。
- [ ] 磁盘空间是否正常。
- [ ] 关键接口是否失败。
- [ ] 后台任务是否失败。
- [ ] 是否有 P0/P1 问题。

### 每周

- [ ] 慢查询或慢报表是否增加。
- [ ] 错误日志是否异常增长。
- [ ] 接口失败是否重复出现。
- [ ] 数据清理任务是否执行。
- [ ] 备份文件是否可访问。

### 每月

- [ ] 做一次恢复演练或抽查。
- [ ] 复核备份保留策略。
- [ ] 复核高权限用户。
- [ ] 月结前后资源巡检。
- [ ] 更新运维台账。
