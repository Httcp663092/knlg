来源：金蝶开发者官网“数据清理服务使用指南，帮你告别存储烦恼”和流程服务云数据清理资料。

## 功能定位

系统长期运行后，流程、业务事件、消息、记录型单据等历史数据会持续增长，导致表数据膨胀、存储空间增加和查询性能下降。苍穹数据清理服务用于把过期或失效数据从当前库表中清理，降低存储压力。

## 适用场景

- 流程审批结束后，清理流程实例相关历史数据，例如流转日志、异常消息。
- 定期清理业务事件中心运行积累的历史数据。
- 定期清理消息中心历史消息和消息日志。
- 对记录型单据按时间或业务完成状态清理过期数据。

## 版本能力

| 版本 | 能力 |
| --- | --- |
| BOS V5.0.011 及以后 | 老版本历史数据自动清理，通过调度计划定时清理，主要清理流程相关数据 |
| BOS V6.0.15 及以后 | 新版本采用可配置清理框架，分为按流程清理和按时间清理 |
| V7.0.14 | 流程结束后清理策略优化 |
| 苍穹 8.0 以下 | 数据归档引擎默认关闭，需要配置 `archive.enable=true` |
| 苍穹 8.0 及以后 | 默认开启 |

## 配置开关

苍穹 8.0 以下版本，如需启用数据归档/清理引擎，需要在 MC 中设置：

```text
archive.enable=true
```

设置后需要发布并重启苍穹集群生效。

## 菜单路径

| 版本 | 菜单路径 |
| --- | --- |
| V5.0 | 系统服务云 -> 配置工具 -> 数据归档 |
| V6.0 | 系统服务云 -> 分布式管理 -> 数据归档 |
| V8.0.7 | 系统服务云 -> 分布式管理 -> 数据清理 |

## 核心配置对象

| 菜单 | 作用 |
| --- | --- |
| 清理规则 | 配置数据清理的数据抽取条件规则 |
| 清理调度计划 | 定义清理规则执行的时间计划 |

## 清理规则设计

清理规则要回答三个问题：

1. 清理什么：清理实体，也就是需要清理的单据或数据对象。
2. 什么时候清理：按流程结束、按时间、按状态或其他业务条件。
3. 清理多少：单次清理范围、保留时长、循环次数、数量限制。

建议字段：

| 字段 | 说明 |
| --- | --- |
| 清理实体 | 需要清理的单据或对象 |
| 条件类型 | 按流程、按时间或其他条件 |
| 保留时间 | 数据保留多久后清理 |
| 开始时间 | 清理任务开始执行时间 |
| 每次范围 | 单次清理的时间或数据范围 |
| 循环次数 | 一个调度内循环执行次数 |
| 数量限制 | 每次查询主表的 limit 值，避免一次拉太多数据 |

## 老版本流程清理参数

官方资料提到老版本清理依赖 `wf_confcenter` 参数，示例 fkey 包括：

```text
workflow.schedule.deleteAddressingMsgJobRecord
workflow.schedule.deleteAddressingMsgEvtLog
workflow.schedule.deleteClosedJob
workflow.schedule.deletefailedJob
```

常见 value 变量：

| 变量 | 说明 |
| --- | --- |
| retentionTime | 数据保留时间 |
| startingTime | 开始删除时间 |
| scope | 每次删除时间范围 |
| loopTimes | 每个调度内循环执行次数 |
| limitQuantity | 查询主表时的 limit 数量，主要用于性能优化 |

## 实施步骤

1. 盘点需要清理的数据类型。
2. 确认业务是否已经结束，数据是否允许清理。
3. 按版本确认菜单路径和开关。
4. 配置清理规则。
5. 配置清理调度计划。
6. 在测试环境执行。
7. 检查清理前后数据量、业务功能、日志。
8. 生产环境低峰执行。
9. 建立清理执行监控和回滚方案。

## 风险控制

- 不要直接在生产环境首次执行新规则。
- 不要对未完成流程或未结业务做清理。
- 清理前确认是否需要备份或归档。
- 调度计划不要放在业务高峰。
- 每次清理数量要控制，避免长事务。
- 清理后抽查业务查询、报表、流程历史。

## 常见问题

### 清理菜单找不到

排查：

1. 产品版本是否支持。
2. 菜单路径是否按版本查找。
3. 当前账号是否有系统服务云、分布式管理权限。
4. 8.0 以下是否启用 `archive.enable=true`。

### 清理后业务查询缺历史记录

处理：

1. 判断是否误清理。
2. 查清理规则、清理条件和执行时间。
3. 查是否有归档库或备份。
4. 必要时恢复数据。
5. 调整保留周期和规则。

### 清理任务执行慢

优化：

1. 降低单次 limit 数量。
2. 拆分规则。
3. 调整到低峰执行。
4. 检查清理条件字段索引。
5. 避免一次清理过长时间跨度。
