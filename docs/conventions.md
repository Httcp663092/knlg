# 记录规范

这份规范用来让知识库长期保持可读、可搜索、可维护。

## 文件命名

- 使用小写英文、数字和连字符。
- 文件名尽量表达主题，例如 `github-deployment-notes.md`。
- 一个文件只记录一个主要主题。
- 索引页统一使用 `README.md`。

## 推荐元信息

每篇笔记顶部可以使用以下字段：

```markdown
---
title: 笔记标题
date: 2026-06-22
tags: [tag-a, tag-b]
status: draft
source:
---
```

## 写作结构

建议优先使用这几个部分：

```markdown
# 标题

## 背景

## 关键结论

## 过程记录

## 后续动作

## 参考资料
```

## 标签建议

- `project`：项目相关。
- `area`：长期领域。
- `resource`：外部资料。
- `decision`：决策记录。
- `troubleshooting`：问题排查。
- `template`：模板。

## 整理规则

- inbox 中的内容每周至少整理一次。
- 结论不确定时写明来源和验证状态。
- 移动或归档重要笔记后，更新相关索引。
- 不确定分类时先放 inbox，不阻塞记录。
