这篇专门记录金蝶财务最核心的落点：业务单据如何变成凭证，以及凭证不对时怎么查。

## 核心结论

在金蝶里，凭证不是孤立录入的。应收、应付、出纳、固定资产、供应链等模块的业务单据，需要通过凭证模板或智能会计平台规则生成总账凭证。财务排查时要同时看业务单据、凭证模板、生成日志和总账凭证。

## 凭证生成链路

```text
业务单据
  -> 单据审核
  -> 凭证模板/账务处理规则匹配
  -> 取科目、取金额、取核算维度
  -> 生成凭证
  -> 凭证审核
  -> 凭证过账
  -> 总账报表
```

## 凭证模板要配置什么

| 配置点 | 说明 | 常见问题 |
| --- | --- | --- |
| 适用单据 | 模板对应哪类业务单据 | 单据类型不匹配导致不生成 |
| 适用组织/账簿 | 模板作用范围 | 多组织时命中错误模板 |
| 借贷方向 | 每条分录借贷方向 | 方向反了导致余额异常 |
| 科目来源 | 固定科目、基础资料映射、字段取值 | 科目为空或取错科目 |
| 金额来源 | 价税合计、金额、税额、折旧额等 | 取错金额字段 |
| 核算维度 | 客户、供应商、部门、项目等 | 辅助核算缺失导致生成失败 |
| 合并规则 | 按单据生成、按科目合并、按维度合并 | 合并过度导致明细不可追溯 |

## 生成凭证前检查

1. 单据已经审核。
2. 单据业务日期对应期间未关闭。
3. 单据组织能找到对应账簿。
4. 单据金额不为 0，借贷金额能平衡。
5. 科目和核算维度都能取到。
6. 生成凭证用户有权限。
7. 凭证字、期间、币别、汇率可用。

## 生成凭证后检查

| 检查项 | 方法 |
| --- | --- |
| 是否生成成功 | 看凭证生成情况查询或单据关联凭证 |
| 是否重复生成 | 查同一源单是否已有关联凭证 |
| 借贷是否平衡 | 看凭证分录金额 |
| 科目是否正确 | 对照凭证模板和业务场景 |
| 辅助核算是否完整 | 查看凭证分录维度 |
| 是否已过账 | 凭证列表或总账查询 |

## 常见业务分录

### 销售确认应收

场景：销售业务形成客户应收。

```text
借：应收账款-客户
贷：主营业务收入
贷：应交税费-应交增值税-销项税额
```

金蝶关注点：

- 应收单客户、结算客户、销售组织、币别、税率要正确。
- 应收科目通常需要客户辅助核算。
- 收入科目可能按物料、收入类别、销售组织或业务类型映射。

### 收到客户回款

```text
借：银行存款
贷：应收账款-客户
```

金蝶关注点：

- 收款单银行账号要能对应银行存款科目。
- 收款单和应收单要核销，否则应收账龄仍会显示未收。
- 银行存款科目可能需要银行账号辅助核算。

### 收款退款

```text
借：应收账款-客户
贷：银行存款
```

金蝶关注点：

- 退款单方向和原收款单要一致。
- 如果退款是冲原收款，需要检查关联和核销状态。

### 采购确认应付

```text
借：原材料/库存商品/费用
借：应交税费-应交增值税-进项税额
贷：应付账款-供应商
```

金蝶关注点：

- 应付确认口径可能按采购入库、采购发票或应付单。
- 存货类科目和费用类科目的来源不同，通常要结合物料、采购类型、费用项目。
- 应付科目通常需要供应商辅助核算。

### 支付供应商款项

```text
借：应付账款-供应商
贷：银行存款
```

金蝶关注点：

- 付款单和应付单要核销。
- 付款账号决定资金科目。
- 多供应商、多币别付款时要特别注意核销对象。

### 固定资产购入

```text
借：固定资产
借：应交税费-应交增值税-进项税额
贷：应付账款/银行存款
```

金蝶关注点：

- 资产卡片的资产类别和折旧政策会影响后续折旧。
- 固定资产科目可能按资产类别映射。

### 计提折旧

```text
借：制造费用/管理费用/销售费用
贷：累计折旧
```

金蝶关注点：

- 折旧费用科目通常按使用部门、资产类别、费用承担部门映射。
- 折旧凭证必须过账后，总账才体现。

### 固定资产处置

常见处理包括转出原值、累计折旧、确认处置损益。具体科目取决于企业会计政策和处置方式。

金蝶关注点：

- 处置单据要审核。
- 处置凭证通常包含固定资产清理、累计折旧、固定资产原值、处置收入或损益。
- 处置后资产卡片状态应变化。

### 期末调汇

外币业务期末需要按期末汇率调整汇兑损益。

```text
借/贷：财务费用-汇兑损益
贷/借：应收账款/应付账款/银行存款等外币科目
```

金蝶关注点：

- 外币科目是否启用外币核算。
- 汇率类型和期末汇率是否维护。
- 调汇凭证是否生成、审核、过账。

### 结转损益

收入类：

```text
借：主营业务收入/其他业务收入
贷：本年利润
```

成本费用类：

```text
借：本年利润
贷：主营业务成本/管理费用/销售费用/财务费用
```

金蝶关注点：

- 损益科目是否全部纳入结转方案。
- 新增损益科目后，要同步检查自动转账或结转模板。
- 结转凭证生成后必须审核过账。

## 凭证生成失败排查表

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 找不到凭证模板 | 模板未启用、单据类型不匹配、组织不匹配 | 检查模板适用范围 |
| 科目为空 | 科目映射缺失、基础资料未维护 | 补科目映射或基础资料 |
| 辅助核算为空 | 客户/供应商/部门/项目字段为空 | 回单据补维度或调整取值规则 |
| 借贷不平 | 金额字段取错、税额方向不对、舍入差异 | 检查金额公式和税额字段 |
| 期间不可用 | 业务日期所在期间已关闭 | 调整业务日期或反结账处理 |
| 重复生成 | 源单已有凭证未删除或未冲销 | 查询凭证生成情况，避免重复 |

## 凭证生成情况查询怎么用

排查业务凭证时，优先从“源单 -> 凭证”方向查；如果源单多，也可以用凭证生成情况查询按组织、期间、单据类型、生成状态过滤。

常用过滤条件：

- 账簿
- 会计期间
- 单据类型
- 业务组织
- 生成状态
- 源单编号
- 凭证编号

## 不建议直接删除凭证的场景

- 凭证已经参与报表和月结。
- 凭证来自已核销的应收应付单据。
- 凭证来自固定资产折旧或处置。
- 凭证已被审计或已经对外报表使用。

更稳的处理方式：

1. 先确认源单和凭证关系。
2. 如果是错误业务单据，优先按业务流程反审核或冲销。
3. 如果是模板配置错误，修模板后重新生成。
4. 如果已经跨期，优先用调整凭证处理，并保留说明。
