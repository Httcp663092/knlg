## 适用范围

适用于采购应付、其他应付、付款、退款、核销、供应商对账和应付月结。

## 核心目标

应付款管理的目标是确认对供应商或其他往来单位的债务，按付款条件安排付款，完成付款核销，并与总账保持一致。

## 业务主线

```text
采购业务/其他业务
  -> 应付单/其他应付单
  -> 付款单/付款退款单
  -> 核销
  -> 账务处理
  -> 期末处理
  -> 报表分析
```

## 基础设置

常见基础资料包括：

- 供应商
- 客户
- 币别
- 结算方式
- 物料
- 费用项目
- 采购部门、采购组、采购员
- 组织信息
- 账簿信息
- 汇率体系
- 付款条件
- 应付付款核销方案
- 应付开票核销方案

## 初始化

应付初始化通常包括：

1. 设置启用日期。
2. 录入期初应付单、期初其他应付单、期初付款单、期初付款退款单。
3. 核对应付明细和总账余额。
4. 结束初始化。

## 日常业务

| 业务 | 单据 | 说明 |
| --- | --- | --- |
| 采购应付 | 应付单 | 确认采购形成的供应商债务 |
| 其他应付 | 其他应付单 | 处理非采购形成的应付款 |
| 付款 | 付款单 | 记录对供应商付款 |
| 付款退款 | 付款退款单 | 处理供应商退款或付款退回 |
| 核销 | 应付与付款核销 | 确认债务和付款的匹配关系 |

## 采购应付完整案例

### 场景

向供应商 B 采购材料，含税金额 5650 元，税额 650 元，不含税材料成本 5000 元，货到后确认应付，一个月后付款。

### 系统处理

1. 采购业务形成应付单。
2. 应付单审核。
3. 生成应付凭证。
4. 到期付款时录入或生成付款单。
5. 付款单审核并生成付款凭证。
6. 应付单和付款单核销。
7. 应付账龄和供应商对账单更新。

### 常见分录

确认应付：

```text
借：原材料/库存商品          5000
借：应交税费-进项税额         650
贷：应付账款-供应商 B        5650
```

支付供应商：

```text
借：应付账款-供应商 B        5650
贷：银行存款                 5650
```

### 金蝶检查点

- 应付单供应商、结算供应商、收款方是否正确。
- 采购费用、税额、价税合计是否正确。
- 应付科目是否带供应商辅助核算。
- 付款单银行账号和结算方式是否正确。
- 付款单是否和应付单完成核销。

## 暂估应付处理口径

企业如果存在采购入库已发生但发票或应付确认滞后的情况，通常需要关注暂估。

常见链路：

```text
采购入库
  -> 暂估入账
  -> 后续收到发票或确认应付
  -> 暂估冲回或调整
  -> 正式应付入账
```

排查重点：

- 暂估科目是否和正式应付科目区分。
- 暂估冲回期间是否正确。
- 发票金额和入库金额差异如何处理。
- 采购入库、应付单、发票之间是否能追溯。

## 应付凭证模板设计要点

| 分录 | 科目来源 | 金额来源 | 辅助核算 |
| --- | --- | --- | --- |
| 借：存货/费用 | 物料、费用项目、采购类型映射 | 不含税金额或成本金额 | 物料、部门、项目可选 |
| 借：进项税额 | 税率或税种映射 | 税额 | 按企业要求 |
| 贷：应付账款 | 供应商或业务类型映射 | 价税合计 | 供应商 |

设计原则：

- 应付科目必须保留供应商维度。
- 费用类采购要确认费用承担部门或项目。
- 存货类采购要和库存、存货核算口径一致。

## 关键控制点

- 应付确认口径：应付单确认还是采购发票确认，要结合账簿设置。
- 付款条件：影响付款计划和账龄。
- 付款申请、付款单、银行付款之间是否存在流程衔接。
- 发票、应付单、付款单之间是否需要开票核销和付款核销双线管理。
- 供应商、结算供应商、收款方是否一致。

## 报表与对账

常用关注点：

- 应付款汇总和明细
- 供应商对账
- 到期债务
- 付款执行情况
- 应付账龄
- 已入库未应付、已应付未开票等差异

## 常见问题

### 应付单不能下推付款单

检查单据状态、付款条件、剩余可付款金额、组织权限、币别、供应商信息和流程配置。

### 付款单无法核销应付单

检查供应商、结算组织、币别、金额方向、核销方案、单据是否审核、是否已被其他核销占用。

### 应付和总账不一致

检查应付单、付款单是否已生成凭证并过账；排查是否存在手工凭证直接影响应付科目。
