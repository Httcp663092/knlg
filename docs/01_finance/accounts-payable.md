# 应付款管理

## 适用范围

适用于采购应付、其他应付、付款、退款、核销、供应商对账和应付月结。

## 核心目标

应付款管理的目标是确认对供应商或其他往来单位的债务，按付款条件安排付款，完成付款核销，并与总账保持一致。

## 业务主线

```text
采购业务/其他业务
  -> 应付单/其他应付单
  -> 付款单/付款退款单
  -> 核销
  -> 账务处理
  -> 期末处理
  -> 报表分析
```

## 基础设置

常见基础资料包括：

- 供应商
- 客户
- 币别
- 结算方式
- 物料
- 费用项目
- 采购部门、采购组、采购员
- 组织信息
- 账簿信息
- 汇率体系
- 付款条件
- 应付付款核销方案
- 应付开票核销方案

## 初始化

应付初始化通常包括：

1. 设置启用日期。
2. 录入期初应付单、期初其他应付单、期初付款单、期初付款退款单。
3. 核对应付明细和总账余额。
4. 结束初始化。

## 日常业务

| 业务 | 单据 | 说明 |
| --- | --- | --- |
| 采购应付 | 应付单 | 确认采购形成的供应商债务 |
| 其他应付 | 其他应付单 | 处理非采购形成的应付款 |
| 付款 | 付款单 | 记录对供应商付款 |
| 付款退款 | 付款退款单 | 处理供应商退款或付款退回 |
| 核销 | 应付与付款核销 | 确认债务和付款的匹配关系 |

## 关键控制点

- 应付确认口径：应付单确认还是采购发票确认，要结合账簿设置。
- 付款条件：影响付款计划和账龄。
- 付款申请、付款单、银行付款之间是否存在流程衔接。
- 发票、应付单、付款单之间是否需要开票核销和付款核销双线管理。
- 供应商、结算供应商、收款方是否一致。

## 报表与对账

常用关注点：

- 应付款汇总和明细
- 供应商对账
- 到期债务
- 付款执行情况
- 应付账龄
- 已入库未应付、已应付未开票等差异

## 常见问题

### 应付单不能下推付款单

检查单据状态、付款条件、剩余可付款金额、组织权限、币别、供应商信息和流程配置。

### 付款单无法核销应付单

检查供应商、结算组织、币别、金额方向、核销方案、单据是否审核、是否已被其他核销占用。

### 应付和总账不一致

检查应付单、付款单是否已生成凭证并过账；排查是否存在手工凭证直接影响应付科目。
