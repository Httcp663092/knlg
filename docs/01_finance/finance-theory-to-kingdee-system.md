这篇把财务理论翻译成金蝶系统对象。学金蝶财务时，最容易卡住的不是会不会点菜单，而是不知道“会计理论中的概念”在系统里落在哪些资料、单据、参数、凭证模板和报表上。

## 一句话框架

财务理论解决“应该如何确认、计量、记录和报告”，金蝶系统解决“这些规则如何被组织、账簿、科目、核算维度、业务单据、凭证模板和报表公式执行”。

## 会计理论和金蝶对象对应表

| 财务理论概念 | 在金蝶中的系统对象 | 项目实施时要确认什么 |
| --- | --- | --- |
| 会计主体 | 核算组织、财务组织、账簿 | 哪些公司独立出报表，哪些组织只做业务 |
| 持续经营 | 会计政策、会计日历、期间 | 会计期间、启用期间、是否允许跨期 |
| 会计分期 | 会计期间、结账、反结账 | 月结顺序、关账规则、跨期调整权限 |
| 货币计量 | 本位币、币别、汇率体系 | 本位币、外币业务、期末调汇规则 |
| 权责发生制 | 应收单、应付单、费用确认、凭证模板 | 收入和成本费用按什么时点确认 |
| 收付实现 | 收款单、付款单、现金银行、出纳 | 资金流水如何和应收应付核销 |
| 资产 | 固定资产卡片、存货、银行账户、应收款 | 资产分类、计价、折旧、减值和台账 |
| 负债 | 应付单、其他应付单、借款、预收预付 | 往来对象、到期日、付款条件 |
| 所有者权益 | 实收资本、资本公积、未分配利润科目 | 权益科目和报表公式 |
| 收入 | 应收单、销售出库、收入科目 | 收入确认口径、税率、客户维度 |
| 费用 | 费用报销、其他应付、费用科目 | 部门、项目、费用项目维度 |
| 利润 | 损益结转、本年利润、利润表 | 损益科目结转、报表公式 |

## 会计主体：从组织到账簿

理论上，一个会计主体要独立核算和出报表。金蝶项目里要落到三件事：

1. 组织：公司、分公司、工厂、事业部到底谁是核算组织。
2. 账簿：每个核算组织是否有主账簿、副账簿。
3. 报表：每个账簿是否要出资产负债表、利润表、现金流量表。

### 项目例子

某制造集团有 A 总部、B 工厂、C 销售公司。

| 组织 | 业务属性 | 财务处理 |
| --- | --- | --- |
| A 总部 | 管理、采购结算、资金集中 | 独立核算，主账簿 |
| B 工厂 | 生产、库存、成本 | 独立核算，主账簿 |
| C 销售公司 | 销售、应收、发货 | 独立核算，主账簿 |

实施判断：

- 如果 B 工厂只是 A 总部下的成本中心，不独立出报表，可以不建独立核算组织。
- 如果 B 工厂需要独立资产负债表和利润表，就要建核算组织和账簿。
- 如果集团需要合并报表，还要设计内部往来和组织间交易口径。

## 权责发生制：从业务发生到应收应付

权责发生制要求收入和费用按发生期间确认，而不是等收付款才确认。金蝶里主要通过应收单、应付单、费用单据和凭证模板落地。

### 销售收入确认

常见口径：

| 确认口径 | 系统来源 | 适用场景 |
| --- | --- | --- |
| 按销售出库确认 | 销售出库单生成应收或收入凭证 | 发货即满足收入确认条件 |
| 按销售发票确认 | 发票或应收单确认收入 | 税票和收入确认强相关 |
| 按验收确认 | 验收单或项目节点单据触发 | 项目型交付、服务收入 |

项目上必须问清楚：

- 收入确认时点是什么。
- 销售出库、开票、收款是否分离。
- 已发货未开票是否需要暂估收入。
- 退货、折让、红字发票如何处理。

### 采购成本费用确认

常见口径：

| 确认口径 | 系统来源 | 风险 |
| --- | --- | --- |
| 入库暂估 | 采购入库单、暂估应付 | 发票到后要冲回或调整 |
| 应付确认 | 应付单 | 入库未应付会影响成本和负债 |
| 费用报销确认 | 费用报销单、其他应付单 | 部门和项目维度要准确 |

## 科目不是孤立的：科目 + 核算维度才是完整口径

在金蝶项目里，科目设计不能只导入科目表。很多财务管理要求要靠核算维度实现。

| 管理诉求 | 科目 | 建议核算维度 |
| --- | --- | --- |
| 查每个客户欠多少钱 | 应收账款 | 客户 |
| 查每个供应商欠款 | 应付账款 | 供应商 |
| 查每个部门费用 | 管理费用、销售费用 | 部门 |
| 查每个项目盈亏 | 收入、成本、费用 | 项目 |
| 查每个银行账号余额 | 银行存款 | 银行账号 |
| 查每类资产原值折旧 | 固定资产、累计折旧 | 资产类别、部门 |

### 维度设计原则

- 报表要按什么维度出，凭证就要尽量带什么维度。
- 业务单据上没有的维度，凭证模板很难凭空生成。
- 应收应付类科目不要允许随意手工制证，否则子模块和总账会对不上。
- 维度越多，录入成本越高，实施时要平衡管理精细度和使用成本。

## 凭证模板是理论落地的关键

凭证模板把“业务事实”转换成“会计分录”。

### 应收单凭证模板示例

业务事实：

- 客户 A 购买商品，价税合计 11300。
- 不含税收入 10000。
- 税额 1300。

理论分录：

```text
借：应收账款-客户A        11300
贷：主营业务收入          10000
贷：应交税费-销项税额      1300
```

系统模板要配置：

| 分录 | 科目来源 | 金额来源 | 核算维度 |
| --- | --- | --- | --- |
| 借方应收账款 | 客户、业务类型或固定科目映射 | 价税合计 | 客户 |
| 贷方收入 | 物料、收入类别或业务类型映射 | 不含税金额 | 部门、项目、物料可选 |
| 贷方销项税 | 税率或税种映射 | 税额 | 按企业要求 |

如果模板没有带客户维度，理论上分录借贷平衡，但管理上无法查客户往来。

## 核销是往来管理的核心，不等于生成凭证

很多项目里，财务以为“收款单生成凭证了，应收就清了”。这不完整。

应收减少有两层含义：

1. 总账层面：收款凭证借银行存款、贷应收账款，科目余额减少。
2. 往来层面：收款单和应收单核销，客户账龄和未收款明细减少。

如果只生成凭证不核销：

- 总账应收余额可能减少。
- 应收账龄仍显示逾期。
- 客户对账单仍有未核销明细。

## 月结理论：完整性、准确性、截止性

月结本质上要保证三件事：

| 目标 | 金蝶检查点 |
| --- | --- |
| 完整性 | 本期业务单据是否全部审核、生成凭证 |
| 准确性 | 科目、金额、税额、核算维度是否正确 |
| 截止性 | 业务日期、凭证日期、期间是否正确 |

### 项目里的月结顺序

```text
业务单据清理
  -> 应收应付核销
  -> 出纳银行对账
  -> 固定资产折旧
  -> 存货核算和成本核算
  -> 业务凭证生成
  -> 总账凭证审核过账
  -> 期末调汇
  -> 自动转账
  -> 结转损益
  -> 总账结账
  -> 报表出具
```

## 报表理论：报表公式必须跟科目体系同步

资产负债表和利润表的理论结构很稳定，但系统报表能不能对，取决于公式是否覆盖当前科目体系。

项目常见问题：

- 新增费用科目后，利润表公式没加。
- 新增其他流动资产科目后，资产负债表公式没加。
- 损益结转模板没包含新增损益科目。
- 科目方向配置错误导致报表正负异常。
