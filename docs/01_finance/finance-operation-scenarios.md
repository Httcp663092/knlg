这篇按真实工作场景写。每个场景都包括业务背景、金蝶系统动作、常见分录、检查点和项目经验。

## 场景 1：销售发货后确认应收和收入

### 业务背景

客户 A 下单购买产品 P100，数量 10 件，含税单价 1130 元，税率 13%，价税合计 11300 元。企业口径是发货后确认收入和应收。

### 金蝶系统动作

```text
销售订单
  -> 销售出库单
  -> 应收单
  -> 应收单审核
  -> 生成凭证
  -> 凭证审核过账
  -> 应收明细表/科目余额表/利润表
```

### 理论分录

```text
借：应收账款-客户A              11300
贷：主营业务收入-产品销售        10000
贷：应交税费-应交增值税-销项税额  1300
```

### 系统配置点

| 配置 | 要点 |
| --- | --- |
| 应收确认来源 | 销售出库、销售发票或应收单，要和企业收入确认政策一致 |
| 客户辅助核算 | 应收账款必须带客户维度 |
| 收入科目映射 | 可按物料、物料分组、销售组织、业务类型映射 |
| 税额科目 | 按税率或税种映射销项税 |
| 凭证模板 | 金额字段要区分价税合计、不含税金额、税额 |

### 检查点

- 应收单客户是否等于实际结算客户。
- 应收单业务日期是否属于正确期间。
- 应收单价税合计、不含税金额、税额是否正确。
- 凭证是否带客户维度。
- 凭证是否过账。
- 利润表收入是否增加。
- 应收账龄是否出现该笔应收。

### 项目经验

如果企业销售出库和开票时间差很大，不能简单用开票作为收入确认口径。项目调研时要问清“财务认收入的证据是什么”：出库、签收、验收、发票、项目节点，还是合同里约定的履约义务完成。

## 场景 2：客户回款、收款核销和账龄清理

### 业务背景

客户 A 支付 11300 元，银行已到账。财务要在金蝶中记录收款，并清掉上一个场景的应收。

### 金蝶系统动作

```text
银行到账
  -> 收款单
  -> 收款单审核
  -> 生成收款凭证
  -> 收款单和应收单核销
  -> 银行对账
  -> 应收账龄更新
```

### 理论分录

```text
借：银行存款-招商银行基本户      11300
贷：应收账款-客户A              11300
```

### 核销动作

核销要把收款单和应收单匹配起来。

| 对象 | 核销前 | 核销后 |
| --- | --- | --- |
| 应收单 | 未收 11300 | 未收 0 |
| 收款单 | 未核销 11300 | 未核销 0 |
| 应收账龄 | 显示未收 | 不再显示该笔逾期 |
| 客户对账单 | 有未核销应收和收款 | 清账 |

### 检查点

- 收款客户、结算客户是否正确。
- 收款币别是否和应收单一致。
- 收款银行账号是否正确。
- 收款凭证是否过账。
- 收款是否核销到对应应收单。
- 银行日记账是否能和银行流水对上。

### 项目经验

很多企业上线后账龄不准，不是凭证错，而是核销没跟上。项目上线验收要单独测“收款核销后账龄是否消失”，不要只测“收款凭证能生成”。

## 场景 3：采购入库、确认应付和后续付款

### 业务背景

向供应商 B 采购材料 M100，入库数量 100 件，含税金额 56500 元，税率 13%，不含税成本 50000 元，税额 6500 元。

### 金蝶系统动作

```text
采购订单
  -> 采购入库单
  -> 应付单
  -> 应付单审核
  -> 生成应付凭证
  -> 付款单
  -> 生成付款凭证
  -> 应付单和付款单核销
```

### 理论分录

确认应付：

```text
借：原材料-M100                 50000
借：应交税费-应交增值税-进项税额  6500
贷：应付账款-供应商B             56500
```

付款：

```text
借：应付账款-供应商B             56500
贷：银行存款-招商银行基本户       56500
```

### 系统配置点

| 配置 | 要点 |
| --- | --- |
| 应付确认口径 | 按入库、发票或应付单确认 |
| 存货科目 | 按物料、物料分组或存货类别映射 |
| 供应商维度 | 应付账款必须带供应商 |
| 进项税科目 | 税额字段取值正确 |
| 核销方案 | 付款单能自动或手工匹配应付单 |

### 检查点

- 入库单是否审核。
- 应付单是否正确引用入库或发票。
- 应付单供应商和结算供应商是否正确。
- 凭证是否按供应商维度入账。
- 付款后是否完成核销。
- 应付账龄是否减少。

### 项目经验

采购入库和应付确认分离的企业，要特别处理“暂估应付”。如果上线时只导总账余额，没有导入应付明细，后续付款核销和供应商对账会非常痛苦。

## 场景 4：费用报销入账并按部门核算

### 业务背景

销售部员工报销差旅费 3200 元，公司要求费用按部门核算，付款通过银行转账。

### 金蝶系统动作

```text
费用报销单
  -> 审批通过
  -> 其他应付单或付款申请
  -> 付款单
  -> 生成费用和付款凭证
  -> 部门费用报表
```

### 常见分录

确认费用：

```text
借：销售费用-差旅费-销售部      3200
贷：其他应付款-员工             3200
```

付款：

```text
借：其他应付款-员工             3200
贷：银行存款                    3200
```

有些企业也会在付款时直接确认费用：

```text
借：销售费用-差旅费-销售部      3200
贷：银行存款                    3200
```

### 系统配置点

- 费用项目要能映射到费用科目。
- 申请部门或费用承担部门要带到凭证维度。
- 员工往来如果要管理未付款报销，需要员工或个人维度。
- 报销附件如果接入 AI 文档识别，可识别发票、支付截图、回单。

### 检查点

- 费用承担部门是否等于报销人部门，还是项目部门。
- 发票金额、报销金额、付款金额是否一致。
- 费用科目是否进入利润表公式。
- 如果先挂其他应付款，付款后是否清账。

## 场景 5：固定资产购入、折旧和月末对账

### 业务背景

公司购入设备一台，不含税 100000 元，进项税额 13000 元，预计使用 5 年，残值率 5%，用于生产部门。

### 金蝶系统动作

```text
采购或资产申请
  -> 固定资产卡片
  -> 资产入账
  -> 生成资产入账凭证
  -> 月末计提折旧
  -> 生成折旧凭证
  -> 固定资产结账
  -> 固定资产台账对总账
```

### 理论分录

资产入账：

```text
借：固定资产-机器设备           100000
借：应交税费-应交增值税-进项税额  13000
贷：应付账款/银行存款           113000
```

月折旧：

```text
借：制造费用-折旧费              1583.33
贷：累计折旧-机器设备            1583.33
```

月折旧测算：

```text
100000 * (1 - 5%) / 60 = 1583.33
```

### 系统配置点

- 资产类别映射固定资产科目和累计折旧科目。
- 折旧政策决定当月新增是否当月折旧。
- 使用部门决定折旧费用科目或费用维度。
- 资产卡片开始使用日期影响折旧起点。

### 检查点

- 资产卡片原值是否等于不含税金额。
- 累计折旧初始值是否正确。
- 预计使用期间和残值率是否正确。
- 折旧凭证是否生成并过账。
- 固定资产原值、累计折旧是否和总账一致。

## 场景 6：期末调汇

### 业务背景

客户 C 有 USD 应收 10000 美元。发生时汇率 7.10，期末汇率 7.20。

### 理论逻辑

原本位币余额：

```text
10000 * 7.10 = 71000
```

期末应按汇率重估：

```text
10000 * 7.20 = 72000
```

汇兑收益：

```text
72000 - 71000 = 1000
```

### 常见分录

```text
借：应收账款-客户C              1000
贷：财务费用-汇兑损益           1000
```

### 金蝶操作要点

- 外币科目启用外币核算。
- 维护期末汇率。
- 调汇方案包含该外币科目。
- 生成期末调汇凭证。
- 调汇凭证审核过账。

### 检查点

- 汇率类型是否正确。
- 期末汇率是否维护。
- 调汇范围是否包含应收、应付、银行外币科目。
- 调汇凭证是否影响当期损益。

## 场景 7：月末结转损益

### 业务背景

本月收入 100000，主营业务成本 60000，销售费用 5000，管理费用 8000。

### 理论结果

本月利润：

```text
100000 - 60000 - 5000 - 8000 = 27000
```

### 常见结转分录

收入结转：

```text
借：主营业务收入                100000
贷：本年利润                    100000
```

成本费用结转：

```text
借：本年利润                    73000
贷：主营业务成本                60000
贷：销售费用                    5000
贷：管理费用                    8000
```

### 金蝶操作要点

- 所有损益凭证先审核过账。
- 自动转账或损益结转方案包含所有损益科目。
- 新增损益科目后同步维护结转方案和利润表公式。
- 结转凭证审核过账后再出报表。

## 场景 8：报表不平的排查

### 现象

资产负债表资产合计比负债和所有者权益多 5000。

### 排查路径

1. 查本期凭证是否全部过账。
2. 查科目余额表借贷是否平衡。
3. 查资产负债表公式是否覆盖新增科目。
4. 查损益结转是否完成。
5. 查本年利润和未分配利润公式。
6. 查是否有科目方向或重分类规则错误。

### 项目经验

报表不平多数不是凭证借贷不平，而是报表公式漏科目。项目上线后，只要新增科目，就必须同步检查三件事：凭证模板、损益结转方案、报表公式。
