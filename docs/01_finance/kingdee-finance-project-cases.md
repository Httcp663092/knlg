这些案例是按金蝶项目常见场景整理的脱敏模拟案例，用来训练项目思路。每个案例都包含背景、方案、系统配置、测试数据、验收点和常见问题。

## 案例 1：制造企业财务上线，从手工账切到金蝶

### 项目背景

某制造企业以前用 Excel 和手工凭证管理财务，准备上线金蝶。企业有一个总部、一个生产工厂、一个销售部门。主要业务包括采购原材料、生产入库、销售发货、客户回款、供应商付款、固定资产折旧。

### 项目目标

- 建立总账、应收、应付、出纳、固定资产基础体系。
- 实现采购到应付、销售到应收、收付款和固定资产折旧自动生成凭证。
- 月结从 10 天缩短到 5 天内。
- 应收应付明细能和总账对上。

### 蓝图方案

| 事项 | 方案 |
| --- | --- |
| 核算组织 | 总部作为核算组织，工厂作为业务组织和成本中心 |
| 账簿 | 建一个主账簿，本位币 CNY |
| 科目表 | 使用企业会计准则科目表，按企业实际科目调整 |
| 辅助核算 | 应收按客户，应付按供应商，费用按部门，收入成本按产品线 |
| 应收确认 | 销售出库后生成应收单，应收单生成收入凭证 |
| 应付确认 | 采购入库后生成应付单，应付单生成应付凭证 |
| 固定资产 | 初始化资产卡片，按部门计提折旧 |
| 月结 | 供应链关账 -> 应收应付 -> 固定资产 -> 总账 |

### 初始化数据

| 数据类型 | 示例 | 核对方式 |
| --- | --- | --- |
| 总账期初 | 应收账款 200000，应付账款 150000，固定资产 800000 | 借贷平衡 |
| 应收期初 | 客户 A 120000，客户 B 80000 | 合计等于应收账款科目 |
| 应付期初 | 供应商 X 90000，供应商 Y 60000 | 合计等于应付账款科目 |
| 固定资产期初 | 设备 5 台，原值 800000，累计折旧 200000 | 原值和累计折旧对总账 |
| 银行期初 | 招行基本户 300000 | 银行余额对总账 |

### 凭证模板设计

销售应收：

```text
借：应收账款-客户
贷：主营业务收入-产品线
贷：应交税费-销项税额
```

采购应付：

```text
借：原材料/库存商品
借：应交税费-进项税额
贷：应付账款-供应商
```

收款：

```text
借：银行存款-银行账号
贷：应收账款-客户
```

付款：

```text
借：应付账款-供应商
贷：银行存款-银行账号
```

折旧：

```text
借：制造费用/管理费用-折旧费-部门
贷：累计折旧
```

### 测试用例

#### 用例 1：销售到收款闭环

测试数据：

- 客户 A。
- 销售产品 P100。
- 数量 10。
- 含税金额 11300。

预期结果：

- 应收单金额 11300。
- 应收凭证生成并过账。
- 收款单 11300。
- 收款凭证生成并过账。
- 应收单和收款单核销。
- 客户 A 应收余额为 0。

#### 用例 2：采购到付款闭环

测试数据：

- 供应商 X。
- 材料 M100。
- 含税金额 56500。

预期结果：

- 应付单金额 56500。
- 应付凭证生成并过账。
- 付款单 56500。
- 付款凭证生成并过账。
- 应付单和付款单核销。
- 供应商 X 应付余额为 0。

#### 用例 3：固定资产折旧

测试数据：

- 设备原值 100000。
- 残值率 5%。
- 使用年限 5 年。

预期结果：

- 月折旧约 1583.33。
- 折旧费用进制造费用或管理费用。
- 固定资产报表原值和累计折旧对总账。

### 上线验收点

- 总账期初借贷平衡。
- 应收期初等于总账应收科目。
- 应付期初等于总账应付科目。
- 固定资产原值和累计折旧对总账。
- 销售、采购、收款、付款、折旧五条主流程跑通。
- 本月凭证全部审核过账。
- 资产负债表平衡，利润表收入成本费用正确。

### 项目中常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 应收期初和总账不一致 | 只导了总账，漏导客户明细 | 补导应收期初明细或调整总账 |
| 收款后账龄不减少 | 收款单未核销应收单 | 执行收款核销 |
| 折旧金额不对 | 资产卡片开始使用日期或折旧政策错误 | 调整资产卡片或折旧政策后重算 |
| 报表不平 | 新增科目未加报表公式 | 修改报表公式 |

## 案例 2：商贸企业应收应付治理项目

### 项目背景

某商贸企业销售客户多、收款频繁，供应商也多。上线前最大问题是：客户回款已到账，但应收账龄不准；供应商付款后，应付明细仍有大量未核销数据。

### 项目目标

- 建立客户和供应商往来明细。
- 收付款必须和应收应付核销。
- 应收账龄准确反映逾期。
- 每月底输出客户/供应商对账单。

### 关键方案

| 事项 | 方案 |
| --- | --- |
| 客户维度 | 应收账款、预收账款启用客户辅助核算 |
| 供应商维度 | 应付账款、预付账款启用供应商辅助核算 |
| 收款认领 | 收款单必须匹配客户和应收单 |
| 付款核销 | 付款单必须核销对应应付单 |
| 账龄口径 | 按到期日而不是单据日期分析 |
| 对账口径 | 子模块余额和总账科目余额必须一致 |

### 典型测试数据

客户 A：

- 1 月 5 日应收 100000，到期日 2 月 5 日。
- 1 月 20 日收款 60000。
- 2 月 10 日收款 40000。

预期：

- 1 月 31 日客户 A 未收 40000。
- 2 月 5 日后如果未收，40000 进入逾期。
- 2 月 10 日核销后余额 0。

供应商 B：

- 1 月 8 日应付 80000，到期日 2 月 8 日。
- 1 月 25 日付款 30000。
- 2 月 15 日付款 50000。

预期：

- 1 月 31 日供应商 B 未付 50000。
- 2 月 15 日核销后余额 0。

### 实操步骤

1. 清理客户和供应商主数据，合并重复档案。
2. 检查往来科目辅助核算。
3. 导入应收应付期初明细。
4. 配置应收收款核销方案。
5. 配置应付付款核销方案。
6. 培训财务：收付款后必须核销。
7. 每周检查未核销收款和付款。
8. 月末输出账龄和对账单。

### 常见问题

#### 客户重复导致对账困难

现象：客户 A 同时存在“A 公司”“A有限公司”“A客户”三个档案。

处理：

- 建立客户主数据治理表。
- 确定唯一客户编码。
- 停用重复客户。
- 历史单据按项目策略处理，必要时做余额转移或对账说明。

#### 总账有余额，子模块没有

原因：财务手工录了应收或应付科目凭证。

处理：

- 查手工凭证。
- 判断是否要在子模块补单。
- 后续限制往来科目手工制证权限。

## 案例 3：月结提速项目，从 10 天缩短到 3 天

### 项目背景

某企业月结慢，主要卡在未审核单据、未生成凭证、应收应付对账、固定资产折旧和报表不平。财务每月 10 号才能出报表。

### 项目目标

- 每月 3 号完成总账结账。
- 每月 4 号输出财务报表。
- 月结异常有清单和责任人。

### 月结排程

| 时间 | 工作 | 责任人 |
| --- | --- | --- |
| D-3 | 清理采购、销售、库存未审核单据 | 业务部门 |
| D-2 | 应收应付核销、银行流水导入 | 往来会计、出纳 |
| D-1 | 固定资产变更和折旧准备 | 资产会计 |
| D | 库存关账、存货核算、成本核算 | 成本会计 |
| D+1 | 业务凭证生成、审核、过账 | 总账会计 |
| D+2 | 调汇、自动转账、结转损益 | 总账会计 |
| D+3 | 总账结账、报表出具 | 财务经理 |

### 系统控制

- 各模块建立未审核单据过滤方案。
- 凭证生成情况查询按单据类型检查。
- 凭证列表固定查询未审核、未过账凭证。
- 应收应付每月输出“子模块 vs 总账”差异表。
- 固定资产结账前输出资产台账对总账表。
- 报表公式变更必须记录。

### 月结异常案例

#### 异常 1：总账结账提示存在未过账凭证

排查：

1. 凭证列表筛选本期未过账。
2. 找到应付模块生成的凭证 5 张未审核。
3. 应付会计确认单据无误。
4. 总账审核并过账。
5. 重新执行结账。

#### 异常 2：资产负债表不平 3000

排查：

1. 科目余额表借贷平衡。
2. 报表不平，说明更可能是公式问题。
3. 查本月新增“其他流动资产-待认证进项税”科目。
4. 报表公式未包含该科目。

处理：

- 把新增科目加入资产负债表对应项目。
- 重新计算报表。
- 把“新增科目检查报表公式”加入月结清单。

### 交付成果

- 月结时间表。
- 未审核单据清单。
- 未生成凭证清单。
- 未过账凭证清单。
- 子模块对总账对账表。
- 报表公式变更记录。

## 案例 4：固定资产上线项目

### 项目背景

企业以前用 Excel 管理固定资产，资产数量 1200 条，存在部门不准确、折旧年限不统一、已报废资产仍在折旧的问题。

### 项目目标

- 建立固定资产卡片。
- 资产原值、累计折旧、净值对总账。
- 月末自动计提折旧并生成凭证。
- 按部门输出资产和折旧报表。

### 数据清理

| 字段 | 清理要求 |
| --- | --- |
| 资产编码 | 唯一，不重复 |
| 资产名称 | 规范命名 |
| 资产类别 | 决定科目和折旧规则 |
| 使用部门 | 决定费用归属 |
| 开始使用日期 | 影响折旧起点 |
| 原值 | 对总账固定资产 |
| 累计折旧 | 对总账累计折旧 |
| 净值 | 原值 - 累计折旧 |
| 使用状态 | 在用、闲置、报废 |

### 折旧政策设计

| 资产类别 | 折旧方法 | 年限 | 残值率 | 费用科目 |
| --- | --- | --- | --- | --- |
| 房屋建筑物 | 年限平均法 | 20 年 | 5% | 管理费用/制造费用 |
| 机器设备 | 年限平均法 | 10 年 | 5% | 制造费用 |
| 办公设备 | 年限平均法 | 5 年 | 5% | 管理费用 |
| 运输设备 | 年限平均法 | 4 年 | 5% | 销售费用/管理费用 |

### 上线测试

测试资产：

- 机器设备 A。
- 原值 100000。
- 累计折旧 20000。
- 剩余使用期间 48 个月。
- 使用部门：生产部。

预期：

- 卡片净值 80000。
- 本月折旧按政策计算。
- 折旧费用进入制造费用-生产部。
- 累计折旧增加。
- 固定资产台账和总账一致。

### 常见问题

#### 折旧费用进错部门

原因：资产卡片使用部门或费用分配部门错误。

处理：做资产变更，必要时调整折旧凭证。

#### 净值和 Excel 不一致

原因：历史 Excel 折旧算法和系统折旧政策不同，或累计折旧录入错误。

处理：先以总账为控制数，再逐条核对资产卡片；必要时按上线口径调整累计折旧。

## 案例 5：业财一体化凭证模板优化项目

### 项目背景

企业已经上线金蝶，但大量凭证仍由财务手工录入。问题包括：凭证录入慢、辅助核算漏填、应收应付和总账不一致、业务单据难追溯。

### 项目目标

- 80% 以上业务凭证由系统自动生成。
- 凭证能追溯到业务单据。
- 应收、应付、出纳、资产凭证模板稳定。
- 手工凭证只用于少量调整。

### 模板优化范围

| 单据 | 凭证模板 |
| --- | --- |
| 应收单 | 确认收入和应收 |
| 收款单 | 确认银行收款和冲应收 |
| 应付单 | 确认采购、费用和应付 |
| 付款单 | 确认银行付款和冲应付 |
| 资产卡片 | 确认固定资产入账 |
| 折旧单 | 计提折旧 |
| 费用报销单 | 确认费用和应付/付款 |

### 模板设计方法

1. 先列业务场景，不先写科目。
2. 每个场景写理论分录。
3. 找出科目来源：固定、物料、客户、供应商、部门、费用项目。
4. 找出金额来源：含税、不含税、税额、折旧额、付款金额。
5. 找出辅助核算来源。
6. 用真实业务单据测试。
7. 财务确认后冻结模板。

### 示例：费用报销模板

场景：员工报销差旅费，先挂其他应付款。

```text
借：销售费用/管理费用-差旅费-部门
贷：其他应付款-员工
```

字段来源：

| 凭证字段 | 来源 |
| --- | --- |
| 费用科目 | 费用项目映射 |
| 部门维度 | 费用承担部门 |
| 员工维度 | 报销人 |
| 金额 | 审批通过金额 |

### 验收标准

- 关键业务单据都能生成凭证。
- 凭证借贷平衡。
- 科目正确。
- 辅助核算完整。
- 摘要能看懂业务。
- 凭证能反查源单。
- 子模块和总账对账一致。

### 项目经验

凭证模板不是一次配置完就结束。上线后第一个月要每天收集生成失败原因，按失败原因分组处理：科目映射缺失、维度缺失、金额字段错误、业务类型没覆盖、单据异常。稳定后再把模板变更纳入正式发布流程。
