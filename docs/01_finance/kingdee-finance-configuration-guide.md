这篇按项目实施顺序整理：从组织、账簿、科目、核算维度，到应收应付、固定资产、凭证模板、报表和权限。它更像一份配置蓝图。

## 配置总顺序

```text
组织和核算体系
  -> 会计政策和会计日历
  -> 账簿
  -> 科目表和科目
  -> 核算维度
  -> 币别和汇率
  -> 凭证字和凭证参数
  -> 应收应付基础设置
  -> 出纳和银行账号
  -> 固定资产基础设置
  -> 智能会计平台/凭证模板
  -> 报表公式
  -> 期初初始化
  -> 业务测试
  -> 月结演练
```

## 第一步：组织和核算体系

要先回答：谁独立核算，谁只是业务组织。

| 问题 | 配置对象 | 判断标准 |
| --- | --- | --- |
| 哪些公司要出财务报表 | 核算组织 | 需要独立资产负债表和利润表 |
| 哪些组织只做采购销售库存 | 业务组织 | 不单独出财务报表 |
| 是否集团合并 | 核算体系、组织关系 | 是否需要内部交易和合并抵消 |

项目检查：

- 核算组织是否和法人主体一致。
- 工厂、门店、事业部是否独立核算。
- 业务组织和财务组织是否存在跨组织交易。

## 第二步：会计政策和会计日历

会计政策决定账簿核算的基本规则。

| 配置 | 影响 |
| --- | --- |
| 会计日历 | 期间、结账、报表期间 |
| 本位币 | 总账本位币金额 |
| 科目表 | 凭证可用科目 |
| 会计准则 | 报表和核算口径 |
| 启用期间 | 初始化和首期业务 |

检查点：

- 启用期间一旦错误，后面反复调整成本很高。
- 多账簿时要确认每个账簿的会计政策是否一致。
- 会计日历要和企业实际月结周期一致。

## 第三步：账簿

账簿是一套财务核算数据的归集口径。

| 账簿设计 | 适用场景 |
| --- | --- |
| 单主账簿 | 大多数普通企业 |
| 主账簿 + 副账簿 | 管理口径、税务口径、不同准则 |
| 多组织多账簿 | 集团、多法人、多核算主体 |

配置检查：

- 账簿关联核算组织。
- 账簿关联会计政策。
- 本位币和启用期间正确。
- 账簿是否允许业务系统生成凭证。

## 第四步：科目和核算维度

科目表导入后，要逐项检查辅助核算。

| 科目 | 是否建议辅助核算 | 维度 |
| --- | --- | --- |
| 应收账款 | 是 | 客户 |
| 预收账款 | 是 | 客户 |
| 应付账款 | 是 | 供应商 |
| 预付账款 | 是 | 供应商 |
| 其他应收款 | 常见需要 | 员工、客户、部门 |
| 其他应付款 | 常见需要 | 员工、供应商、部门 |
| 银行存款 | 建议 | 银行账号 |
| 管理费用 | 建议 | 部门、费用项目 |
| 销售费用 | 建议 | 部门、费用项目 |
| 主营业务收入 | 视管理需要 | 产品线、部门、项目 |
| 主营业务成本 | 视管理需要 | 产品线、部门、项目 |

项目经验：

- 不要为了精细而盲目加维度。维度越多，单据和凭证越容易缺字段。
- 但往来科目必须保留客户/供应商，否则后续对账困难。
- 费用类科目至少要考虑部门维度。

## 第五步：应收应付配置

### 应收配置

| 配置 | 要点 |
| --- | --- |
| 客户 | 编码唯一，结算客户明确 |
| 收款条件 | 决定到期日和账龄 |
| 应收确认口径 | 销售出库、销售发票、应收单 |
| 核销方案 | 收款如何匹配应收 |
| 凭证模板 | 应收、收入、销项税、客户维度 |

### 应付配置

| 配置 | 要点 |
| --- | --- |
| 供应商 | 编码唯一，结算供应商和收款方明确 |
| 付款条件 | 决定到期付款和账龄 |
| 应付确认口径 | 入库、发票、应付单 |
| 核销方案 | 付款如何匹配应付 |
| 凭证模板 | 存货/费用、进项税、应付、供应商维度 |

## 第六步：出纳和银行

| 配置 | 作用 |
| --- | --- |
| 银行账号 | 收付款和银行对账 |
| 结算方式 | 现金、银行转账、票据等 |
| 资金科目映射 | 决定收付款凭证银行存款科目 |
| 银行对账 | 银行流水和企业账匹配 |

检查点：

- 银行账号和银行存款科目是否一一对应。
- 是否需要银行账号辅助核算。
- 银行流水导入格式是否确认。
- 银行手续费、利息、扣款等非业务流水如何处理。

## 第七步：固定资产

| 配置 | 作用 |
| --- | --- |
| 资产类别 | 决定资产科目、累计折旧科目和折旧规则 |
| 折旧方法 | 年限平均法等 |
| 折旧政策 | 当月新增是否折旧、处置当月是否折旧 |
| 使用部门 | 决定折旧费用归属 |
| 资产状态 | 在用、闲置、报废等 |

上线前必须测试：

- 新增资产。
- 资产变更。
- 资产处置。
- 计提折旧。
- 折旧凭证生成。
- 资产台账对总账。

## 第八步：凭证模板

凭证模板是金蝶财务自动化的核心。

### 模板设计步骤

1. 列业务场景。
2. 写理论分录。
3. 确定科目来源。
4. 确定金额来源。
5. 确定核算维度来源。
6. 用真实单据测试。
7. 财务确认后发布。

### 模板测试清单

- [ ] 凭证能生成。
- [ ] 借贷平衡。
- [ ] 科目正确。
- [ ] 金额正确。
- [ ] 税额正确。
- [ ] 客户/供应商/部门/项目维度完整。
- [ ] 摘要能看懂。
- [ ] 凭证能反查源单。
- [ ] 重复生成有控制。

## 第九步：报表公式

报表公式必须和科目体系同步。

| 报表 | 检查点 |
| --- | --- |
| 资产负债表 | 新增资产、负债、权益科目是否纳入 |
| 利润表 | 新增收入、成本、费用科目是否纳入 |
| 现金流量表 | 现金流量项目是否完整 |
| 管理报表 | 部门、项目、产品线维度是否取到 |

项目要求：

- 新增科目必须同步检查报表公式。
- 新增损益科目必须同步检查损益结转规则。
- 报表公式变更要留痕。

## 第十步：期初初始化

初始化不只是录余额。要保证总账和子模块明细一致。

| 初始化内容 | 控制口径 |
| --- | --- |
| 总账科目余额 | 借贷平衡 |
| 应收期初 | 合计等于应收科目余额 |
| 应付期初 | 合计等于应付科目余额 |
| 现金银行期初 | 等于银行和现金盘点余额 |
| 固定资产卡片 | 原值、累计折旧、净值对总账 |
| 存货期初 | 数量金额对库存和存货科目 |

## 第十一步：测试场景

上线前至少跑这些测试：

| 场景 | 验收结果 |
| --- | --- |
| 销售到应收 | 应收单、收入凭证、客户余额正确 |
| 收款核销 | 收款凭证、核销、账龄正确 |
| 采购到应付 | 应付单、应付凭证、供应商余额正确 |
| 付款核销 | 付款凭证、核销、应付账龄正确 |
| 费用报销 | 费用科目、部门维度、付款正确 |
| 固定资产折旧 | 折旧金额、折旧凭证、资产对账正确 |
| 月结 | 子模块结账、总账结账、报表出具 |

## 配置交付物清单

- 财务组织和账簿方案。
- 科目表和辅助核算设计。
- 应收应付基础资料和核销方案。
- 银行账号和出纳方案。
- 固定资产折旧政策。
- 凭证模板清单。
- 报表公式清单。
- 期初导入模板。
- 测试用例和测试结果。
- 月结 SOP。
