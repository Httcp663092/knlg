税务发票管理连接业务、税务、应收应付、总账和档案。金蝶项目中，发票不是孤立单据：销售发票影响应收和收入确认，采购发票影响应付、进项税和暂估冲回；红字发票、折扣、跨期开票、发票匹配都会影响月结和对账。

## 一、适用范围

- 销售开票和应收确认。
- 采购收票和应付确认。
- 发票和业务单据匹配。
- 红字发票和退货。
- 税额、税率、价税合计核对。
- 发票平台或税务系统集成。
- 发票附件和电子档案归档。

## 二、核心概念

| 概念 | 说明 |
| --- | --- |
| 销售发票 | 对客户开具的发票，通常影响应收和收入 |
| 采购发票 | 从供应商取得的发票，通常影响应付和进项税 |
| 蓝字发票 | 正常发票 |
| 红字发票 | 冲销或退货发票 |
| 价税合计 | 含税金额 |
| 不含税金额 | 计税基础 |
| 税额 | 增值税等税额 |
| 发票匹配 | 发票和订单、出入库、应收应付的关联 |

## 三、销售发票流程

```text
销售订单
  -> 销售出库
  -> 应收单
  -> 销售发票
  -> 收款
  -> 核销
  -> 凭证
```

常见开票口径：

- 按销售订单开票。
- 按销售出库开票。
- 按应收单开票。
- 先开票后发货。
- 分批开票。

项目中必须确认收入确认时点：是按出库、按发票、按应收，还是按合同履约。

## 四、采购发票流程

```text
采购订单
  -> 采购入库
  -> 暂估应付
  -> 采购发票
  -> 应付确认
  -> 付款
  -> 核销
  -> 凭证
```

采购发票关注：

- 发票供应商是否和采购供应商一致。
- 发票金额是否和入库金额一致。
- 税率和税额是否正确。
- 发票跨月时暂估如何处理。
- 部分到票和部分付款如何处理。

## 五、税率和金额校验

发票金额常用关系：

```text
价税合计 = 不含税金额 + 税额
税额 = 不含税金额 * 税率
```

检查点：

- 税率是否来自物料、客户、供应商或业务类型。
- 含税单价和不含税单价换算是否正确。
- 小数精度和尾差处理是否统一。
- 红字发票金额方向是否正确。
- 外币发票汇率是否正确。

## 六、发票匹配

### 销售侧匹配

销售发票要能追溯：

- 客户。
- 销售订单。
- 销售出库。
- 应收单。
- 收款单。
- 凭证。

如果开票数量大于出库数量，可能导致收入、应收和库存成本期间错配。

### 采购侧匹配

采购发票要能追溯：

- 供应商。
- 采购订单。
- 采购入库。
- 应付单。
- 付款单。
- 凭证。

采购入库和发票跨期时，要重点检查暂估和暂估冲回。

## 七、红字发票和退货

### 销售红冲

```text
客户退货
  -> 销售退货
  -> 红字应收或红字发票
  -> 收款退款或核销调整
  -> 红字凭证
```

### 采购红冲

```text
采购退料
  -> 红字采购发票
  -> 红字应付
  -> 付款退款或核销调整
  -> 红字凭证
```

红字处理要保留原单关联，避免只做手工凭证导致业务和总账不一致。

## 八、和总账凭证的关系

常见凭证方向：

销售确认：

```text
借：应收账款
  贷：主营业务收入
  贷：应交税费-销项税
```

采购到票：

```text
借：库存商品 / 原材料
借：应交税费-进项税
  贷：应付账款
```

如果存在暂估：

```text
采购入库暂估：
借：原材料
  贷：应付暂估

收到发票后：
借：应付暂估
借：应交税费-进项税
  贷：应付账款
```

实际科目和凭证模板以企业会计政策为准。

## 九、发票集成

发票平台或税务系统集成要关注：

- 发票号码。
- 发票代码。
- 开票日期。
- 购买方和销售方税号。
- 不含税金额。
- 税额。
- 价税合计。
- 红蓝字关系。
- 发票状态。
- 附件或版式文件。

接口必须有幂等，避免重复生成发票或重复匹配。

## 十、常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 发票金额和应收不一致 | 开票口径、折扣、税率不同 | 固定开票来源和金额规则 |
| 采购发票无法匹配入库 | 供应商、物料、数量、组织不一致 | 查采购订单和入库关联 |
| 税额尾差 | 小数精度和分录分摊规则 | 统一尾差处理 |
| 红字发票无法冲原票 | 原票关联缺失或状态不允许 | 检查红蓝字关系 |
| 总账税金不对 | 凭证模板或税额字段来源错误 | 回到模板和单据查来源 |

## 十一、实际案例

### 案例：采购入库 100 万，发票跨月到票

背景：

- 6 月采购入库含税 100 万。
- 发票 7 月才收到。

处理：

1. 6 月采购入库生成暂估应付。
2. 6 月月结时存货和暂估入账。
3. 7 月收到采购发票。
4. 系统按发票确认进项税和应付。
5. 暂估冲回或差异调整。
6. 应付和总账对账。

检查：

- 6 月存货金额正确。
- 7 月进项税入账正确。
- 暂估余额已冲回。
- 发票和入库有可追溯关联。
