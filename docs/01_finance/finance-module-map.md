这份地图用于快速理解金蝶财务模块之间的关系。

## 一句话理解

金蝶财务模块不是孤立录凭证，而是把采购、销售、库存、资产、费用、资金等业务数据逐步转成财务确认、往来核销、总账凭证和财务报表。

## 主线流程

```text
基础资料
  -> 业务单据
  -> 应收/应付/出纳/资产等子模块处理
  -> 智能会计平台或总账生成凭证
  -> 凭证审核、出纳复核、过账
  -> 子模块结账
  -> 总账期末处理
  -> 财务报表与管理分析
```

## 模块关系

| 上游 | 财务模块 | 结果 |
| --- | --- | --- |
| 销售出库、销售订单、销售发票 | 应收款管理 | 确认客户债权、收款、核销、账龄 |
| 采购入库、采购订单、采购发票 | 应付款管理 | 确认供应商债务、付款、核销 |
| 现金、银行、票据 | 出纳/资金 | 收付款流水、银行对账、资金监控 |
| 资产采购、资产变动 | 固定资产 | 资产卡片、折旧、处置、资产账务 |
| 各业务子系统 | 总账 | 凭证、账簿、科目余额、报表 |

## 财务实施时最容易混淆的概念

- 核算组织：从会计核算角度承担账务的组织。
- 账簿：一套财务数据的归集口径，决定会计政策、科目表、币别和启用期间。
- 科目表：会计科目的体系。
- 核算维度：客户、供应商、部门、项目、物料等辅助核算维度。
- 业务单据：销售、采购、库存、费用、资产等业务发生的原始记录。
- 财务单据：应收单、应付单、收付款单等财务确认记录。
- 凭证：进入总账核算的会计分录。

## 学习路线

1. 先看总账：理解账簿、科目、凭证、过账和结账。
2. 再看应收应付：理解往来确认、收付款、核销和账龄。
3. 再看出纳与资金：理解现金银行、流水和对账。
4. 再看固定资产：理解资产卡片、折旧、变更和处置。
5. 最后看月结：把所有子模块串起来。
