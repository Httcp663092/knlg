这篇给没有系统财务背景、但需要做金蝶项目或使用金蝶财务模块的人看。目标不是把会计教材重写一遍，而是把基础财务知识和金蝶里的系统对象对应起来。

## 财务系统到底在做什么

企业每天发生业务：采购、销售、收款、付款、报销、资产购置、折旧、发工资、缴税。财务系统要把这些业务变成可以核算、可以追溯、可以出报表的数据。

```text
业务发生
  -> 业务单据
  -> 财务确认
  -> 会计凭证
  -> 总账
  -> 报表
  -> 管理分析和审计追溯
```

在金蝶里常见落点：

| 业务动作 | 金蝶单据/对象 | 财务结果 |
| --- | --- | --- |
| 卖货 | 销售出库、应收单 | 确认收入和应收 |
| 收客户钱 | 收款单 | 银行增加，应收减少 |
| 买材料 | 采购入库、应付单 | 存货或费用增加，应付增加 |
| 付供应商钱 | 付款单 | 银行减少，应付减少 |
| 员工报销 | 费用报销单、其他应付单、付款单 | 费用增加，银行或应付变化 |
| 买设备 | 固定资产卡片 | 固定资产增加 |
| 每月折旧 | 折旧单 | 费用增加，累计折旧增加 |
| 月末结账 | 总账结账、报表 | 固定当期财务结果 |

## 会计六要素

会计六要素是资产、负债、所有者权益、收入、费用、利润。理解它们，可以更快看懂金蝶里的科目和报表。

| 要素 | 通俗解释 | 金蝶常见对象 | 报表位置 |
| --- | --- | --- | --- |
| 资产 | 企业拥有或控制的资源 | 银行账户、应收单、存货、固定资产卡片 | 资产负债表左边 |
| 负债 | 企业欠别人的钱或义务 | 应付单、付款计划、借款 | 资产负债表右边 |
| 所有者权益 | 老板或股东投入和留存收益 | 实收资本、资本公积、未分配利润科目 | 资产负债表右边 |
| 收入 | 经营活动带来的经济利益 | 应收单、收入凭证 | 利润表 |
| 费用 | 为取得收入发生的耗费 | 费用报销、折旧、成本费用凭证 | 利润表 |
| 利润 | 收入减成本费用后的结果 | 损益结转、本年利润 | 利润表/权益 |

## 借贷记账基础

借和贷不是“增加”和“减少”的固定意思，要看科目类型。

| 科目类型 | 借方表示 | 贷方表示 |
| --- | --- | --- |
| 资产类 | 增加 | 减少 |
| 负债类 | 减少 | 增加 |
| 所有者权益类 | 减少 | 增加 |
| 收入类 | 减少或结转 | 增加 |
| 成本费用类 | 增加 | 减少或结转 |

## 常见业务分录的理解

### 销售产生应收

客户买东西但还没付款，企业资产增加，因为多了一笔“应收账款”；同时收入增加，税也增加。

```text
借：应收账款
贷：主营业务收入
贷：应交税费-销项税额
```

金蝶落地：

- 销售业务生成应收单。
- 应收单生成凭证。
- 应收账款科目带客户维度。
- 收入科目可带部门、项目、产品线维度。

### 客户付款

客户付款后，企业银行存款增加，应收账款减少。

```text
借：银行存款
贷：应收账款
```

金蝶落地：

- 录入或生成收款单。
- 收款单生成凭证。
- 收款单和应收单核销。
- 银行日记账和银行流水对账。

### 采购产生应付

企业买材料或服务但还没付款，资产或费用增加，同时欠供应商的钱增加。

```text
借：原材料/库存商品/费用
借：应交税费-进项税额
贷：应付账款
```

金蝶落地：

- 采购入库、发票或应付单确认应付。
- 应付单生成凭证。
- 应付账款科目带供应商维度。

### 付款给供应商

企业支付货款，银行存款减少，应付账款减少。

```text
借：应付账款
贷：银行存款
```

金蝶落地：

- 付款单生成凭证。
- 付款单和应付单核销。
- 应付账龄和供应商对账单更新。

### 计提折旧

固定资产使用过程中价值逐步转移到费用，资产原值不直接减少，而是通过累计折旧反映。

```text
借：制造费用/管理费用/销售费用
贷：累计折旧
```

金蝶落地：

- 固定资产模块计提折旧。
- 折旧单生成凭证。
- 折旧费用按使用部门或费用承担部门入账。

## 科目、核算维度和报表

科目解决“是什么类型的钱”，核算维度解决“是谁、哪个部门、哪个项目、哪个产品线的钱”。

| 查询需求 | 只靠科目够不够 | 金蝶建议 |
| --- | --- | --- |
| 看总应收余额 | 基本够 | 查应收账款科目余额 |
| 看每个客户欠款 | 不够 | 应收账款启用客户辅助核算 |
| 看每个部门费用 | 不够 | 费用科目启用部门维度 |
| 看每个项目利润 | 不够 | 收入、成本、费用都带项目维度 |
| 看每个银行账号余额 | 不够 | 银行存款启用银行账号维度 |

## 三大报表的关系

### 资产负债表

看企业某一天的财务状况。

```text
资产 = 负债 + 所有者权益
```

金蝶关注：

- 资产负债表公式是否覆盖全部资产、负债、权益科目。
- 新增科目后是否更新公式。
- 总账凭证是否全部过账。

### 利润表

看一段期间赚了多少钱。

```text
利润 = 收入 - 成本 - 费用
```

金蝶关注：

- 收入、成本、费用凭证是否完整。
- 损益结转是否完成。
- 新增损益科目是否进入利润表公式。

### 现金流量表

看现金从哪里来，到哪里去。

金蝶关注：

- 现金流量项目是否维护。
- 涉及现金银行科目的凭证是否指定现金流量项目。
- 经营、投资、筹资分类是否正确。

## 从财务基础到金蝶操作的学习路线

1. 先学会看凭证：借方、贷方、科目、金额、辅助核算。
2. 再学会看单据：应收单、收款单、应付单、付款单、资产卡片。
3. 再学会看凭证来源：凭证是手工录入还是业务单据生成。
4. 再学会对账：子模块余额和总账科目余额是否一致。
5. 最后学月结：全部单据、凭证、对账、报表是否完成。

## 新手最容易踩的坑

| 坑 | 后果 | 正确做法 |
| --- | --- | --- |
| 只看凭证，不看源单 | 子模块和总账对不上 | 凭证要能追溯到业务单据 |
| 收款后不核销 | 账龄仍显示逾期 | 收款单必须核销应收单 |
| 付款后不核销 | 应付账龄仍显示未付 | 付款单必须核销应付单 |
| 新增科目不改报表 | 报表漏数或不平 | 新增科目同步检查报表公式 |
| 往来科目手工制证 | 应收应付和总账差异 | 往来业务优先走应收应付模块 |
| 固定资产只录总账 | 折旧和资产台账断层 | 必须录资产卡片 |
| 月末才查错误 | 结账被卡住 | 每天查未审核、未生成凭证、未过账 |

## 练习题

### 练习 1

客户 A 购买商品，价税合计 11300，其中不含税收入 10000，销项税 1300，尚未付款。

应该在金蝶里看到：

- 应收单金额 11300。
- 凭证借应收账款 11300。
- 贷主营业务收入 10000。
- 贷销项税额 1300。
- 应收账款带客户 A 维度。

### 练习 2

客户 A 后续付款 11300。

应该在金蝶里看到：

- 收款单金额 11300。
- 凭证借银行存款 11300，贷应收账款 11300。
- 收款单和应收单核销。
- 客户 A 应收余额为 0。

### 练习 3

公司买入设备原值 100000，残值率 5%，使用 5 年。

应该能算出：

```text
月折旧 = 100000 * (1 - 5%) / 60 = 1583.33
```

应该在金蝶里检查：

- 资产卡片原值 100000。
- 折旧方法和使用期间正确。
- 折旧凭证借费用、贷累计折旧。
- 固定资产台账和总账一致。
