## 适用范围

适用于销售应收、其他应收、收款、退款、核销、账龄分析和客户对账。

## 核心目标

应收款管理的目标是确认客户债权，跟踪到期收款，完成收款核销，并在期末与总账和客户往来数据保持一致。

## 业务主线

```text
销售业务/其他业务
  -> 应收单/其他应收单
  -> 收款单/收款退款单
  -> 核销
  -> 账务处理
  -> 期末处理
  -> 报表分析
```

## 基础设置

常见基础资料包括：

- 客户
- 币别
- 结算方式
- 物料
- 销售部门、销售组、销售员
- 组织信息
- 账簿信息
- 汇率体系
- 收款条件
- 应收收款核销方案
- 应收开票核销方案

## 初始化

应收初始化通常包括：

1. 设置启用日期。
2. 录入期初应收单、期初其他应收单、期初收款单、期初收款退款单。
3. 检查期初往来明细和总账余额。
4. 结束初始化。

## 日常业务

| 业务 | 单据 | 说明 |
| --- | --- | --- |
| 销售应收 | 应收单 | 确认销售业务形成的客户债权 |
| 其他应收 | 其他应收单 | 处理非销售形成的应收款 |
| 收款 | 收款单 | 记录客户回款 |
| 退款 | 收款退款单 | 处理退回客户款项 |
| 核销 | 应收与收款核销 | 确认债权与收款的匹配关系 |

## 销售应收完整案例

### 场景

客户 A 购买商品，含税销售金额 11300 元，税额 1300 元，不含税收入 10000 元，约定 30 天后付款。

### 系统处理

1. 销售业务形成应收单。
2. 应收单审核。
3. 生成应收凭证。
4. 客户付款后录入或生成收款单。
5. 收款单审核并生成收款凭证。
6. 应收单和收款单核销。
7. 应收账龄不再显示该笔逾期。

### 常见分录

确认应收：

```text
借：应收账款-客户 A        11300
贷：主营业务收入           10000
贷：应交税费-销项税额       1300
```

收到回款：

```text
借：银行存款               11300
贷：应收账款-客户 A        11300
```

### 金蝶检查点

- 应收单上的客户和结算客户是否是客户 A。
- 税率和价税合计是否正确。
- 应收科目是否带客户辅助核算。
- 收款单银行账号是否正确。
- 收款单是否和应收单完成核销。

## 应收账龄管理

账龄分析不是简单按凭证日期算，它通常和应收单日期、到期日、收款条件、核销状态有关。

| 账龄异常 | 排查 |
| --- | --- |
| 已收款仍显示逾期 | 查收款是否核销到应收单 |
| 到期日不对 | 查收款条件、应收单业务日期、到期日 |
| 账龄金额和余额不一致 | 查是否包含未审核单据、是否按同一币别 |
| 客户账龄串户 | 查客户、结算客户、收款客户是否一致 |

## 应收凭证模板设计要点

| 分录 | 科目来源 | 金额来源 | 辅助核算 |
| --- | --- | --- | --- |
| 借：应收账款 | 客户或业务类型映射 | 价税合计 | 客户 |
| 贷：主营业务收入 | 物料、收入类别、销售组织映射 | 不含税金额 | 部门、项目、物料可选 |
| 贷：销项税额 | 税率或税种映射 | 税额 | 可不启用或按要求启用 |

设计原则：

- 客户往来科目必须保留客户维度。
- 收入科目是否按物料、产品线、销售部门拆分，要在上线前确定。
- 税额取值要和发票及税务口径一致。

## 关键控制点

- 应收确认口径：应收单确认还是销售发票确认，要结合账簿参数和业务要求。
- 收款条件：影响到期日、账龄和催收。
- 核销方案：影响自动核销和匹配效率。
- 税额逻辑：关注含税价、不含税价、价内税、价外税。
- 多方交易：注意客户、结算客户、收款客户的区别。

## 报表与对账

常用报表包括：

- 应收款汇总表
- 应收款明细表
- 应收账龄分析表
- 到期债权表
- 往来对账明细表
- 应收单跟踪表
- 客户对账单
- 应收未开票明细表

## 常见问题

### 应收单和收款单无法自动核销

检查客户、结算组织、币别、金额、到期日、核销方案、单据状态和是否存在关联关系。

### 账龄不准

检查收款条件、到期日、核销状态、期初单据日期、账龄分析基准日期。

### 应收余额和总账不一致

检查本期应收单、收款单是否都生成凭证，凭证是否审核过账，以及是否存在手工凭证直接影响应收科目。
