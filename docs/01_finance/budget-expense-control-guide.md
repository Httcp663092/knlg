预算和费用控制是把财务从“事后记账”推进到“事前控制、事中预警、事后分析”的关键。金蝶系统里，预算可以控制采购、费用报销、付款、项目支出和部门费用，但预算方案如果设计不清，很容易变成形式化录入，既挡不住超支，也让业务操作变得很痛苦。

## 一、预算管理的目标

预算管理要解决：

- 钱花在哪里。
- 谁可以花。
- 什么时候可以花。
- 花多少算合理。
- 超预算谁审批。
- 执行结果和预算差异是什么原因。

预算不是财务一个部门的事情，它需要业务部门、项目负责人、采购、HR、管理层一起参与。

## 二、预算类型

| 类型 | 示例 | 控制对象 |
| --- | --- | --- |
| 部门费用预算 | 差旅费、办公费、招待费 | 部门、费用项目 |
| 项目预算 | 项目收入、成本、费用 | 项目 |
| 采购预算 | 采购金额、采购品类 | 采购组织、物料类别 |
| 资金预算 | 收款、付款、现金余额 | 银行账户、期间 |
| 资本预算 | 固定资产购置、工程支出 | 资产类别、项目 |
| 销售预算 | 销售收入、回款 | 客户、产品、区域 |

## 三、预算维度设计

预算维度常见有：

- 组织。
- 部门。
- 期间。
- 费用项目。
- 科目。
- 项目。
- 客户。
- 供应商。
- 物料类别。
- 员工。

维度越细，控制越精确，但维护成本越高。

### 费用预算推荐维度

```text
组织 + 部门 + 期间 + 费用项目
```

例如：

| 组织 | 部门 | 期间 | 费用项目 | 预算 |
| --- | --- | --- | --- | --- |
| A 公司 | 销售部 | 2026-06 | 差旅费 | 50,000 |
| A 公司 | 销售部 | 2026-06 | 招待费 | 20,000 |

## 四、预算编制

预算编制流程：

```text
预算模板下发
  -> 部门填报
  -> 财务初审
  -> 管理层审批
  -> 预算发布
  -> 预算控制
  -> 预算调整
  -> 执行分析
```

预算编制要明确：

- 编制周期：月度、季度、年度。
- 编制单位：部门、项目、组织。
- 编制口径：含税、不含税，是否含已签合同。
- 审批流程。
- 发布后是否允许修改。

## 五、预算控制方式

| 控制方式 | 说明 |
| --- | --- |
| 不控制 | 只做预算分析 |
| 提醒控制 | 超预算时提示，但允许继续 |
| 强控制 | 超预算不允许提交或审核 |
| 审批控制 | 超预算进入特殊审批 |
| 预占控制 | 申请或订单阶段占用预算 |
| 实耗控制 | 报销、付款或凭证阶段占用预算 |

项目中要避免一上来全部强控制，否则业务容易被卡死。可以先提醒，再逐步强控高风险费用。

## 六、费用报销控制

费用报销常见流程：

```text
费用申请
  -> 预算占用
  -> 审批
  -> 费用报销
  -> 发票校验
  -> 应付或付款
  -> 凭证
  -> 预算执行
```

控制点：

- 报销人。
- 部门。
- 费用项目。
- 项目。
- 发票。
- 金额标准。
- 预算余额。
- 审批权限。

## 七、采购预算控制

采购预算可以在：

- 采购申请。
- 采购订单。
- 采购入库。
- 应付确认。

不同阶段控制含义不同：

| 阶段 | 含义 |
| --- | --- |
| 采购申请 | 事前申请控制 |
| 采购订单 | 合同或订单承诺控制 |
| 采购入库 | 实际到货控制 |
| 应付确认 | 财务确认控制 |

建议采购类预算在采购申请或采购订单阶段预占，避免等发票到财务时才发现超预算。

## 八、预算调整

预算调整必须有流程：

- 追加预算。
- 调减预算。
- 部门间调剂。
- 项目间调剂。
- 期间间结转。

调整记录要包括：

- 调整原因。
- 原预算。
- 调整金额。
- 调整后预算。
- 审批人。
- 生效期间。

## 九、预算执行分析

常用指标：

```text
预算执行率 = 实际发生 / 预算
预算余额 = 预算 - 实际发生
预算差异 = 实际发生 - 预算
```

分析维度：

- 部门。
- 费用项目。
- 项目。
- 期间。
- 负责人。

不要只看总执行率。某部门总预算没超，但招待费严重超支、培训费完全没用，也需要分析。

## 十、金蝶落地关注点

在金蝶中要确认：

- 预算组织。
- 预算期间。
- 预算维度。
- 预算项目和科目对应。
- 预算控制单据。
- 预算占用和释放规则。
- 超预算审批流程。
- 预算执行报表。
- 与凭证、费用报销、采购订单的关系。

## 十一、常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 预算控制不到单据 | 控制对象未配置或维度不一致 | 检查预算维度和单据字段 |
| 预算余额不对 | 预占未释放、单据反审核未回滚 | 查预算占用明细 |
| 超预算仍能提交 | 控制方式是提醒或未命中 | 检查控制策略 |
| 报销选不到预算项目 | 部门、期间、费用项目不匹配 | 检查预算发布和维度 |
| 执行分析和总账不一致 | 预算按申请占用，总账按凭证发生 | 明确执行口径 |

## 十二、实际案例：销售部差旅费预算控制

背景：

- 销售部每月差旅费预算 50,000。
- 报销时经常超支，月底才发现。

方案：

1. 预算维度设置为组织 + 部门 + 期间 + 费用项目。
2. 费用申请阶段预占预算。
3. 报销阶段按实际金额释放或补占。
4. 超过 90% 时提醒部门负责人。
5. 超过 100% 时走总经理审批。
6. 月末输出预算执行分析。

效果：

- 财务从事后发现变成事前预警。
- 业务部门能看到预算余额。
- 超支有审批证据。

- [ ] 预算控制单据已确认。
