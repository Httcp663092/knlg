多组织和多账簿是金蝶项目中最容易设计复杂、也最容易影响后续报表和月结的部分。单公司单账簿项目可以只关注凭证和报表；集团型企业必须先设计组织、核算体系、账簿、币别、内部交易和合并口径。

## 一、核心概念

| 概念 | 说明 |
| --- | --- |
| 业务组织 | 承担采购、销售、库存、生产等业务的组织 |
| 核算组织 | 承担会计核算责任的组织 |
| 库存组织 | 管理库存数量和仓库的组织 |
| 账簿 | 一套会计核算口径，包括科目、期间、币别、会计政策 |
| 主账簿 | 企业主要核算账簿 |
| 副账簿 | 其他准则、管理或税务口径 |
| 合并范围 | 集团报表纳入哪些公司 |

## 二、为什么需要多组织

常见原因：

- 集团有多个法人。
- 一个法人下有多个工厂。
- 销售公司和生产工厂分离。
- 共享采购、共享财务。
- 多利润中心独立考核。
- 跨组织调拨和内部结算。

## 三、为什么需要多账簿

常见原因：

- 法定账和管理账不同。
- 中国准则和国际准则不同。
- 税务口径和财务口径不同。
- 本位币和外币报表不同。
- 集团管理需要调整口径。

多账簿不是越多越好。每多一套账簿，就多一套科目、凭证、报表、月结和对账成本。

## 四、组织和账簿设计步骤

```text
梳理法人和经营单元
  -> 确认核算组织
  -> 确认业务组织和库存组织
  -> 确认账簿数量
  -> 确认科目表和币别
  -> 确认组织间交易
  -> 确认报表和合并口径
```

## 五、组织模型设计

### 简单模型

```text
A 公司
  -> A 公司核算组织
  -> A 公司业务组织
  -> A 公司库存组织
```

适合单法人、业务简单企业。

### 集团模型

```text
集团
  -> 销售公司
  -> 生产工厂
  -> 采购中心
  -> 仓储公司
```

需要明确：

- 谁对外签合同。
- 谁确认收入。
- 谁持有库存。
- 谁承担成本。
- 谁开票收款。
- 谁出具财务报表。

## 六、账簿设计

账簿配置要确认：

- 所属核算组织。
- 本位币。
- 会计日历。
- 启用期间。
- 科目表。
- 凭证字。
- 会计政策。
- 报表体系。
- 是否允许业务生成凭证。

## 七、多账簿凭证

同一业务可能需要生成不同账簿的凭证。

例如固定资产折旧：

- 法定账按税法或会计政策折旧。
- 管理账按内部管理口径折旧。

金蝶中要确认：

- 业务单据是否同时生成多账簿凭证。
- 副账簿是否需要独立凭证模板。
- 报表按哪个账簿出。
- 月结顺序如何安排。

## 八、组织间交易

跨组织业务必须处理：

- 库存转移。
- 内部结算。
- 内部应收应付。
- 内部收入成本。
- 集团合并抵消。

如果只做调拨不做结算，库存数量可能对了，但内部财务关系不完整。

## 九、集团合并视角

集团合并要关注：

- 各公司报表是否同一会计政策。
- 科目映射是否一致。
- 内部往来是否对平。
- 内部交易是否抵消。
- 少数股东权益是否处理。
- 外币报表是否折算。

金蝶知识库中可把合并作为后续专题继续加深，但多组织项目从一开始就要为合并预留口径。

## 十、常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 业务单据生成不到正确账簿 | 组织和账簿映射错误 | 查核算组织和账簿关系 |
| 报表组织口径不对 | 查询组织和核算组织混淆 | 明确报表组织口径 |
| 跨组织调拨只动库存不动财务 | 未配置组织间结算 | 补内部结算流程 |
| 副账簿凭证缺失 | 凭证模板或生成规则未配置 | 补副账簿模板 |
| 集团内部往来不平 | 双方凭证、期间、币别不同 | 做内部往来对账 |

## 十一、实际案例：销售公司和工厂分离

背景：

- B 销售公司对外签合同、收款。
- A 工厂生产和发货。
- 集团希望看到 A 工厂利润和 B 销售公司利润。

设计：

1. A 工厂和 B 销售公司分别作为核算组织。
2. A 工厂持有生产库存。
3. B 销售公司确认外部收入和应收。
4. A 工厂发货给 B 销售公司形成组织间结算。
5. A 工厂确认内部收入或成本转出。
6. B 销售公司确认内部采购成本。
7. 集团合并时抵消内部交易。

检查：

- 外部客户应收只在 B 公司。
- A 和 B 内部往来相等。
- B 对外收入和成本能反映销售利润。
- 集团报表能抵消内部交易。

## 十二、设计检查清单

- [ ] 法人和核算组织已确认。
- [ ] 业务组织、库存组织、核算组织关系已确认。
- [ ] 账簿数量和用途已确认。
- [ ] 本位币、会计期间、科目表已确认。
- [ ] 多账簿凭证生成规则已测试。
- [ ] 组织间结算规则已测试。
- [ ] 内部往来对账表已准备。
- [ ] 集团报表和合并口径已确认。
- [ ] 权限按组织隔离已测试。
