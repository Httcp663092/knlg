财务系统不是只要能录单、能出报表就算完成。真正上线后，最容易出问题的是权限、反审核、手工凭证、跨期调整、月结证据和数据追溯。这篇用于建立金蝶财务项目的内控意识。

## 内控目标

| 目标 | 在金蝶中的体现 |
| --- | --- |
| 业务真实 | 单据有来源、附件、审批和状态 |
| 记录完整 | 单据生成凭证，凭证审核过账 |
| 金额准确 | 金额、税额、汇率、数量正确 |
| 权责清晰 | 制单、审核、付款、过账职责分离 |
| 期间正确 | 业务日期、凭证日期、会计期间正确 |
| 可追溯 | 凭证能反查源单，源单能看流程和附件 |
| 可审计 | 调整、反审核、反结账、删除有记录 |

## 职责分离

| 职责 | 不建议同一人同时拥有 |
| --- | --- |
| 录入应收单 | 审核应收单、修改客户资料 |
| 录入付款单 | 审核付款单、维护供应商收款账户 |
| 出纳付款 | 付款审核、总账过账 |
| 制单 | 审核凭证 |
| 维护凭证模板 | 审核由该模板生成的凭证 |
| 反结账 | 日常制单和审核 |

项目配置时要和客户确认岗位：

- 总账会计。
- 应收会计。
- 应付会计。
- 出纳。
- 资产会计。
- 财务经理。
- 系统管理员。

## 权限控制

### 基础资料权限

重点控制：

- 客户。
- 供应商。
- 银行账号。
- 科目。
- 核算维度。
- 资产类别。
- 凭证模板。

风险：

- 客户或供应商被随意新增，导致重复档案。
- 银行账号被随意修改，付款风险高。
- 科目或维度被随意调整，报表口径失控。

### 单据权限

建议按岗位划分：

| 单据 | 录入 | 审核 | 反审核 |
| --- | --- | --- | --- |
| 应收单 | 应收会计 | 财务主管 | 财务主管或系统管理员 |
| 收款单 | 出纳/应收会计 | 财务主管 | 财务主管 |
| 应付单 | 应付会计 | 财务主管 | 财务主管 |
| 付款单 | 应付会计/出纳 | 财务经理 | 财务经理 |
| 固定资产卡片 | 资产会计 | 财务主管 | 财务主管 |
| 凭证 | 总账会计 | 财务主管 | 财务主管 |

## 手工凭证治理

手工凭证不是不能用，但要控制范围。

### 不建议手工制证的科目

- 应收账款。
- 预收账款。
- 应付账款。
- 预付账款。
- 固定资产。
- 累计折旧。
- 存货类科目。
- 银行存款。

原因：

- 这些科目通常有子模块台账。
- 手工凭证会造成子模块和总账不一致。
- 后续对账和审计追溯困难。

### 允许手工凭证的典型场景

- 管理调整。
- 税金计提。
- 工资社保计提。
- 损益调整。
- 无业务子模块承载的费用摊销。

要求：

- 摘要清楚。
- 附件完整。
- 审批完整。
- 辅助核算完整。
- 月末单独复核。

## 反审核、反过账、反结账控制

反向操作风险很高，必须有审批和留痕。

| 操作 | 风险 | 控制 |
| --- | --- | --- |
| 反审核单据 | 已确认业务被修改 | 需要主管审批 |
| 删除单据 | 业务记录丢失 | 原则上用作废或红冲 |
| 反过账凭证 | 总账余额变化 | 记录原因和凭证号 |
| 反结账 | 已关闭期间被改动 | 财务经理审批，记录影响 |
| 修改凭证模板 | 后续凭证口径变化 | 测试后发布，版本留痕 |

## 月结内控检查

### 结账前

- [ ] 本期业务单据无异常未审核。
- [ ] 业务单据已生成凭证。
- [ ] 凭证已审核。
- [ ] 凭证已过账。
- [ ] 应收和总账核对一致。
- [ ] 应付和总账核对一致。
- [ ] 银行和总账核对一致。
- [ ] 固定资产和总账核对一致。
- [ ] 报表公式已检查。

### 结账后

- [ ] 保存科目余额表。
- [ ] 保存应收账龄表。
- [ ] 保存应付账龄表。
- [ ] 保存银行对账结果。
- [ ] 保存固定资产折旧表。
- [ ] 保存资产负债表。
- [ ] 保存利润表。
- [ ] 保存差异说明。
- [ ] 保存调整凭证清单。

## 审计追溯路径

### 从报表追溯到凭证

```text
报表项目
  -> 报表公式
  -> 科目余额表
  -> 明细账
  -> 凭证
  -> 源单
  -> 审批流/附件
```

### 从业务单据追溯到报表

```text
业务单据
  -> 生成凭证
  -> 凭证审核过账
  -> 科目余额
  -> 报表项目
```

项目上线验收时，应该抽样做这两条追溯。

## 常见审计问题

### 问题 1：凭证没有源单

可能原因：

- 手工凭证。
- 业务凭证和源单关联丢失。
- 迁移历史凭证。

处理：

- 要求附件说明。
- 对往来、资产、存货科目重点复核。
- 后续业务尽量通过子模块生成凭证。

### 问题 2：报表数和明细账对不上

可能原因：

- 报表公式漏科目。
- 查询期间或账簿不同。
- 凭证未过账。
- 重分类规则影响。

处理：

- 先用科目余额表核对。
- 再查报表公式。
- 最后查凭证状态。

### 问题 3：月结后又改了历史期间

可能原因：

- 反结账权限过宽。
- 业务部门补录历史单据。
- 财务缺少调整流程。

处理：

- 收紧反结账权限。
- 建立历史期间调整审批。
- 用调整凭证替代直接改历史。

## 内控成熟度分级

| 等级 | 特征 |
| --- | --- |
| 初级 | 能录单、能生成凭证、能出报表 |
| 标准 | 权限分离、凭证审核、月结检查 |
| 稳定 | 子模块和总账定期对账，异常有清单 |
| 优秀 | 报表追溯完整，模板和权限变更有流程 |
| 成熟 | 自动监控异常，月结证据包标准化 |

## 项目交付建议

上线交付时，不只交配置，还要交这些内控材料：

- 权限矩阵。
- 岗位职责表。
- 反审核反结账流程。
- 手工凭证管理规则。
- 月结检查表。
- 报表公式维护规则。
- 凭证模板变更流程。
- 审计追溯样例。
