会计循环是理解财务软件的底层主线。无论是手工账、传统财务软件还是金蝶云，核心都是把经济业务按照会计规则记录、分类、汇总、调整、结账，最后形成财务报表。金蝶的价值在于把采购、销售、库存、资产、资金等业务数据自动转化为会计结果，但前提是你理解每一步在财务上意味着什么。

## 一、会计循环总览

```text
经济业务发生
  -> 原始单据
  -> 业务单据
  -> 财务确认
  -> 生成凭证
  -> 审核凭证
  -> 过账
  -> 试算平衡
  -> 期末调整
  -> 结转损益
  -> 结账
  -> 出具报表
```

在金蝶里，这条主线对应：

| 会计循环步骤 | 金蝶系统对象 | 示例 |
| --- | --- | --- |
| 经济业务发生 | 业务场景 | 采购入库、销售出库、收款付款 |
| 原始单据 | 附件、发票、合同、回单 | 发票 PDF、银行回单 |
| 业务单据 | 采购、销售、库存、资产单据 | 采购入库单、销售出库单 |
| 财务确认 | 应收、应付、出纳、资产 | 应收单、应付单、付款单 |
| 生成凭证 | 智能会计平台或凭证模板 | 业务凭证生成 |
| 审核凭证 | 总账凭证状态 | 审核、反审核 |
| 过账 | 总账余额更新 | 科目余额表更新 |
| 期末调整 | 手工凭证、计提、摊销 | 折旧、费用计提 |
| 结转损益 | 期末处理 | 收入费用转本年利润 |
| 结账 | 总账和子模块结账 | 关闭会计期间 |
| 出报表 | 报表系统 | 资产负债表、利润表 |

## 二、第一步：业务发生和原始凭据

财务核算不是从凭证开始，而是从业务和证据开始。

常见原始凭据：

- 合同。
- 采购订单。
- 送货单。
- 入库单。
- 销售出库单。
- 发票。
- 银行回单。
- 验收单。
- 报销单。
- 固定资产验收单。

金蝶里要关注：

- 原始凭据是否作为附件上传。
- 附件和业务单据是否关联。
- 发票、回单、合同是否能追溯到凭证。
- 审计时是否能从凭证反查原始资料。

## 三、第二步：业务单据和财务单据

业务单据记录业务事实，财务单据记录会计确认。

### 采购例子

```text
采购订单
  -> 采购入库单
  -> 应付单
  -> 采购发票
  -> 付款单
  -> 核销
  -> 凭证
```

采购入库说明货已经到，是否确认应付、是否确认进项税、是否暂估，要看企业的会计政策和发票情况。

### 销售例子

```text
销售订单
  -> 销售出库单
  -> 应收单
  -> 销售发票
  -> 收款单
  -> 核销
  -> 凭证
```

销售出库说明货已经发出，但收入确认要看合同履约、风险报酬转移、系统配置和企业政策。

## 四、第三步：凭证生成

凭证是业务进入总账的入口。

凭证核心要素：

- 日期。
- 凭证字和凭证号。
- 摘要。
- 科目。
- 借方金额。
- 贷方金额。
- 核算维度。
- 币别和汇率。
- 附件张数。
- 制单人、审核人、过账状态。

金蝶中凭证来源通常有：

| 来源 | 示例 | 风险 |
| --- | --- | --- |
| 业务生成 | 应收、应付、存货、资产生成凭证 | 模板配置错误会批量影响 |
| 手工录入 | 调整凭证、计提凭证 | 容易破坏子模块和总账一致 |
| 导入凭证 | 外部系统或 Excel 导入 | 辅助核算和附件容易缺失 |
| 接口凭证 | OA、费用系统、共享平台 | 幂等、审批、追溯要控制 |

## 五、第四步：审核和过账

审核和过账是两个概念：

- 审核：确认凭证内容被复核。
- 过账：凭证正式更新总账账簿余额。

常见误区：

| 误区 | 解释 |
| --- | --- |
| 凭证保存了就影响余额 | 未过账凭证通常不进入正式余额 |
| 审核等于过账 | 审核只是复核，过账才入账 |
| 删除凭证就能解决业务问题 | 如果来自业务单据，应回到源单和模板处理 |
| 手工改凭证最省事 | 可能导致业务模块和总账不一致 |

## 六、第五步：试算平衡

试算平衡检查借贷是否平衡。

```text
全部借方发生额 = 全部贷方发生额
全部借方余额 = 全部贷方余额
```

金蝶里常用：

- 科目余额表。
- 凭证汇总表。
- 试算平衡表。
- 辅助余额表。

试算平衡只能说明借贷平衡，不代表账务完全正确。例如把费用记到错误部门，试算仍然平衡，但管理报表错了。

## 七、第六步：期末调整

期末调整用于把权责发生制落实到正确期间。

常见期末调整：

| 类型 | 示例 |
| --- | --- |
| 计提 | 工资、奖金、水电费、利息 |
| 摊销 | 预付费用、长期待摊费用 |
| 折旧 | 固定资产折旧 |
| 减值 | 存货跌价、坏账准备 |
| 汇兑 | 外币应收应付、银行存款 |
| 暂估 | 已入库未到票 |
| 成本结转 | 销售成本、制造成本 |

金蝶里要确保：

- 调整凭证有依据。
- 调整金额有计算表。
- 调整凭证有附件。
- 次月是否需要冲回。
- 调整影响哪些报表和辅助维度。

## 八、第七步：结转损益

损益类科目期末要结转到本年利润。

```text
收入类科目
  -> 转入本年利润贷方

费用和成本类科目
  -> 转入本年利润借方
```

结转后：

- 损益类科目期末余额通常为 0。
- 本年利润反映当期利润。
- 利润表按发生额反映经营结果。

金蝶里要关注：

- 结转损益方案是否包含新增科目。
- 多账簿是否分别结转。
- 结转凭证是否审核过账。
- 结转后利润表是否正确。

## 九、第八步：结账

结账是关闭本期间，进入下一期间。

结账前要完成：

- 业务单据审核。
- 子模块结账。
- 凭证生成。
- 凭证审核过账。
- 期末调整。
- 折旧计提。
- 存货核算。
- 成本计算。
- 往来核销。
- 银行对账。
- 结转损益。
- 报表检查。

金蝶里通常先结子模块，再结总账。

```text
库存/存货/成本
  -> 应收应付
  -> 固定资产
  -> 出纳
  -> 总账
```

具体顺序以企业模块范围为准。

## 十、第九步：出具报表

报表是会计循环的结果。

| 报表 | 说明 |
| --- | --- |
| 资产负债表 | 某一时点的资产、负债、所有者权益 |
| 利润表 | 某一期间的收入、成本、费用、利润 |
| 现金流量表 | 某一期间现金流入和流出 |
| 科目余额表 | 科目期初、发生额、期末 |
| 辅助余额表 | 按客户、供应商、部门、项目等维度 |
| 应收账龄表 | 客户欠款账龄 |
| 应付账龄表 | 供应商欠款账龄 |

报表出错通常不是报表最后一步的问题，而是前面的单据、凭证、模板、科目、维度、期间或过账状态有问题。

## 十一、实际案例：一笔采购业务完整会计循环

业务：

- 6 月 5 日采购原材料 100,000 元，税额 13,000 元。
- 当月入库，发票 7 月收到。
- 7 月付款。

6 月处理：

1. 采购入库单审核。
2. 因发票未到，确认暂估。
3. 生成暂估凭证。

```text
借：原材料 100,000
  贷：应付暂估 100,000
```

7 月处理：

1. 收到采购发票。
2. 确认进项税和应付账款。
3. 冲回暂估。
4. 付款并核销。

可能凭证：

```text
借：应付暂估 100,000
借：应交税费-进项税 13,000
  贷：应付账款 113,000

借：应付账款 113,000
  贷：银行存款 113,000
```

金蝶检查：

- 采购入库是否审核。
- 暂估凭证是否生成。
- 发票是否匹配入库。
- 应付和付款是否核销。
- 应付账龄和总账是否一致。

## 十二、实际案例：一笔销售业务完整会计循环

业务：

- 6 月 10 日销售商品，含税 226,000 元。
- 不含税收入 200,000 元，销项税 26,000 元。
- 商品成本 120,000 元。
- 6 月收款。

流程：

1. 销售订单审核。
2. 销售出库审核。
3. 应收单确认。
4. 销售发票开具。
5. 收款单登记。
6. 收款核销。
7. 生成收入、成本、收款凭证。

凭证示例：

```text
借：应收账款 226,000
  贷：主营业务收入 200,000
  贷：应交税费-销项税 26,000

借：主营业务成本 120,000
  贷：库存商品 120,000

借：银行存款 226,000
  贷：应收账款 226,000
```

金蝶检查：

- 销售出库是否生成成本。
- 应收是否生成收入凭证。
- 收款是否核销应收。
- 应收余额是否为 0。
- 利润表收入和成本是否体现。

## 十三、学习检查清单

- [ ] 能说清业务单据和财务凭证的区别。
- [ ] 能从采购入库追到应付、付款、凭证和报表。
- [ ] 能从销售出库追到应收、收款、凭证和报表。
- [ ] 知道审核和过账的区别。
- [ ] 知道期末调整有哪些类型。
- [ ] 知道损益结转的目的。
- [ ] 知道结账前要检查哪些子模块。
- [ ] 知道报表差异要往前追源单和凭证。
