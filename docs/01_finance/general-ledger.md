## 适用范围

适用于理解金蝶总账的核心对象、日常凭证处理、期末处理和与其他模块的关系。

## 核心概念

总账是财务会计系统的核心，围绕凭证、账簿和账表进行管理。业务系统最终通过凭证进入总账，总账再输出科目余额、明细账、辅助账和财务报表。

## 总账关键对象

| 对象 | 作用 |
| --- | --- |
| 账簿 | 财务数据归集口径，关联核算组织、会计政策、科目表、本位币和启用期间 |
| 科目表 | 科目体系，决定凭证科目和报表取数基础 |
| 科目 | 会计核算最小分类，可启用辅助核算、数量金额、外币等属性 |
| 凭证字 | 记账凭证类别，例如记、收、付、转 |
| 凭证 | 会计分录，审核和过账后登记到账簿 |
| 核算维度 | 辅助核算口径，例如客户、供应商、部门、项目、物料 |

## 日常流程

1. 业务系统生成或手工录入凭证。
2. 财务人员查询和检查凭证。
3. 主管会计审核凭证。
4. 涉及现金或银行存款科目的凭证可进行出纳复核。
5. 凭证过账，登记到账簿。
6. 查询科目余额、明细账、辅助账和报表。

## 总账日常工作清单

| 动作 | 具体检查 | 结果 |
| --- | --- | --- |
| 查未生成凭证业务 | 按账簿、期间、单据类型查询凭证生成情况 | 业务单据没有遗漏 |
| 查未审核凭证 | 凭证列表按审核状态过滤 | 凭证可以进入过账 |
| 查未过账凭证 | 凭证列表按过账状态过滤 | 总账余额完整 |
| 查异常科目余额 | 科目余额表按方向、余额、辅助核算检查 | 尽早发现错账 |
| 查手工凭证 | 对应收、应付、资产、存货科目筛手工来源 | 防止子模块和总账不一致 |
| 查报表公式 | 对新增科目和损益结转科目检查公式 | 防止报表漏项 |

## 凭证检查方法

### 看来源

凭证来源决定排查路径：

- 来源于应收：回查应收单、其他应收单、收款单、收款退款单。
- 来源于应付：回查应付单、其他应付单、付款单、付款退款单。
- 来源于固定资产：回查资产卡片、折旧、变更、处置单据。
- 来源于库存或成本：回查出入库单、成本计算和存货核算。
- 手工凭证：看制单人、摘要、附件和审批说明。

### 看分录

凭证分录至少检查：

- 借贷是否平衡。
- 科目方向是否符合业务。
- 辅助核算维度是否完整。
- 币别、汇率、本位币金额是否正确。
- 摘要是否能说明业务。
- 是否跨期。

### 看状态

凭证状态会影响报表：

- 未审核：不能作为正式账务。
- 已审核未过账：仍未进入总账余额。
- 已过账：进入账簿和报表。
- 已作废或冲销：要看冲销关系和原凭证。

## 总账典型场景

### 新增科目后报表不显示

处理顺序：

1. 确认凭证已经使用新科目并过账。
2. 在科目余额表确认新科目有发生额或余额。
3. 检查资产负债表或利润表公式是否包含新科目。
4. 如果是重分类项目，检查重分类规则。
5. 重新计算报表。

### 损益结转后仍有损益余额

处理顺序：

1. 查损益类科目余额表。
2. 确认新增损益科目是否纳入结转方案。
3. 查结转凭证是否生成。
4. 查结转凭证是否审核、过账。
5. 必要时重新生成结转凭证。

### 期末调汇金额异常

处理顺序：

1. 确认科目是否启用外币核算。
2. 确认期末汇率是否维护。
3. 确认调汇范围是否包含对应科目。
4. 对比原币余额、本位币余额和期末汇率。
5. 生成调汇凭证后检查财务费用-汇兑损益。

## 期末处理

期末处理的重点不是简单点结账，而是确认所有业务已经完整进入总账。常见动作包括：

- 检查本期凭证是否全部审核、过账。
- 检查相关业务系统是否已结账。
- 检查应收、应付、出纳、固定资产、存货等是否已生成凭证并过账。
- 有外币业务时进行期末调汇。
- 根据模板生成自动转账凭证。
- 结转损益，并审核过账。
- 执行总账期末结账。

## 总账和其他模块的关系

- 应收应付负责往来确认和核销，总账负责最终会计分录。
- 出纳负责资金流水和银行对账，总账负责现金银行科目账务。
- 固定资产负责资产卡片和折旧，总账负责资产、累计折旧和费用凭证。
- 存货核算和成本核算完成后，相关凭证进入总账。

## 常见问题

### 科目余额和明细账不一致

优先确认查询条件是否一致，包括账簿、期间、币别、科目范围、是否包含未过账凭证、辅助核算维度。

### 凭证不能过账

检查凭证状态、审核状态、期间是否开放、科目是否禁用、辅助核算是否完整、借贷是否平衡。

### 结账提示业务系统未完成

按提示模块回查对应期间，优先检查未审核单据、未生成凭证单据、未过账凭证和子模块结账状态。
