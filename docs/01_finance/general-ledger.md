## 适用范围

这篇是金蝶财务总账的完整教程，适合三类人使用：

| 角色 | 你要重点掌握什么 |
| --- | --- |
| 财务新人 | 凭证、科目、账簿、过账、结账和账表查询的基础逻辑 |
| 总账会计 | 日常凭证处理、账表查询、期末处理、对账和异常排查 |
| 金蝶实施顾问 | 总账初始化、账簿方案、科目和核算维度设计、上线验收和月结闭环 |

总账不是一个孤立模块。应收、应付、出纳、固定资产、存货、成本、费用等业务模块，最终都要通过凭证进入总账。总账再输出科目余额、明细账、辅助账、试算平衡表和财务报表。

```text
业务单据
  -> 凭证模板或智能会计平台
  -> 总账凭证
  -> 凭证审核
  -> 凭证过账
  -> 科目余额和明细账
  -> 期末调汇、自动转账、损益结转
  -> 总账结账
  -> 财务报表和管理分析
```

## 一、先建立总账的大图

### 总账在金蝶里的位置

金蝶官方产品手册把总账定位为财务会计系统的核心，总账以凭证处理为中心，管理账簿和报表，并且和业务系统共享数据。理解这句话，可以拆成三层：

| 层级 | 系统对象 | 财务含义 |
| --- | --- | --- |
| 核算规则层 | 核算体系、核算组织、会计政策、会计日历、科目表、账簿 | 决定“谁用什么规则记账” |
| 业务处理层 | 凭证录入、凭证查询、凭证审核、出纳复核、凭证过账 | 决定“业务如何进入账簿” |
| 输出分析层 | 总分类账、明细账、科目余额表、试算平衡表、核算维度账表、报表 | 决定“如何查账和出报表” |

### 学总账要抓住五个问题

1. 谁在记账：核算组织和账簿。
2. 用什么口径记账：会计政策、会计日历、科目表、本位币。
3. 记到哪里：科目、核算维度、币别、现金流量项目。
4. 凭证怎么来：业务单据生成、自动转账、手工凭证、导入凭证。
5. 什么时候锁账：审核、过账、结账、反结账。

### 总账学习路线

```text
基础资料
  -> 初始化
  -> 凭证管理
  -> 账表查询
  -> 期末处理
  -> 对账和报表
  -> 权限和内控
  -> 异常排查
```

## 二、总账关键对象详解

### 账簿

账簿是一套财务数据的归集口径。金蝶账簿通常关联核算体系、核算组织、会计政策、会计日历、科目表、本位币、默认凭证字和启用期间。

| 字段 | 解释 | 实施注意 |
| --- | --- | --- |
| 核算体系 | 一套会计核算组织结构 | 集团、多法人、多业务组织时要先设计 |
| 核算组织 | 独立核算主体 | 一般对应需要独立出报表的公司 |
| 会计政策 | 会计制度、日历、本位币等规则 | 不同准则或不同管理口径可能需要不同政策 |
| 会计日历 | 会计期间划分 | 决定每月结账期间 |
| 科目表 | 可用科目体系 | 决定凭证和报表取数基础 |
| 账簿类型 | 主账簿或副账簿 | 主账簿用于主要核算，副账簿用于管理或准则差异 |
| 记账本位币 | 总账本位币 | 账表综合本位币通常按账簿本位币 |
| 默认汇率类型 | 外币折算规则 | 外币业务必须提前确认 |
| 默认凭证字 | 录凭证默认类别 | 可以按记、收、付、转或统一“记” |
| 启用期间 | 总账开始使用的期间 | 期初初始化和历史数据导入的边界 |

账簿设计最容易出错的是：把业务组织当成核算组织、账簿启用期间选错、主副账簿口径没说清、账簿本位币和会计政策本位币不一致、没有提前考虑应收应付确认方式。

### 科目表和科目

科目表是一套科目体系，科目是凭证分录的基本分类。

| 科目属性 | 用途 | 示例 |
| --- | --- | --- |
| 科目编码 | 科目层级和排序 | 1002 银行存款 |
| 科目类别 | 资产、负债、权益、收入、成本费用 | 应收账款属于资产类 |
| 余额方向 | 借方或贷方 | 资产类通常借方余额 |
| 辅助核算 | 科目之外的细分维度 | 应收账款挂客户 |
| 外币核算 | 原币和本位币并行 | 外币银行、外币应收 |
| 数量金额 | 金额之外还记录数量 | 存货类科目可能用 |
| 现金流量 | 是否影响现金流量表 | 银行存款、库存现金 |

科目设计要回答两个问题：

1. 财务报表要怎样看。
2. 管理分析要怎样看。

如果只按报表设计科目，后续部门、项目、客户、产品线分析会缺口径。如果把所有管理要求都拆成末级科目，科目表又会爆炸。所以金蝶项目里通常用“科目 + 核算维度”解决。

### 核算维度

核算维度是辅助核算口径。常见维度包括客户、供应商、部门、项目、员工、物料、银行账号、费用项目、资产类别。

| 管理问题 | 建议科目 | 建议维度 |
| --- | --- | --- |
| 哪个客户欠款 | 应收账款 | 客户 |
| 欠哪个供应商钱 | 应付账款 | 供应商 |
| 哪个部门花了费用 | 管理费用、销售费用 | 部门 |
| 哪个项目盈利 | 收入、成本、费用 | 项目 |
| 哪个银行账号有余额 | 银行存款 | 银行账号 |
| 哪个员工有借款 | 其他应收款 | 员工 |

维度设计原则：

- 能从业务单据稳定带出的维度，优先作为凭证辅助核算。
- 业务单据没有字段的维度，不要指望凭证模板凭空生成。
- 应收、应付、其他应收、其他应付这类往来科目，必须重点控制维度完整性。
- 维度一旦启用，历史凭证和报表查询都会受影响，上线后不要随便改。

### 凭证字

凭证字是凭证类别，例如记、收、付、转。金蝶项目里常见两种设计：

| 方式 | 优点 | 缺点 |
| --- | --- | --- |
| 统一使用“记”字 | 简单，系统生成凭证不用复杂区分 | 收付转分类不明显 |
| 使用收、付、转、记 | 传统会计习惯清晰 | 模板和凭证处理要配置更细 |

如果企业业务量大、凭证主要由系统自动生成，统一凭证字会更容易维护。如果企业仍保留传统手工会计习惯，可以按收付转设计，但要把凭证模板、出纳复核和权限同步考虑进去。

### 凭证

凭证是总账的核心单据。一个完整凭证至少包含：

- 账簿。
- 凭证日期和会计期间。
- 凭证字和凭证号。
- 摘要。
- 科目。
- 借贷方向。
- 金额。
- 辅助核算。
- 币别、汇率、原币、本位币。
- 制单人、审核人、过账状态。
- 来源单据或附件。

凭证不是录完就进入正式账。只有审核和过账后，才真正影响科目余额、明细账和报表。

## 三、总账基础设置教程

### 设置顺序

```text
币别和汇率体系
  -> 会计日历
  -> 会计要素
  -> 科目表和科目
  -> 核算维度
  -> 会计政策
  -> 会计核算体系
  -> 账簿
  -> 凭证字
  -> 现金流量项目
  -> 总账参数
```

### 币别和汇率体系

如果企业有外币银行、外币应收、外币应付、外币借款，就必须先设计币别和汇率。

检查项：

- 本位币是什么。
- 是否有美元、欧元、港币等外币业务。
- 汇率是按日、按月初、按月末，还是按业务发生日。
- 期末调汇使用什么汇率类型。
- 外币科目是否需要期末调汇。

### 会计日历

会计日历决定期间。大多数企业是一年 12 期，也可能有 13 期或调整期间。

检查项：

- 年度起止日期。
- 每期起止日期。
- 是否允许调整期间。
- 是否和集团报表期间一致。
- 是否和税务申报期间一致。

### 科目表

科目表要先满足准则和报表，再满足管理分析。

实施步骤：

1. 确认企业适用会计准则。
2. 导入或建立标准科目表。
3. 根据企业业务补充明细科目。
4. 标记哪些科目启用辅助核算。
5. 标记哪些科目启用外币、数量金额、现金流量。
6. 和财务确认科目编码、名称、类别、余额方向。
7. 用典型业务分录测试科目是否够用。

### 科目辅助核算设计

| 科目 | 推荐辅助核算 | 原因 |
| --- | --- | --- |
| 应收账款 | 客户 | 客户对账、账龄、催收 |
| 预收账款 | 客户 | 预收转应收、客户余额 |
| 应付账款 | 供应商 | 供应商对账、付款计划 |
| 预付账款 | 供应商 | 预付核销 |
| 其他应收款 | 员工、客户、其他往来单位 | 借款和往来清理 |
| 其他应付款 | 员工、供应商、其他往来单位 | 报销和临时往来 |
| 银行存款 | 银行账号 | 银行对账和资金余额 |
| 管理费用 | 部门、项目、费用项目 | 部门费用分析 |
| 销售费用 | 部门、项目、费用项目 | 销售费用分析 |
| 主营业务收入 | 客户、产品线、项目 | 收入结构分析 |
| 主营业务成本 | 产品线、项目 | 毛利分析 |

### 账簿创建和审核

常见路径：

```text
财务会计
  -> 总账
  -> 基础资料
  -> 账簿
```

操作步骤：

1. 选择核算体系。
2. 选择核算组织。
3. 选择会计政策。
4. 确认会计日历自动带出是否正确。
5. 选择科目表。
6. 选择账簿类型：主账簿或副账簿。
7. 维护记账本位币。
8. 维护默认汇率类型。
9. 维护默认凭证字。
10. 维护启用期间。
11. 确认应收、应付确认方式。
12. 保存并审核。

账簿审核前必须复核启用期间。启用期间错了，后续初始化、业务单据生成凭证、结账都会被牵连。

### 总账参数

总账参数会影响凭证、过账、结账和账表。不同版本参数名称可能略有差异，项目上至少要确认：

| 参数方向 | 要确认的问题 |
| --- | --- |
| 凭证控制 | 是否允许制单人审核自己凭证，是否允许审核后修改，是否允许断号 |
| 过账控制 | 过账前是否必须审核，是否必须出纳复核 |
| 期间控制 | 是否允许跨期制证，是否允许反结账 |
| 现金流量 | 现金银行科目是否强制指定现金流量项目 |
| 附件控制 | 是否要求凭证附件 |
| 账表控制 | 是否默认包含未过账凭证 |

## 四、总账初始化教程

初始化的目标是：让系统启用前的历史余额正确进入金蝶，并且能够从启用期开始继续核算。

### 初始化范围

| 初始化内容 | 目的 |
| --- | --- |
| 科目初始数据 | 建立总账期初余额 |
| 辅助核算初始数据 | 建立客户、供应商、部门、项目等明细余额 |
| 外币初始数据 | 建立原币、本位币和汇兑基础 |
| 数量金额初始数据 | 建立数量和金额余额 |
| 现金流量初始数据 | 非年初启用时补现金流量累计数 |
| 子模块期初 | 应收、应付、资产、库存等明细台账 |

### 初始化步骤

1. 确认启用期间，例如 2026 年第 1 期。
2. 从旧系统导出启用前一期期末余额。
3. 整理科目余额表。
4. 整理辅助核算余额明细。
5. 整理外币余额原币和本位币。
6. 整理现金流量初始数据。
7. 在总账录入或导入科目初始数据。
8. 录入或导入子模块期初。
9. 做总账试算平衡。
10. 做总账和子模块期初对账。
11. 结束初始化。

### 初始化检查表

| 检查项 | 标准 |
| --- | --- |
| 借贷平衡 | 资产 + 成本费用类借方余额，与负债 + 权益 + 收入类贷方余额平衡 |
| 辅助核算完整 | 启用辅助核算的科目，每一笔余额都有维度 |
| 应收期初 | 应收明细合计等于总账应收科目余额 |
| 应付期初 | 应付明细合计等于总账应付科目余额 |
| 固定资产期初 | 资产卡片原值、累计折旧、净值等于总账 |
| 银行现金期初 | 和银行对账单、现金盘点一致 |
| 存货期初 | 库存数量金额和存货科目一致 |
| 现金流量初始 | 非年初启用时，现金流量累计口径正确 |

### 初始化常见错误

| 错误 | 后果 | 处理 |
| --- | --- | --- |
| 只导总账，不导应收应付明细 | 后续收付款无法核销 | 补导子模块期初明细 |
| 辅助核算维度为空 | 余额表能平，但客户/供应商查不到 | 补辅助核算余额 |
| 外币只导本位币 | 期末调汇基础不对 | 补原币、汇率、本位币 |
| 资产只导总账余额 | 折旧和资产台账断层 | 建资产卡片 |
| 启用期间选错 | 业务期间、凭证期间全错 | 上线前修正，必要时重建账簿 |

## 五、凭证管理完整教程

### 凭证生命周期

```text
来源业务
  -> 生成或录入凭证
  -> 保存
  -> 提交
  -> 审核
  -> 出纳复核
  -> 过账
  -> 查询账表
  -> 必要时冲销或调整
```

不同企业状态流可能不同，有的环境没有提交，有的环境不启用出纳复核。核心原则是：未过账凭证不应该作为正式账务余额。

### 凭证来源

| 来源 | 说明 | 排查方式 |
| --- | --- | --- |
| 手工录入 | 总账会计直接录入 | 看摘要、附件、制单人 |
| 应收生成 | 应收单、收款单等生成 | 反查应收单和核销 |
| 应付生成 | 应付单、付款单等生成 | 反查应付单和付款 |
| 出纳生成 | 收付款、银行业务生成 | 查银行账号和日记账 |
| 固定资产生成 | 资产新增、折旧、处置生成 | 查资产卡片和折旧单 |
| 存货成本生成 | 入库、出库、成本计算生成 | 查存货核算和成本结果 |
| 自动转账 | 月末模板自动生成 | 查自动转账方案 |
| 期末调汇 | 外币调汇生成 | 查汇率和外币余额 |
| 损益结转 | 结转收入费用生成 | 查损益结转方案 |
| 导入凭证 | Excel 或接口导入 | 查导入模板和日志 |

### 手工凭证录入教程

常见路径：

```text
财务会计
  -> 总账
  -> 凭证管理
  -> 凭证录入
```

操作步骤：

1. 选择正确账簿。
2. 选择凭证日期。
3. 检查系统带出的会计期间是否正确。
4. 选择凭证字。
5. 输入摘要。
6. 录入借方科目。
7. 补齐借方辅助核算。
8. 录入借方金额。
9. 录入贷方科目。
10. 补齐贷方辅助核算。
11. 录入贷方金额。
12. 如果有外币，录入币别、汇率、原币金额。
13. 如果涉及现金银行，指定现金流量项目。
14. 上传附件或关联业务依据。
15. 保存。
16. 提交或直接审核，按系统流程执行。

### 手工凭证示例：计提房租

业务：6 月计提办公室租金 10000 元，尚未付款，费用归属管理部。

```text
借：管理费用-租赁费        10000
贷：其他应付款-房东        10000
```

金蝶录入要点：

- 管理费用如果启用部门辅助核算，要选择管理部。
- 其他应付款如果启用往来单位辅助核算，要选择房东或其他往来单位。
- 摘要建议写“计提 2026 年 6 月办公室租金”。
- 附件放合同、审批单或计提说明。

### 手工凭证示例：银行手续费

业务：银行扣手续费 25 元。

```text
借：财务费用-手续费        25
贷：银行存款-基本户        25
```

金蝶录入要点：

- 银行存款科目如果启用银行账号维度，要选具体账号。
- 现金流量项目一般按企业口径指定。
- 如果手续费来自银行流水，优先通过出纳或银行流水流程生成，不建议重复手工录入。

### 凭证审核教程

审核不是机械点按钮。审核人要检查：

| 检查项 | 看什么 |
| --- | --- |
| 来源 | 手工凭证是否有附件，业务凭证是否能反查源单 |
| 日期 | 凭证日期是否属于正确期间 |
| 摘要 | 是否能解释业务 |
| 科目 | 是否符合业务性质 |
| 辅助核算 | 客户、供应商、部门、项目等是否完整 |
| 金额 | 借贷是否平衡，金额和税额是否正确 |
| 现金流量 | 涉及现金银行科目是否指定项目 |
| 附件 | 合同、发票、审批、回单是否齐全 |

审核控制建议：

- 制单人和审核人尽量分离。
- 手工凭证必须有附件或说明。
- 往来、资产、存货科目的手工凭证要重点审核。
- 大额凭证设置复核规则。

### 出纳复核

出纳复核通常用于涉及库存现金、银行存款等资金科目的凭证。它的目的不是替代凭证审核，而是从资金角度确认收付款真实性。

检查点：

- 银行账号是否正确。
- 收付款方向是否正确。
- 金额是否与银行回单一致。
- 结算方式是否正确。
- 是否重复入账。

### 凭证过账教程

过账是把凭证登记到账簿。过账后，科目余额表、明细账和报表才会体现。

常见路径：

```text
财务会计
  -> 总账
  -> 凭证管理
  -> 凭证过账
```

操作步骤：

1. 选择账簿。
2. 选择会计期间。
3. 筛选已审核未过账凭证。
4. 检查是否包含不应过账的凭证。
5. 执行过账。
6. 查看过账结果。
7. 进入科目余额表抽查发生额。

过账前检查：

- 本期凭证是否全部审核。
- 是否存在错误凭证未处理。
- 期间是否开放。
- 是否需要出纳复核。
- 是否有现金流量项目缺失。

### 凭证修改、反审核、反过账和冲销

| 动作 | 适用场景 | 风险 |
| --- | --- | --- |
| 修改未审核凭证 | 凭证刚录入发现错误 | 风险较低 |
| 反审核 | 已审核但未过账发现错误 | 需要权限控制 |
| 反过账 | 已过账发现错误 | 会影响账表和报表 |
| 冲销 | 跨期或已报表后发现错误 | 更利于留痕 |
| 调整凭证 | 历史期间不宜改动时 | 需要说明原因 |

原则：

- 当期未结账、未对外报表前，可以按流程反审核或反过账修正。
- 已结账或已对外报表后，优先用调整凭证或冲销凭证。
- 来自业务单据的凭证，优先回源单和凭证模板处理，不要直接改总账凭证。

## 六、业务凭证和总账的衔接

### 为什么不建议直接在总账补往来凭证

应收、应付、固定资产、存货这些业务都有子模块台账。如果直接在总账补凭证，会出现：

- 总账有余额，子模块没有明细。
- 应收账龄不准。
- 应付付款无法核销。
- 固定资产折旧和总账不一致。
- 存货账和库存账不一致。

### 正确处理路径

| 场景 | 优先处理 |
| --- | --- |
| 客户欠款少录 | 补应收单，再生成凭证 |
| 客户回款漏录 | 补收款单，生成凭证并核销 |
| 供应商发票漏录 | 补应付单，再生成凭证 |
| 供应商付款漏录 | 补付款单，生成凭证并核销 |
| 固定资产漏建卡 | 补资产卡片和入账凭证 |
| 折旧少提 | 回固定资产模块补折旧 |
| 存货成本错 | 回存货核算或成本核算处理 |

### 凭证生成后总账要查什么

| 检查项 | 查询位置 |
| --- | --- |
| 是否生成成功 | 凭证生成情况查询、源单关联凭证 |
| 是否审核 | 凭证列表 |
| 是否过账 | 凭证列表、科目余额表 |
| 科目是否正确 | 凭证分录、凭证模板 |
| 维度是否完整 | 凭证分录辅助核算 |
| 报表是否体现 | 科目余额表、财务报表 |

## 七、账表查询教程

### 查询前固定口径

查账前必须固定：

- 账簿。
- 会计期间。
- 币别。
- 科目范围。
- 是否包含未过账凭证。
- 核算维度过滤条件。
- 是否显示禁用科目或无发生额科目。

很多“账不平”不是数据错，而是查询口径不一致。

### 科目余额表

用途：看科目期初余额、本期借方、本期贷方、期末余额。

常见问题：

| 问题 | 查法 |
| --- | --- |
| 某科目余额异常 | 按科目查余额，再钻取明细账 |
| 本期发生额不对 | 查是否有未过账凭证或跨期凭证 |
| 新增科目报表没有 | 先确认余额表有数，再查报表公式 |
| 余额方向异常 | 查科目性质和具体凭证分录 |

### 明细分类账

用途：看某个科目的每一笔发生。

查询步骤：

1. 选择账簿。
2. 选择期间。
3. 输入科目。
4. 设置是否包含未过账凭证。
5. 根据摘要、凭证号、辅助核算进一步过滤。
6. 从明细账打开凭证。
7. 从凭证反查源单。

### 总分类账

用途：看总账级别的期初、发生、累计和期末。适合财务主管快速看大类科目的整体情况。

适合场景：

- 月末检查资产、负债、权益、收入、费用是否整体合理。
- 查看本年累计发生额。
- 对报表项目做粗略核对。

### 核算维度明细账

用途：按客户、供应商、部门、项目等维度查询。

示例：

| 查询目标 | 科目 | 维度 |
| --- | --- | --- |
| 客户 A 应收明细 | 应收账款 | 客户 A |
| 供应商 B 应付明细 | 应付账款 | 供应商 B |
| 销售部费用明细 | 销售费用 | 销售部 |
| 项目 P 成本费用 | 成本费用科目 | 项目 P |

### 多栏式明细账

用途：费用类或收入类科目按明细科目展示。比如管理费用下面办公费、差旅费、招待费等。

适合场景：

- 部门费用分析。
- 费用预算执行检查。
- 月末费用波动分析。

### 试算平衡表

用途：检查借贷是否平衡。

检查点：

- 期初借贷是否平衡。
- 本期发生借贷是否平衡。
- 期末余额借贷是否平衡。
- 不平时优先查初始化、凭证状态、数据修复或异常中断。

### 凭证汇总表

用途：按凭证汇总借贷发生额。适合查某期间凭证数量、凭证字、科目发生概览。

## 八、现金流量处理

### 为什么总账要管现金流量

现金流量表不是简单等于银行存款发生额。涉及现金银行科目的凭证，需要判断经营、投资、筹资活动类别，以及现金流量项目。

### 现金流量项目指定

常见路径：

```text
财务会计
  -> 总账
  -> 现金流量
  -> 凭证指定现金流量
```

处理方式：

| 方式 | 适用场景 |
| --- | --- |
| 凭证模板自动指定 | 业务类型稳定，例如销售收款、采购付款 |
| 凭证录入时手工指定 | 手工凭证或特殊业务 |
| 批量预设 | 同类凭证批量处理 |
| T 型账分析 | 从现金银行科目反向分析 |
| 现金流量检查 | 月末查漏填和异常 |

### 常见现金流量项目

| 业务 | 常见现金流量分类 |
| --- | --- |
| 销售收款 | 销售商品、提供劳务收到的现金 |
| 采购付款 | 购买商品、接受劳务支付的现金 |
| 支付工资 | 支付给职工以及为职工支付的现金 |
| 支付税费 | 支付的各项税费 |
| 购建固定资产 | 购建固定资产、无形资产支付的现金 |
| 借款收到 | 取得借款收到的现金 |
| 偿还借款 | 偿还债务支付的现金 |

### 月末现金流量检查

- 涉及现金银行科目的凭证是否都指定项目。
- 现金流量项目方向是否正确。
- 手工凭证是否漏填。
- 内部转账是否正确处理。
- 现金流量表和银行存款、现金科目是否能解释差异。

## 九、期末处理详细教程

总账期末处理不是只点“结账”。官方流程里，期末处理前要确认基本业务凭证均已过账，相关业务系统已结账，相关业务单据已生成凭证并过账。然后再做期末调汇、自动转账、结转损益和总账结账。

### 期末处理顺序

```text
业务单据清理
  -> 子模块生成凭证
  -> 凭证审核过账
  -> 子模块结账
  -> 期末调汇
  -> 自动转账
  -> 摊销预提
  -> 结转损益
  -> 试算平衡
  -> 总账结账
  -> 报表出具
```

### 期末调汇

适用场景：外币应收、外币应付、外币银行、外币借款等。

操作前提：

- 科目启用外币核算。
- 外币余额存在。
- 期末汇率已维护。
- 调汇方案或范围正确。

操作步骤：

1. 检查外币科目余额。
2. 维护期末汇率。
3. 选择账簿和期间。
4. 执行期末调汇。
5. 检查调汇结果。
6. 生成调汇凭证。
7. 审核并过账调汇凭证。
8. 查询财务费用-汇兑损益和外币科目余额。

调汇凭证常见分录：

```text
借/贷：财务费用-汇兑损益
贷/借：应收账款/应付账款/银行存款等外币科目
```

### 自动转账

适用场景：月末固定规则的结转、分摊、计提。

常见业务：

- 制造费用结转。
- 辅助生产费用分配。
- 税金及附加计提。
- 工资社保计提。
- 长期待摊费用摊销。
- 预提费用。

操作步骤：

1. 建自动转账模板。
2. 设置借贷科目。
3. 设置取数公式或固定金额。
4. 设置核算维度取值。
5. 月末执行生成凭证。
6. 审核并过账。
7. 检查相关科目余额。

### 凭证摊销和预提

摊销是把已经发生或已经支付的成本费用按期间分摊。预提是把已经受益但尚未支付或尚未收到发票的费用先确认。

示例：一年房租 120000 元，按月摊销。

付款时：

```text
借：长期待摊费用        120000
贷：银行存款            120000
```

每月摊销：

```text
借：管理费用-租赁费      10000
贷：长期待摊费用          10000
```

### 结转损益

结转损益是把收入、成本、费用等损益类科目结转到本年利润。

收入类结转：

```text
借：主营业务收入
借：其他业务收入
贷：本年利润
```

成本费用类结转：

```text
借：本年利润
贷：主营业务成本
贷：销售费用
贷：管理费用
贷：财务费用
```

检查点：

- 新增收入科目是否纳入结转方案。
- 新增费用科目是否纳入结转方案。
- 结转凭证是否生成。
- 结转凭证是否审核过账。
- 结转后损益类科目期末余额是否为 0。

### 总账结账

结账前检查：

- 本期凭证全部审核。
- 本期凭证全部过账。
- 业务模块已经结账。
- 期末调汇完成。
- 自动转账完成。
- 结转损益完成。
- 试算平衡。
- 子模块和总账对账完成。

结账后结果：

- 当前账簿期间进入下一期间。
- 已结账期间一般不再允许普通业务修改。
- 如发现错误，需要走反结账、调整或冲销流程。

### 反结账控制

反结账风险很高，因为它会打开历史期间。

建议规则：

- 只有财务经理或系统管理员有权限。
- 必须有书面审批。
- 必须说明反结账原因。
- 反结账后只处理审批范围内的问题。
- 重新结账后保存前后差异。
- 已对外报表的期间优先使用调整凭证，不轻易反结账。

## 十、总账和子模块对账

### 应收对总账

目标：应收账款科目余额等于应收模块客户余额。

排查路径：

```text
应收余额表
  -> 应收明细
  -> 是否生成凭证
  -> 凭证是否过账
  -> 总账科目余额表
  -> 手工凭证筛查
  -> 期初差异
```

常见差异：

- 应收单未生成凭证。
- 收款单未生成凭证。
- 凭证未过账。
- 总账手工录了应收账款。
- 应收期初和总账期初不一致。
- 查询条件不一致。

### 应付对总账

目标：应付账款科目余额等于应付模块供应商余额。

重点检查：

- 应付单是否生成凭证。
- 付款单是否生成凭证。
- 付款是否核销。
- 是否存在手工应付凭证。
- 应付期初是否等于总账期初。

### 出纳对总账

目标：银行日记账、银行流水、总账银行存款科目能够解释一致。

重点检查：

- 银行账号维度是否正确。
- 收付款单是否生成凭证。
- 银行日记账是否完整。
- 银行对账是否完成。
- 未达账项是否有说明。
- 总账银行科目是否存在手工凭证。

### 固定资产对总账

目标：固定资产台账原值、累计折旧、净值等于总账相关科目。

重点检查：

- 资产卡片是否完整。
- 新增资产入账凭证是否过账。
- 折旧凭证是否过账。
- 资产处置凭证是否过账。
- 资产变更是否影响折旧。

### 存货和成本对总账

目标：库存、存货核算、成本核算和总账存货类科目一致。

重点检查：

- 入库、出库单据是否全部审核。
- 存货核算是否完成。
- 成本计算是否完成。
- 存货凭证是否生成并过账。
- 是否存在总账手工调整存货科目。

## 十一、多账簿和多核算口径

### 什么情况下需要多账簿

| 场景 | 说明 |
| --- | --- |
| 法定账和管理账不同 | 法人口径和内部管理口径不同 |
| 不同会计准则 | 中国准则、国际准则或集团准则不同 |
| 不同本位币 | 境外公司或特殊管理要求 |
| 税务口径差异 | 财务账和税务调整口径不同 |
| 集团合并 | 母子公司、多法人、多组织汇总 |

### 多账簿设计要点

- 哪个是主账簿。
- 副账簿是否要从主账簿复制凭证。
- 科目表是否一致。
- 会计政策是否一致。
- 本位币是否一致。
- 凭证生成规则是否需要区分账簿。
- 报表和对账按哪个账簿出。

### 多账簿风险

- 同一业务在不同账簿重复或漏记。
- 主副账簿科目映射不一致。
- 副账簿没有同步业务凭证。
- 多账簿查询时口径混淆。
- 合并报表抵消口径不清。

## 十二、调整期间业务

调整期间用于处理年度审计调整、年末调整等特殊事项。不是每个企业都会启用。

适用场景：

- 年度审计调整。
- 上年报表调整。
- 不希望影响正常 12 期业务查询。
- 需要单独展示调整凭证。

注意事项：

- 调整期间凭证要单独说明。
- 调整期间是否参与报表，要和财务经理确认。
- 调整期间是否影响本年期初，要确认结转逻辑。
- 调整期间权限要严格控制。

## 十三、总账权限和内控

### 岗位职责

| 岗位 | 主要职责 |
| --- | --- |
| 总账会计 | 凭证录入、凭证检查、期末处理、账表查询 |
| 主管会计 | 凭证审核、结账、反结账审批、报表复核 |
| 出纳 | 资金凭证复核、银行对账、现金银行流水 |
| 财务经理 | 月结审批、报表审批、重大调整审批 |
| 系统管理员 | 权限、参数、基础资料维护 |

### 权限分离建议

- 制单人不要审核自己凭证。
- 过账权限不要给所有制单人。
- 反审核、反过账、反结账权限要限制。
- 科目和凭证模板维护权限要限制。
- 现金银行科目凭证需要出纳复核。
- 手工凭证和自动生成凭证要能区分来源。

### 总账内控检查

| 控制点 | 检查方法 |
| --- | --- |
| 手工凭证 | 每月导出手工凭证清单，重点看往来、资产、存货科目 |
| 反操作 | 导出反审核、反过账、反结账记录 |
| 大额凭证 | 按金额阈值抽查附件和审批 |
| 跨期凭证 | 查凭证日期和业务日期差异 |
| 新增科目 | 查是否同步报表公式和结转方案 |
| 现金流量 | 查现金银行凭证是否指定项目 |

## 十四、总账常见业务案例

### 案例 1：新增费用科目后利润表没有数

现象：

- 凭证已经录入新科目。
- 科目余额表有发生额。
- 利润表仍然没有显示。

排查：

1. 查凭证是否过账。
2. 查科目余额表是否有数。
3. 查利润表公式是否包含新科目。
4. 查损益结转方案是否包含新科目。
5. 重新计算利润表。

处理：

- 补报表公式。
- 补损益结转方案。
- 重新生成结转凭证。

预防：

- 新增科目必须同步检查凭证模板、报表公式、损益结转方案。

### 案例 2：应收账款总账比应收模块多 50000

现象：

- 应收模块客户余额合计 100000。
- 总账应收账款科目余额 150000。

排查：

1. 固定账簿、期间、币别、客户维度。
2. 查应收模块余额。
3. 查总账应收账款明细账。
4. 筛选来源为空或手工凭证。
5. 找到一张手工借记应收账款 50000 的凭证。

处理：

- 如果确实是应收业务，应补应收单并冲销手工凭证。
- 如果是调整事项，应保留审批说明，并确认是否需要同步客户余额。

预防：

- 应收账款科目手工凭证必须审批。
- 月结时单独导出往来科目手工凭证。

### 案例 3：结转损益后管理费用仍有余额

现象：

- 执行损益结转后，管理费用某明细科目仍有余额。

排查：

1. 查该科目是否是损益类。
2. 查是否新增科目。
3. 查损益结转方案是否包含该科目。
4. 查结转凭证是否生成并过账。

处理：

- 把该科目加入结转方案。
- 删除或冲销错误结转凭证。
- 重新生成结转凭证。
- 审核过账。

### 案例 4：凭证过账失败，提示辅助核算缺失

现象：

- 凭证保存或审核时没发现问题，过账时报辅助核算缺失。

排查：

1. 查报错分录科目。
2. 打开科目资料，看启用了哪些辅助核算。
3. 回凭证分录补齐客户、供应商、部门或项目。
4. 如果来自业务单据生成，检查凭证模板维度来源。

处理：

- 手工凭证直接补维度。
- 业务凭证回模板或源单补维度后重新生成。

### 案例 5：现金流量表经营活动现金流不准

现象：

- 银行存款科目发生额没问题。
- 现金流量表项目分类明显不对。

排查：

1. 查涉及现金银行科目的凭证。
2. 查现金流量项目是否为空。
3. 查项目分类是否错误。
4. 查内部转账是否错误计入经营活动。
5. 查手工凭证是否漏填项目。

处理：

- 补指定现金流量项目。
- 修凭证模板或现金流量预设。
- 月末执行现金流量检查。

## 十五、总账排错清单

### 凭证不能保存

| 可能原因 | 检查 |
| --- | --- |
| 借贷不平 | 借方合计和贷方合计 |
| 科目不可用 | 科目是否禁用或未分配 |
| 辅助核算缺失 | 科目要求的维度是否完整 |
| 期间不可用 | 凭证日期对应期间是否开放 |
| 币别汇率缺失 | 外币凭证是否有汇率 |

### 凭证不能审核

| 可能原因 | 检查 |
| --- | --- |
| 权限不足 | 用户角色和按钮权限 |
| 制单审核未分离 | 是否禁止制单人审核 |
| 凭证附件不完整 | 是否启用附件控制 |
| 分录异常 | 科目、维度、金额 |

### 凭证不能过账

| 可能原因 | 检查 |
| --- | --- |
| 未审核 | 凭证状态 |
| 未出纳复核 | 是否涉及资金科目 |
| 期间关闭 | 总账期间状态 |
| 辅助核算缺失 | 分录维度 |
| 现金流量项目缺失 | 现金银行科目 |
| 科目被禁用 | 科目状态 |

### 账表查询不一致

| 可能原因 | 检查 |
| --- | --- |
| 是否包含未过账凭证不同 | 查询条件 |
| 账簿不同 | 账簿 |
| 期间不同 | 会计年度和期间 |
| 币别不同 | 原币、本位币、综合本位币 |
| 维度过滤不同 | 客户、供应商、部门、项目 |
| 报表公式漏科目 | 报表公式 |

### 结账失败

| 可能原因 | 检查 |
| --- | --- |
| 存在未审核凭证 | 凭证列表 |
| 存在未过账凭证 | 凭证过账列表 |
| 子模块未结账 | 应收、应付、出纳、固定资产、存货 |
| 未生成业务凭证 | 凭证生成情况查询 |
| 期末调汇未完成 | 外币科目 |
| 损益未结转 | 损益类科目余额 |

## 十六、总账日常工作 SOP

### 每日

- 查本日业务单据是否生成凭证。
- 查未审核凭证。
- 审核并过账当天凭证。
- 抽查大额凭证附件。
- 查现金银行凭证是否正确。

### 每周

- 查手工凭证清单。
- 查往来科目手工凭证。
- 查新增科目和新增辅助核算。
- 查异常余额方向。
- 查银行存款和出纳日记账。

### 月末

- 清理未审核凭证。
- 清理未过账凭证。
- 清理未生成凭证业务单据。
- 完成期末调汇。
- 完成自动转账和摊销预提。
- 完成损益结转。
- 做试算平衡。
- 对应收、应付、出纳、固定资产、存货和总账。
- 结账。
- 输出月结包。

## 十八、练习题

### 练习 1：录入手工凭证

业务：6 月支付银行手续费 30 元，从招商银行基本户扣款。

要求：

1. 写出分录。
2. 说明银行存款科目需要什么辅助核算。
3. 说明凭证过账后要查哪张账表。

参考：

```text
借：财务费用-手续费       30
贷：银行存款-招商银行     30
```

检查：银行账号维度、现金流量项目、科目余额表、银行存款明细账。

### 练习 2：排查应收差异

现象：应收模块客户 A 余额为 0，总账应收账款-客户 A 余额为贷方 500。

排查步骤：

1. 固定账簿、期间、币别、客户 A。
2. 查应收模块余额和核销记录。
3. 查总账应收账款客户 A 明细账。
4. 查是否有手工凭证或跨期收款凭证。
5. 判断是多收、退款、尾差还是错账。

### 练习 3：新增费用科目

业务：新增“管理费用-软件服务费”。

必须同步检查：

- 科目类别和余额方向。
- 是否启用部门或项目辅助核算。
- 费用报销或应付凭证模板是否使用。
- 利润表公式是否包含。
- 损益结转方案是否包含。
- 预算或管理报表是否包含。

## 十九、学习顺序建议

如果你是从零学总账，按这个顺序：

1. 看懂会计六要素和借贷方向。
2. 学科目和辅助核算。
3. 学账簿和会计期间。
4. 学手工凭证录入、审核、过账。
5. 学业务单据生成凭证。
6. 学科目余额表和明细账。
7. 学应收应付、资产、出纳和总账对账。
8. 学期末调汇、自动转账、损益结转。
9. 学结账和反结账控制。
10. 学报表公式和月结包。
