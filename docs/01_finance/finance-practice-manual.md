这份手册按“上线前准备 -> 日常业务 -> 凭证 -> 对账 -> 月结 -> 报表”的顺序整理。它不是目录，而是日常做金蝶财务时可以直接查的工作说明。

## 适用范围

- 产品口径：以金蝶云·星空财务会计相关公开资料为主。
- 场景口径：总账、应收、应付、出纳、固定资产、报表、月结。
- 使用方式：遇到具体业务时先按本文定位模块，再进入对应专题页。

## 财务人员每天要做什么

| 角色 | 每日动作 | 产出 | 不做的风险 |
| --- | --- | --- | --- |
| 总账会计 | 检查凭证生成、审核、过账，处理手工调整凭证 | 当日凭证状态清晰 | 月末堆积未过账凭证，结账失败 |
| 应收会计 | 检查应收单、收款单、核销和账龄 | 客户往来清晰 | 回款已发生但未核销，账龄失真 |
| 应付会计 | 检查应付单、付款单、核销和到期付款 | 供应商债务清晰 | 付款计划不准，供应商对账困难 |
| 出纳 | 处理收付款、现金银行、银行对账 | 资金流水清晰 | 银行账和总账不一致 |
| 资产会计 | 维护资产卡片、变更、处置、折旧 | 资产台账准确 | 折旧错误，资产和总账不一致 |
| 财务经理 | 检查关键报表、异常余额、月结进度 | 管理口径准确 | 报表滞后，差异无人解释 |

## 上线前必须确认的财务口径

### 组织和账簿

| 问题 | 为什么重要 | 记录示例 |
| --- | --- | --- |
| 哪些组织需要独立核算 | 决定核算组织和账簿 | A 公司、B 公司分别核算 |
| 是否有多账簿 | 决定主账簿、副账簿 | 主账簿按企业会计准则，副账簿按管理口径 |
| 本位币是什么 | 影响凭证、报表、汇率 | CNY |
| 会计期间如何设置 | 影响启用和结账 | 自然月，2026-01 启用 |
| 是否需要外币核算 | 影响汇率和调汇 | USD、EUR 业务需要期末调汇 |

### 科目和辅助核算

做科目设计时，不要只看科目编码，要同时看辅助核算，否则后期报表和对账会很痛。

| 科目类型 | 常见科目 | 建议辅助核算 |
| --- | --- | --- |
| 应收 | 应收账款、预收账款 | 客户、销售组织、项目 |
| 应付 | 应付账款、预付账款 | 供应商、采购组织、项目 |
| 资金 | 银行存款、库存现金 | 银行账号、币别 |
| 存货 | 原材料、库存商品、发出商品 | 物料、仓库、成本中心 |
| 资产 | 固定资产、累计折旧 | 资产类别、部门 |
| 损益 | 主营业务收入、管理费用、销售费用 | 部门、项目、费用项目 |

### 期初数据

期初数据至少要分成四块核对：

1. 总账科目余额：借贷平衡。
2. 应收应付往来明细：能解释应收账款、应付账款等科目余额。
3. 现金银行余额：和银行对账单、现金盘点表一致。
4. 固定资产卡片：原值、累计折旧、净值和总账一致。

## 日常业务流转

### 销售到应收

```text
销售订单/销售出库/销售发票
  -> 应收单
  -> 审核
  -> 生成凭证
  -> 收款单
  -> 收款核销
  -> 应收账龄和客户对账
```

关键检查：

- 应收单客户和结算客户是否正确。
- 应收单金额、税额、币别、汇率是否正确。
- 收款单是否和应收单核销。
- 应收凭证是否过账。
- 应收余额是否能和总账应收科目核对。

### 采购到应付

```text
采购订单/采购入库/采购发票
  -> 应付单
  -> 审核
  -> 生成凭证
  -> 付款单
  -> 付款核销
  -> 应付账龄和供应商对账
```

关键检查：

- 应付单供应商和结算供应商是否正确。
- 应付确认口径是入库、发票还是应付单。
- 付款单是否和应付单核销。
- 应付凭证是否过账。
- 应付余额是否能和总账应付科目核对。

### 资金收付

```text
收款/付款业务
  -> 收款单/付款单
  -> 出纳处理
  -> 银行流水或银行对账单
  -> 银行对账
  -> 现金银行科目核对
```

关键检查：

- 银行账号、结算方式、币别是否正确。
- 是否重复导入银行流水。
- 未达账项是否有解释。
- 涉及现金银行科目的凭证是否审核、过账、出纳复核。

### 固定资产

```text
资产购置
  -> 资产卡片
  -> 入账
  -> 变更/处置/盘点
  -> 折旧计提
  -> 生成凭证
  -> 固定资产结账
```

关键检查：

- 资产类别、折旧方法、使用部门、开始使用日期是否正确。
- 本期新增资产是否需要本期折旧。
- 本期变更是否影响折旧。
- 折旧凭证是否生成并过账。
- 固定资产台账是否和总账固定资产科目一致。

## 凭证处理原则

### 不建议频繁手工改业务凭证

如果业务凭证来自应收、应付、出纳、固定资产等模块，优先回到业务单据和凭证模板处理。直接手工改凭证可能导致：

- 子模块和总账不一致。
- 以后重新生成凭证覆盖或重复。
- 审计追溯困难。

### 凭证生成失败先看这五个点

1. 单据是否审核。
2. 凭证模板是否启用并命中。
3. 借贷科目是否能取到。
4. 辅助核算维度是否完整。
5. 期间是否打开，账簿是否正确。

## 月末硬性检查

月末真正要盯住的是“所有业务是否都进入总账”。建议每月固定出一张检查表：

| 检查项 | 判断方法 | 责任人 |
| --- | --- | --- |
| 未审核业务单据 | 各模块单据列表按状态过滤 | 各模块会计 |
| 未生成凭证业务单据 | 凭证生成情况查询 | 总账会计 |
| 未审核凭证 | 凭证列表按状态过滤 | 总账会计 |
| 未过账凭证 | 凭证列表按过账状态过滤 | 总账会计 |
| 应收应付和总账差异 | 往来报表 vs 科目余额表 | 往来会计 |
| 银行对账差异 | 银行日记账 vs 银行对账单 | 出纳 |
| 固定资产差异 | 资产台账 vs 总账科目 | 资产会计 |
| 报表异常 | 报表公式和科目余额表核对 | 财务经理 |

## 常见业务分录示例

以下只是常见会计处理示例，实际科目必须按企业科目表、税务口径、会计政策和金蝶凭证模板调整。

| 场景 | 借方 | 贷方 |
| --- | --- | --- |
| 销售确认应收 | 应收账款 | 主营业务收入、应交税费-应交增值税-销项税额 |
| 收到客户回款 | 银行存款 | 应收账款 |
| 采购确认应付 | 原材料/库存商品/费用、应交税费-应交增值税-进项税额 | 应付账款 |
| 支付供应商款项 | 应付账款 | 银行存款 |
| 固定资产购入 | 固定资产、应交税费-进项税额 | 应付账款/银行存款 |
| 计提折旧 | 制造费用/管理费用/销售费用 | 累计折旧 |
| 结转损益 | 主营业务收入等收入科目 | 本年利润 |
| 结转成本费用 | 本年利润 | 主营业务成本/费用类科目 |
