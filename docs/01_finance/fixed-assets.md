## 适用范围

适用于固定资产启用、资产卡片、资产入账、变更、处置、折旧、盘点和固定资产月结。

## 核心目标

固定资产模块用于管理资产从入账到处置的全生命周期，并将折旧、处置等结果同步到总账。

## 启用流程

1. 设置固定资产参数。
2. 启用固定资产系统，设置启用年度和期间。
3. 维护基础资料。
4. 录入初始化卡片。
5. 处理日常资产业务。
6. 执行期末处理。

## 基础资料

常见基础资料包括：

- 资产类别
- 资产状态
- 变动方式
- 折旧方法
- 资产位置
- 折旧政策
- 会计政策

## 日常业务

| 业务 | 说明 |
| --- | --- |
| 资产申请 | 资产购置或使用申请 |
| 资产入账 | 建立资产卡片，确认资产入账 |
| 资产处置 | 报废、出售、转让等退出处理 |
| 资产变更 | 部门、使用人、折旧政策等变更 |
| 资产借用 | 临时借出或借入 |
| 资产领用 | 资产投入使用 |
| 批量变更 | 批量调整资产信息 |

## 固定资产完整案例

### 场景

购入一台设备，不含税 100000 元，进项税 13000 元，预计使用 5 年，残值率 5%，采用年限平均法，从投入使用月份开始折旧。

### 系统处理

1. 录入资产卡片或由采购业务生成资产。
2. 确认资产类别、使用部门、折旧方法、预计使用期间、残值率。
3. 审核资产卡片。
4. 生成资产入账凭证。
5. 月末计提折旧。
6. 生成折旧凭证。
7. 固定资产结账。

### 常见分录

资产入账：

```text
借：固定资产                100000
借：应交税费-进项税额        13000
贷：应付账款/银行存款       113000
```

每月折旧，示例：

```text
借：制造费用/管理费用
贷：累计折旧
```

### 月折旧额口径示例

```text
月折旧额 = 原值 * (1 - 残值率) / 预计使用月数
         = 100000 * (1 - 5%) / 60
         = 1583.33
```

实际系统结果还要看折旧政策、开始使用日期、当期新增是否当期计提、舍入规则和资产变更。

## 资产变更场景

常见变更：

- 使用部门变更。
- 使用人变更。
- 存放地点变更。
- 折旧费用分配部门变更。
- 原值调整。
- 折旧方法或预计使用期间调整。

处理原则：

1. 先确认变更生效日期。
2. 判断是否影响折旧。
3. 审核变更单。
4. 重新计算或检查当期折旧。
5. 保留变更原因。

## 资产处置场景

常见处置：

- 报废。
- 出售。
- 调拨或转让。
- 盘亏。

检查点：

- 处置日期是否正确。
- 是否已计提到处置月份应计折旧。
- 处置收入和税额是否正确。
- 资产原值、累计折旧、净值是否正确转出。
- 处置损益科目是否正确。

## 期末处理

固定资产期末通常关注：

- 资产盘点是否完成。
- 本期新增、变更、处置是否审核。
- 折旧是否计提。
- 折旧或处置凭证是否生成并过账。
- 固定资产系统是否结账。

## 常见问题

### 折旧金额不符合预期

检查资产原值、累计折旧、净值、预计使用期间、残值率、折旧方法、开始使用日期、是否当期新增当期折旧。

### 资产卡片不能结账

检查本期资产变动、处置、折旧、盘点、凭证生成和过账状态。

### 固定资产和总账不一致

检查资产入账、折旧、处置相关凭证是否完整生成并过账；排查是否存在手工凭证影响固定资产科目。
