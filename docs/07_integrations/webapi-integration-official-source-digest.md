## 二、WebAPI 的项目定位

WebAPI 是外部系统和金蝶交换数据的重要方式。项目里不要把它理解成“能调接口就行”，而要设计成：

```text
认证
  -> 权限
  -> 参数
  -> 字段映射
  -> 业务校验
  -> 幂等
  -> 日志
  -> 状态回写
  -> 对账
```

## 三、常见 API 操作类型

官方 WebAPI 接口通常围绕表单操作展开。项目中最常见：

| 操作 | 用途 |
| --- | --- |
| 保存 | 创建或更新单据 |
| 提交 | 进入审核或工作流 |
| 审核 | 让单据正式生效 |
| 反审核 | 撤回生效状态 |
| 删除 | 删除未正式生效数据 |
| 查询 | 按字段或条件读取数据 |
| 下推 | 由源单生成目标单 |
| 分配 | 基础资料跨组织分配 |

实际接口名称和参数以环境版本和官方接口说明为准。

## 四、常见参数理解

不同接口参数会随产品版本变化，但项目里通常需要理解：

| 参数含义 | 项目解释 |
| --- | --- |
| 表单标识 | 要操作哪个单据或基础资料 |
| 数据包 | 业务字段和分录内容 |
| 自动提交审核 | 保存后是否自动提交或审核 |
| 是否校验基础资料 | 外部传入编码是否要校验 |
| 是否返回单据编号 | 用于外部系统回写 |
| 操作用户 | 决定权限、组织和审计日志 |
| 数据中心 | 决定接口访问哪个账套或环境 |

## 五、第三方登录授权

接口认证要解决：

- 谁调用。
- 调哪个环境。
- 有哪些权限。
- 是否有组织范围。
- 密钥如何保管。
- 失败如何记录。

项目建议：

- 不使用个人账号做接口账号。
- 一个外部系统一个专用接口账号。
- 权限最小化。
- 定期轮换密码或密钥。
- 接口账号不用于前台人工操作。

## 六、集成平台和直接接口的区别

| 方式 | 适合场景 |
| --- | --- |
| 直接 WebAPI | 简单、实时、点对点接口 |
| 集成平台 | 多系统、多流程、需要编排、监控、转换 |
| 中间库 | 大批量、异步、老系统改造困难 |
| 文件导入导出 | 低频、人工确认、上线过渡 |

复杂项目不要把所有集成写成点对点接口，否则后续日志、重试、监控和对账会非常困难。

## 七、字段映射和编码映射

外部系统传“名称”通常不可靠，应优先传编码或唯一键。

常见映射：

- 客户编码。
- 供应商编码。
- 物料编码。
- 仓库编码。
- 部门编码。
- 业务组织编码。
- 币别编码。
- 单位编码。

如果外部系统和金蝶编码不同，必须建立映射表。

## 八、幂等和重复提交

接口失败后外部系统通常会重试。如果没有幂等，可能重复创建单据。

建议幂等键：

```text
外部系统 + 外部单号 + 业务类型 + 组织
```

处理逻辑：

1. 首次请求创建金蝶单据。
2. 重复请求且内容一致，返回已成功。
3. 重复请求但内容不同，拒绝并进入人工处理。
4. 超时不确定时，先查询金蝶是否已创建，再决定补偿。

## 九、安全管理中心和接口安全

安全管理不只管前台用户，也影响接口：

- 接口账号是否启用。
- 是否允许登录。
- 密码策略。
- IP 或网络限制。
- 操作日志。
- 数据权限。
- 敏感字段权限。

接口上线前要做“反向测试”：

- 接口账号不能看到无关组织数据。
- 接口账号不能调用不需要的单据。
- 接口账号不能越权审核或删除。

## 十、接口联调问题定位

| 现象 | 优先排查 |
| --- | --- |
| 登录失败 | 账号、密码、数据中心、授权方式 |
| 保存失败 | 必填字段、基础资料、字段标识 |
| 提交失败 | 单据状态、工作流、校验规则 |
| 审核失败 | 权限、审批流、插件、期间 |
| 下推失败 | 源单状态、转换路线、数量 |
| 查询为空 | 过滤条件、组织权限、数据状态 |
| 重复单据 | 幂等缺失、超时重试 |

## 十一、上线检查清单

- [ ] 接口清单和接口用途已确认。
- [ ] 接口账号独立且权限最小。
- [ ] 字段映射和编码映射已确认。
- [ ] 必填字段和基础资料校验已测试。
- [ ] 保存、提交、审核、下推全流程已测试。
- [ ] 幂等键已实现。
- [ ] 请求和响应日志已保存。
- [ ] 错误码和失败重试策略已确认。
- [ ] 对账报表已确认。
- [ ] 安全权限反向测试通过。
