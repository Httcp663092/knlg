金蝶集成项目常见外部系统包括电商平台、WMS、MES、OA、CRM、银企、发票平台、BI 和电子会计档案。每类系统的数据方向、主数据依赖、失败处理和对账方式不同，不能只用“接口字段映射”一张表解决。

## 一、集成总原则

- 金蝶和外部系统要明确主从关系。
- 主数据编码必须统一或有映射表。
- 每个接口必须有幂等键。
- 每个接口必须有日志。
- 财务相关数据必须可追溯。
- 接口失败要能重试和人工处理。
- 上线后必须对账。

## 二、电商订单集成

### 常见链路

```text
电商平台订单
  -> 中台或 OMS
  -> 金蝶销售订单
  -> WMS 发货
  -> 金蝶销售出库
  -> 应收或收入确认
```

### 关键字段

- 平台订单号。
- 店铺。
- 客户或会员。
- 商品编码。
- 数量。
- 成交价。
- 优惠金额。
- 运费。
- 税率。
- 发货仓库。
- 收货地址。

### 风险

| 风险 | 处理 |
| --- | --- |
| 重复订单 | 平台订单号做幂等键 |
| 商品编码不一致 | 建商品和物料映射 |
| 退款退货 | 设计红字或退货流程 |
| 优惠分摊 | 明确收入和折扣口径 |
| 跨平台多店铺 | 店铺映射组织或销售渠道 |

## 三、WMS 集成

### 数据方向

| 方向 | 数据 |
| --- | --- |
| 金蝶 -> WMS | 入库通知、出库通知、物料、仓库 |
| WMS -> 金蝶 | 实收入库、实际出库、盘点、调拨 |

### 对账重点

- 金蝶单据数量和 WMS 执行数量。
- 仓库、库位、批号。
- 库存状态。
- 已发未回、已回未审。
- 取消和作废。

### 常见问题

| 现象 | 原因 |
| --- | --- |
| WMS 收不到出库通知 | 接口失败、单据未审核、状态不满足 |
| 金蝶库存和 WMS 不一致 | 回传失败、批号库位不一致、人工调整 |
| 重复入库 | 缺少幂等或重试处理错误 |
| 退货无法匹配原单 | 外部单号或源单关联缺失 |

## 四、MES 集成

### 常见链路

```text
金蝶生产订单
  -> MES 派工和报工
  -> MES 领料或用料
  -> 金蝶生产领料
  -> MES 完工
  -> 金蝶完工入库
  -> 成本核算
```

### 重点字段

- 生产订单号。
- 物料。
- BOM 版本。
- 工序。
- 工单数量。
- 合格数量。
- 不良数量。
- 领料数量。
- 完工仓库。
- 批号。

### 风险

- MES 报工和金蝶完工数量不一致。
- 领料方式不清，导致材料成本不准。
- 返工、报废、补料没有闭环。
- 委外和自制流程混用。
- 成本核算缺少工时或费用数据。

## 五、OA 费用和审批集成

### 常见链路

```text
OA 费用申请
  -> OA 审批
  -> 金蝶费用报销或其他应付
  -> 付款申请
  -> 付款单
  -> 凭证
```

### 重点字段

- 申请人。
- 部门。
- 费用项目。
- 金额。
- 税额。
- 发票信息。
- 项目。
- 付款对象。
- 附件。

### 财务关注

- 费用科目映射。
- 部门和项目辅助核算。
- 发票是否合规。
- 是否重复报销。
- 附件能否追溯。

## 六、CRM 集成

### 常见数据

- 客户资料。
- 联系人。
- 商机。
- 报价。
- 销售订单。
- 回款状态。

### 风险

- CRM 客户和金蝶客户重复。
- 客户编码主从不清。
- 信用控制在 CRM 还是金蝶。
- 销售订单价格口径不一致。
- 回款状态回写延迟。

## 七、银企直连

### 常见链路

```text
付款申请
  -> 付款审批
  -> 银行支付
  -> 银行回单
  -> 金蝶付款单
  -> 凭证
```

### 风险控制

- 付款审批和银行支付权限分离。
- 银行账号和供应商账号校验。
- 重复付款控制。
- 支付状态回写。
- 银行回单附件留存。
- 大额付款复核。

## 八、发票平台集成

### 数据方向

- 销售发票开具状态回传。
- 采购发票采集。
- 红字发票。
- 发票验真。
- 税额和价税合计。

### 对账重点

- 发票金额和应收应付金额。
- 税率和税额。
- 红冲关系。
- 发票号码。
- 开票日期和入账期间。

## 九、BI 和数据仓库

### 取数方式

- API 取数。
- 数据库只读视图。
- 中间库。
- 定时文件。

### 风险

- 直接查生产库影响性能。
- 报表口径和金蝶标准报表不一致。
- 数据权限被绕过。
- 历史数据被覆盖。
