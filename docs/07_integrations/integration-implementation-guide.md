金蝶集成的目标不是“接口调通一次”，而是让外部系统和金蝶长期稳定交换数据，并且失败可追踪、重复可控制、金额可对账、业务可审计。

## 一、常见集成场景

| 场景 | 数据方向 | 关键风险 |
| --- | --- | --- |
| 电商订单进金蝶 | 外部 -> 金蝶 | 重复订单、价格税率、库存占用 |
| CRM 客户同步 | 外部 -> 金蝶 | 客户重复、编码规则、信用资料 |
| WMS 出入库同步 | 双向 | 库存差异、批号库位、时序 |
| OA 费用报销进金蝶 | 外部 -> 金蝶 | 费用科目、部门项目、附件 |
| 银企直连 | 银行 <-> 金蝶 | 付款安全、回单、重复付款 |
| 发票平台 | 税务/发票 -> 金蝶 | 税额、发票状态、红冲 |
| BI 报表取数 | 金蝶 -> 外部 | 口径一致、权限、性能 |

## 二、接口设计流程

```text
明确业务场景
  -> 确认主数据口径
  -> 确认单据和字段映射
  -> 确认认证方式
  -> 确认幂等键
  -> 确认错误处理
  -> 联调
  -> 压测
  -> 上线监控
  -> 对账
```

## 三、字段映射

字段映射表至少包含：

| 字段 | 说明 |
| --- | --- |
| 外部字段名 | 外部系统字段 |
| 金蝶字段名 | 金蝶单据字段 |
| 类型 | 文本、数字、日期、枚举、基础资料 |
| 是否必填 | 是否必须传 |
| 转换规则 | 编码映射、单位换算、金额计算 |
| 默认值 | 外部不传时如何处理 |
| 校验规则 | 长度、范围、状态 |
| 示例 | 真实样例 |

基础资料字段要特别小心。外部系统传“客户名称”通常不够，最好传客户编码或统一主数据编码。

## 四、幂等设计

幂等是接口集成的生命线。没有幂等，重试可能生成重复订单、重复应收、重复付款。

常用幂等键：

- 外部系统编码。
- 外部单据编号。
- 外部单据行号。
- 业务类型。
- 金蝶组织。

处理规则：

| 情况 | 处理 |
| --- | --- |
| 首次请求 | 创建金蝶单据 |
| 同一幂等键重复请求，内容一致 | 返回已成功结果 |
| 同一幂等键重复请求，内容不一致 | 拒绝并记录异常 |
| 前次失败后重试 | 根据失败阶段恢复 |

## 五、状态回写

如果外部系统推单到金蝶，通常需要金蝶回写状态：

- 已接收。
- 已校验。
- 已创建。
- 已提交。
- 已审核。
- 已生成凭证。
- 已失败。

状态回写要包含：

- 外部单号。
- 金蝶单号。
- 状态。
- 错误码。
- 错误信息。
- 时间。

## 六、错误处理

### 错误分类

| 错误 | 示例 | 处理 |
| --- | --- | --- |
| 参数错误 | 必填字段为空 | 不重试，返回业务错误 |
| 主数据错误 | 客户不存在 | 补资料后重试 |
| 业务规则错误 | 超信用、库存不足 | 业务处理后重试 |
| 权限错误 | 接口用户无权限 | 修权限 |
| 系统错误 | 网络超时、服务异常 | 自动重试 |
| 幂等冲突 | 同单号内容不同 | 人工处理 |

### 错误日志必须包含

- 请求时间。
- 请求方。
- 接口名称。
- 外部单号。
- 金蝶单号。
- 请求报文。
- 响应报文。
- 错误信息。
- 重试次数。
- 处理状态。

## 七、对账机制

接口上线后必须对账。

| 对账 | 口径 |
| --- | --- |
| 单据数量 | 外部成功数 = 金蝶成功数 |
| 金额 | 外部金额 = 金蝶金额 |
| 状态 | 外部已审核 = 金蝶已审核 |
| 凭证 | 需要入账的单据是否生成凭证 |
| 库存 | WMS 出入库和金蝶库存一致 |

建议每天输出接口对账表，至少包含成功、失败、处理中、重复、待人工处理五类。

## 八、财务接口特别注意

涉及财务的接口必须关注：

- 组织和账簿。
- 客户、供应商、部门、项目等核算维度。
- 金额、税额、币别、汇率。
- 单据日期和会计期间。
- 是否需要附件。
- 是否生成凭证。
- 凭证能否追溯外部来源。

不要让外部系统直接推总账凭证，除非业务确实没有更合适的子模块入口，并且已建立审批、附件、幂等、冲销和审计规则。
