接口集成真正的难点不是第一次调通，而是上线后持续稳定运行。联调要能定位字段、权限、主数据和业务规则问题；监控要能提前发现失败；重试要能避免重复数据；对账要能证明外部系统和金蝶数据一致。

## 一、联调准备

### 必备资料

- 接口清单。
- 字段映射表。
- 主数据映射表。
- 接口账号和权限说明。
- 请求示例。
- 响应示例。
- 错误码清单。
- 幂等规则。
- 状态回写规则。
- 对账口径。

### 测试数据

测试数据要覆盖：

- 正常数据。
- 必填字段缺失。
- 主数据不存在。
- 金额税额异常。
- 重复请求。
- 更新请求。
- 取消或红冲。
- 大批量数据。

## 二、联调流程

```text
连通性测试
  -> 认证测试
  -> 单条正常数据
  -> 单条异常数据
  -> 重复请求测试
  -> 批量测试
  -> 状态回写测试
  -> 对账测试
  -> 性能测试
```

## 三、接口日志

每个接口必须记录：

| 字段 | 说明 |
| --- | --- |
| 请求时间 | 何时调用 |
| 请求系统 | 谁调用 |
| 接口名称 | 调用哪个接口 |
| 幂等键 | 外部单号或唯一键 |
| 金蝶单号 | 创建或更新后的单号 |
| 请求报文 | 脱敏后保存 |
| 响应报文 | 成功或失败 |
| 状态 | 成功、失败、处理中 |
| 错误信息 | 原始错误和友好说明 |
| 重试次数 | 已重试几次 |
| 处理人 | 人工处理时记录 |

## 四、错误定位

### 认证失败

排查：

- 接口地址是否正确。
- 账号密码或密钥是否正确。
- 账号是否启用。
- 账号是否有接口权限。
- IP 白名单是否限制。
- 时间戳或签名是否正确。

### 主数据错误

常见：

- 客户不存在。
- 物料不存在。
- 仓库不存在。
- 部门不存在。
- 币别或单位不匹配。

处理：

- 建主数据映射表。
- 对缺失资料建立待处理队列。
- 不要用名称模糊匹配生成正式业务单据。

### 业务规则错误

常见：

- 库存不足。
- 超信用。
- 期间未打开。
- 单据状态不允许修改。
- 下游已存在不能反审核。

处理：

- 返回明确错误码。
- 让业务处理后重试。
- 不要简单无限重试。

## 五、重试机制

### 可重试

- 网络超时。
- 服务临时不可用。
- 数据库连接异常。
- 外部系统短时失败。

### 不应自动重试

- 必填字段为空。
- 主数据不存在。
- 金额校验失败。
- 幂等冲突。
- 权限不足。
- 业务规则不允许。

### 重试策略

```text
首次失败
  -> 短间隔重试
  -> 仍失败进入待处理
  -> 人工修复数据或配置
  -> 手工触发重试
  -> 成功后关闭问题
```

## 六、幂等和补偿

### 幂等判断

| 情况 | 处理 |
| --- | --- |
| 同幂等键第一次成功 | 创建单据 |
| 同幂等键再次请求且内容一致 | 返回已成功 |
| 同幂等键内容不同 | 拒绝并提示冲突 |
| 前次部分成功 | 查询金蝶结果后补偿 |

### 补偿场景

- 金蝶已创建单据，但响应外部系统超时。
- 外部系统已扣库存，金蝶未收到回传。
- 金蝶已付款，外部系统未收到状态。

补偿原则：

- 先查询结果，再决定是否重发。
- 不要盲目再次创建。
- 补偿操作要写日志。

## 七、接口监控

### 监控指标

- 成功数量。
- 失败数量。
- 失败率。
- 平均耗时。
- 最大耗时。
- 待处理数量。
- 重试次数。
- 幂等冲突数量。
- 对账差异数量。

### 告警规则

| 告警 | 示例 |
| --- | --- |
| 失败率告警 | 10 分钟失败率超过 5% |
| 积压告警 | 待处理超过 100 条 |
| 耗时告警 | 平均耗时超过 5 秒 |
| 财务告警 | 付款接口失败或重复付款风险 |
| 库存告警 | WMS 回传失败导致库存差异 |

## 八、对账

### 对账频率

| 接口 | 建议 |
| --- | --- |
| 订单 | 每日对账 |
| 库存 | 每日或更高频 |
| 付款 | 每笔或每日 |
| 发票 | 每日 |
| 凭证 | 月结前集中对账 |

### 对账内容

- 单据数量。
- 单据金额。
- 单据状态。
- 明细行数量。
- 关键字段。
- 凭证生成状态。
- 库存数量。

### 对账差异处理

```text
发现差异
  -> 按外部单号定位
  -> 查接口日志
  -> 查金蝶单据
  -> 判断失败阶段
  -> 补主数据或修配置
  -> 重试或人工处理
  -> 关闭差异
```

## 九、实际案例

### 案例：WMS 回传重复导致金蝶重复入库

现象：

- WMS 同一收货单重试两次。
- 金蝶生成两张采购入库单。
- 库存数量翻倍。

原因：

- 接口没有使用 WMS 收货单号做幂等。
- 金蝶侧没有检查外部单号是否已存在。
- WMS 网络超时后自动重试。

处理：

1. 停止接口。
2. 找到重复入库单。
3. 按业务规则红冲或作废重复单据。
4. 增加外部单号唯一校验。
5. 重试逻辑改为先查结果再补偿。
6. 建立每日 WMS 和金蝶入库对账。
