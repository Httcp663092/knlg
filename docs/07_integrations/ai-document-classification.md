来源：金蝶开发者官网“文档分类（大模型）使用手册”。

## 功能定位

文档分类能力用于处理混合附件和多类型文档。官方资料中提到，业务单据录入或审核时，用户一次处理的文件往往包含多种类型，例如费用明细表、发票、支付截图、回单凭证等；组合识别器可以自动分类，并把不同类别交给对应算法进行识别，最终输出结构化字段。

## 适用版本演进

| 版本 | 能力变化 |
| --- | --- |
| V5.0.019 | 初始版本 |
| V7.0.16 | 支持通过大模型实现自定义文档分类，可自定义分类类别，并支持分类后自动结构化提取；支持使用 AI 能力中心配置大模型进行分类 |
| V8.0 | 适配金蝶自研多模态模型和复杂文档解析 V2；支持自研多模态模型分类，支持复杂文档解析 V2 + 文本大模型分类 |
| V8.0.8 | 支持对单页混贴票据自动分类和结构化提取，分类和提取类别支持用户自定义 |

## 业务场景

以费用报销审核为例，附件可能包含：

- 费用明细表。
- 发票。
- 支付截图。
- 回单凭证。
- 混合 PDF。
- 多张图片。
- 单页混贴票据。

人工处理的问题：

- 需要先分拣附件类型。
- 需要找对应模板识别。
- 需要手工核对关键字段。
- 审核效率低，遗漏风险高。

AI 文档分类的目标：

- 自动判断附件类别。
- 自动调用对应识别能力。
- 自动提取结构化字段。
- 为单据校验、审核和归档提供数据。

## 核心对象：组合识别器

官方资料中提到，组合识别器允许业务单据开发者从自定义模板和预置模板中选择两个以上算法服务组合在一起，得到一个可以自动分类识别的新服务。

组合识别器的分类能力依赖：

- 每个模板配置的关键词。
- 自定义模板标注的锚点信息。
- 大模型或多模态模型能力。
- 复杂文档解析能力。

## 配置思路

```text
准备文档类型
  -> 准备预置模板或自定义模板
  -> 配置分类类别
  -> 组合为组合识别器
  -> 发布服务
  -> 通过低代码、微服务或 OpenAPI 调用
  -> 输出分类结果和结构化字段
```

## 字段输出设计

做系统集成时，不要只关注“识别成功”，还要设计识别结果怎么进入业务单据。

| 输出字段 | 示例 | 用途 |
| --- | --- | --- |
| 文档类别 | 发票、回单、支付截图 | 决定后续校验规则 |
| 票据号码 | 发票号码、回单号 | 防重复和追溯 |
| 金额 | 含税金额、付款金额 | 与单据金额核对 |
| 日期 | 开票日期、付款日期 | 与业务日期核对 |
| 往来单位 | 销售方、购买方、收款方 | 与客户/供应商核对 |
| 置信度 | 分类或识别可信度 | 决定是否人工复核 |

## 与业务单据结合

可落地到这些系统场景：

- 费用报销：附件自动分类，发票金额与报销金额核对。
- 应付审核：发票、入库单、合同、付款申请附件自动识别。
- 收款认领：银行回单识别付款方、金额、日期。
- 合同归档：合同文件分类和关键条款抽取。
- 电子档案：附件分类后按业务类型归档。

## 调用方式

官方资料提到，配置完成并发布后，开发者可以通过这些方式调用组合识别器：

- 低代码配置。
- 微服务。
- OpenAPI。

## 实施检查清单

- [ ] 文档类别定义清楚。
- [ ] 每类文档至少有样例数据。
- [ ] 预置模板和自定义模板已经选择。
- [ ] 关键词或锚点信息已配置。
- [ ] AI 能力中心模型配置完成。
- [ ] 输出字段和业务单据字段映射明确。
- [ ] 低置信度结果有人工复核流程。
- [ ] 调用日志、错误日志和重试机制明确。

## 常见问题

### 分类不准

排查：

1. 分类类别是否过于相似。
2. 模板关键词是否太少。
3. 样本是否覆盖真实附件。
4. 是否存在混贴、拍照歪斜、模糊图片。
5. 是否需要改用多模态模型或复杂文档解析。

### 结构化字段缺失

排查：

1. 当前类别是否配置了提取模板。
2. 字段锚点是否稳定。
3. 原始文件质量是否足够。
4. 多页 PDF 是否正确拆分。
5. 单页混贴票据是否使用支持版本。

### 和业务单据对不上

排查：

1. 金额字段是含税还是不含税。
2. 日期字段是开票日期、付款日期还是上传日期。
3. 往来单位名称是否需要映射到客户/供应商主数据。
4. 币别是否识别或默认。
5. 识别结果是否经过人工确认。
