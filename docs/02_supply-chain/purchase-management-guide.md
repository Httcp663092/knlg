采购管理的目标，是把企业的采购需求转成可执行、可收货、可对账、可付款、可追溯的业务闭环。金蝶采购不能只看采购订单，还要看供应商、价格、入库、应付、暂估、付款和总账。

## 一、采购业务总流程

```text
采购需求
  -> 采购申请
  -> 采购询价/比价
  -> 采购订单
  -> 到货/收料
  -> 质检
  -> 采购入库
  -> 应付确认
  -> 付款申请/付款单
  -> 付款核销
  -> 存货核算和总账凭证
```

不同企业会裁剪流程。小企业可能从采购订单直接入库，大型制造企业通常会有采购申请、采购订单、到货、质检、入库、发票、应付、付款和供应商考核。

## 二、采购基础资料

### 供应商

供应商不是简单名称。它会影响采购订单、结算、付款、对账和应付凭证。

| 字段 | 作用 | 检查点 |
| --- | --- | --- |
| 供应商编码 | 唯一识别 | 不允许重复供应商 |
| 供应商名称 | 展示和单据打印 | 名称变更要留痕 |
| 结算供应商 | 应付和付款对象 | 采购供应商和结算供应商可能不同 |
| 税率 | 应付税额计算 | 与发票税率一致 |
| 付款条件 | 到期日和付款计划 | 月结、账期、预付 |
| 银行账号 | 付款使用 | 必须经过审批 |
| 供应商分类 | 科目映射、统计分析 | 可影响应付科目 |

### 物料采购属性

采购前要确认物料是否允许采购。

- 采购单位和库存单位换算正确。
- 是否启用批号、保质期、序列号。
- 是否需要质检。
- 是否有最小采购量、采购提前期。
- 是否有默认供应商。
- 是否有采购价目表。

### 采购价格

价格管理常见方式：

| 方式 | 适合场景 |
| --- | --- |
| 手工录价 | 采购量小，价格不稳定 |
| 采购价目表 | 常规采购价格稳定 |
| 询价比价 | 多供应商比价 |
| 框架协议 | 长期合作供应商 |
| 合同价格 | 按合同执行 |

价格控制点：

- 是否允许低权限用户改价。
- 改价是否需要审批。
- 价目表是否有有效期。
- 含税价和不含税价是否区分。
- 税率变化后价格如何维护。

## 三、采购申请

采购申请用于把业务部门需求提交给采购部门。

### 操作步骤

1. 申请部门提出采购需求。
2. 录入物料、数量、需求日期、用途。
3. 选择建议供应商或为空。
4. 提交审批。
5. 审批通过后下推采购订单。

### 检查点

- 需求是否真实。
- 是否已有库存。
- 是否已有在途采购。
- 是否预算内。
- 是否有替代物料。
- 需求日期是否合理。

### 常见问题

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 申请不能下推订单 | 未审核、已关闭、数量已下推完 | 查状态和下推数量 |
| 物料选不到 | 物料未启用采购属性或权限不足 | 查物料和权限 |
| 需求重复 | 部门重复申请或 MRP 已产生需求 | 查在途和计划 |

## 四、采购订单

采购订单是和供应商约定采购内容的核心单据。

### 操作步骤

1. 选择采购组织。
2. 选择供应商和结算供应商。
3. 录入物料、数量、单位。
4. 录入含税单价或不含税单价。
5. 确认税率、价税合计。
6. 确认交货日期、交货仓库。
7. 确认付款条件。
8. 保存、提交、审核。

### 订单检查

| 检查项 | 说明 |
| --- | --- |
| 供应商 | 是否准入，是否正确 |
| 采购组织 | 是否和库存组织、结算组织匹配 |
| 物料 | 是否允许采购 |
| 数量 | 是否超需求、是否符合最小采购量 |
| 价格 | 是否来自价目表、合同或审批 |
| 税率 | 是否和供应商发票口径一致 |
| 交期 | 是否满足需求 |
| 付款条件 | 是否影响应付到期日 |

### 订单变更

订单审核后如果要改数量、价格、交期，建议走变更流程，而不是随意反审核修改。

变更要记录：

- 变更原因。
- 原数量和新数量。
- 原价格和新价格。
- 是否影响入库。
- 是否影响应付。
- 是否需要供应商确认。

## 五、采购到货和质检

有些企业采购先到货，再质检，合格后入库。

```text
采购订单
  -> 到货单
  -> 质检单
  -> 合格入库
  -> 不合格退货或让步接收
```

质检关注：

- 抽检数量。
- 合格数量。
- 不合格数量。
- 让步接收。
- 退货处理。
- 质量原因。

如果启用质检，采购入库不能绕过质检结果。

## 六、采购入库

采购入库会增加库存数量，是采购和库存的关键交接点。

### 操作步骤

1. 从采购订单、到货单或质检结果下推入库单。
2. 选择库存组织。
3. 选择仓库和库位。
4. 核对物料、单位、实收数量。
5. 录入批号、生产日期、保质期。
6. 保存并审核。
7. 查询即时库存。

### 入库检查

- 是否允许超订单入库。
- 是否必须质检合格后入库。
- 实收数量和订单数量是否一致。
- 批号和保质期是否完整。
- 物料单位换算是否正确。
- 入库仓库是否属于正确组织。

### 入库对财务的影响

采购入库通常影响存货数量，后续通过存货核算确认存货金额。是否立即形成应付，取决于企业应付确认口径：

| 口径 | 说明 |
| --- | --- |
| 入库确认应付 | 货到入库就确认应付或暂估 |
| 发票确认应付 | 发票到后确认正式应付 |
| 应付单确认 | 由财务确认应付单后入账 |

## 七、采购退料

采购退料用于把已入库物料退回供应商。

流程：

```text
发现退料原因
  -> 采购退料申请
  -> 采购退料单
  -> 库存减少
  -> 红字应付或应付调整
```

退料原因：

- 质量不合格。
- 供应商发错货。
- 采购数量多。
- 价格或合同争议。
- 物料不再需要。

检查点：

- 是否关联原采购入库。
- 是否退回同一供应商。
- 是否影响已确认应付。
- 是否需要红字发票。
- 是否影响存货成本。

## 八、暂估应付

暂估是采购模块和财务模块最常见的交叉点。

### 为什么需要暂估

货已经入库，企业已经取得库存和使用权，但发票还没到。如果不暂估，存货和成本可能不完整。

### 暂估流程

```text
采购入库
  -> 月末暂估
  -> 生成暂估凭证
  -> 发票到达
  -> 生成正式应付
  -> 冲回暂估
```

### 暂估检查

- 暂估金额是否按订单价、入库价或暂估价。
- 暂估应付是否带供应商维度。
- 暂估科目和正式应付科目是否区分。
- 发票到达后是否冲暂估。
- 暂估余额是否长期挂账。

## 九、采购付款

付款前要保证付款依据清楚。

### 付款前检查

- 采购订单是否存在。
- 入库或发票是否已确认。
- 应付单是否审核。
- 供应商银行账号是否审批。
- 是否到付款条件。
- 是否有预付款、扣款、折让。
- 是否存在重复付款。

### 付款流程

```text
应付单
  -> 付款申请
  -> 付款审批
  -> 付款单
  -> 银行付款
  -> 付款凭证
  -> 应付核销
```

### 核销

付款单必须和应付单核销，否则供应商账龄仍会显示未付款。

核销失败常见原因：

- 供应商不一致。
- 结算组织不一致。
- 币别不一致。
- 单据未审核。
- 已经被核销。
- 金额方向不匹配。

## 十、采购月结

月末采购要清理：

- 未审核采购订单。
- 已到货未入库。
- 已入库未应付。
- 已应付未生成凭证。
- 已付款未核销。
- 长期暂估。
- 采购退料未处理。

月结输出：

- 采购入库明细。
- 暂估应付清单。
- 供应商应付余额。
- 供应商账龄。
- 采购退料清单。
- 采购价格异常清单。

## 十一、采购常见问题

### 采购订单不能审核

排查：

1. 必填字段是否完整。
2. 供应商是否启用。
3. 物料是否允许采购。
4. 价格是否超控制范围。
5. 审批流是否配置。
6. 用户是否有审核权限。

### 采购入库后库存没增加

排查：

1. 入库单是否审核。
2. 查询库存组织和仓库是否一致。
3. 是否查询了正确批号、库位。
4. 是否入到待检或冻结状态。
5. 是否被后续退料或调拨。

### 应付和采购入库不一致

排查：

1. 应付确认口径。
2. 入库单是否生成应付。
3. 发票是否到达。
4. 是否存在暂估。
5. 是否有退料或折让。

## 十二、采购实施验收清单

- [ ] 供应商资料完整且去重。
- [ ] 采购价目表或价格审批可用。
- [ ] 采购申请到采购订单流程跑通。
- [ ] 采购订单到入库流程跑通。
- [ ] 质检流程按需启用。
- [ ] 采购退料流程跑通。
- [ ] 暂估应付口径确认。
- [ ] 应付单和付款单能生成凭证。
- [ ] 付款核销能更新供应商账龄。
- [ ] 采购、库存、应付、总账能对账。
