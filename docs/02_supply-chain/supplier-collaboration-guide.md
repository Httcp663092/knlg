供应商协同用于让供应商参与采购订单确认、送货、对账、发票和质量反馈。它的价值不只是把采购信息发给供应商，而是减少采购员手工沟通、减少到货差异、提升应付对账效率。

## 一、适用场景

- 供应商数量多，采购员沟通压力大。
- 采购订单需要供应商确认交期。
- 送货需要提前预约。
- 到货和入库差异频繁。
- 供应商对账周期长。
- 质量问题需要供应商在线反馈。

## 二、协同范围

| 场景 | 说明 |
| --- | --- |
| 供应商准入 | 供应商资料、资质、审批 |
| 订单协同 | 采购订单发布、供应商确认 |
| 送货协同 | 供应商创建送货通知或预约 |
| 到货协同 | 收货、质检、入库状态反馈 |
| 对账协同 | 采购入库、应付、发票对账 |
| 质量协同 | 不合格处理、整改反馈 |
| 绩效协同 | 交期、质量、价格、服务评价 |

## 三、主数据准备

供应商协同依赖高质量供应商资料。

必须维护：

- 供应商编码和名称。
- 联系人和邮箱或手机号。
- 供应商分类。
- 付款条件。
- 税率和发票信息。
- 结算币别。
- 银行账号。
- 准入状态。
- 供应物料范围。

建议维护：

- 资质证照。
- 质量等级。
- 交付等级。
- 合同和价格协议。
- 供应商门户账号。

## 四、采购订单协同

```text
采购订单审核
  -> 发布给供应商
  -> 供应商确认数量和交期
  -> 如有差异发起变更
  -> 采购员确认变更
  -> 形成最终交付计划
```

控制点：

- 哪些订单需要发布。
- 供应商是否允许改数量。
- 供应商是否允许改交期。
- 交期变更是否需要采购员审批。
- 超期未确认是否提醒。

## 五、送货协同

```text
供应商基于采购订单创建送货通知
  -> 录入本次送货数量、批号、车牌、预计到货时间
  -> 打印送货单或标签
  -> 仓库按送货通知收货
  -> 系统反馈实收数量
```

好处：

- 仓库提前知道到货计划。
- 采购订单和收货自动关联。
- 到货差异可追踪。
- 批号和包装信息提前进入系统。

## 六、对账协同

供应商对账通常围绕：

- 采购订单。
- 采购入库。
- 退料。
- 应付单。
- 发票。
- 付款。

### 对账流程

```text
供应商查看入库和退料
  -> 提交对账确认
  -> 采购或财务复核
  -> 生成应付或发票匹配
  -> 付款计划
```

对账口径：

- 按入库日期还是发票日期。
- 暂估是否纳入。
- 退料是否冲减。
- 含税还是不含税。
- 币别和汇率。

## 七、质量协同

质量异常可以让供应商参与：

- 不合格通知。
- 8D 整改报告。
- 退货确认。
- 让步接收确认。
- 质量扣款。
- 供应商评分。

关键是把质量问题和采购入库、批号、供应商、物料关联起来。

## 八、权限和账号

供应商账号必须严格隔离：

- 只能看到本供应商相关订单。
- 不能看到价格之外的敏感信息，除非业务需要。
- 不能看到其他供应商数据。
- 不能直接修改金蝶内部审核后的关键字段。
- 登录、确认、变更、附件上传要有日志。

## 九、常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 供应商看不到订单 | 未发布、账号未绑定供应商、权限不足 | 查发布状态和账号映射 |
| 供应商确认交期后系统未更新 | 变更需要采购员确认 | 查变更流程 |
| 送货数量超采购订单 | 超收规则未控制 | 设置超收比例和审批 |
| 对账金额不一致 | 入库、退料、税率、暂估口径不同 | 固定对账口径 |
| 供应商看到其他数据 | 数据权限配置错误 | 立即收紧供应商数据范围 |

## 十、实际案例

### 案例：供应商送货差异频繁

背景：

- 采购订单 1000 件。
- 供应商经常分批送货，但未提前通知。
- 仓库收货后采购员才知道差异。

方案：

1. 采购订单审核后发布给供应商。
2. 供应商必须创建送货通知。
3. 每次送货填本次数量、批号、预计到货时间。
4. 仓库按送货通知扫码收货。
5. 差异数量自动反馈采购员。
6. 月末按实收入库对账。

效果：

- 采购员能提前看到未到货。
- 仓库收货有来源。
- 供应商对账不再只靠线下 Excel。
