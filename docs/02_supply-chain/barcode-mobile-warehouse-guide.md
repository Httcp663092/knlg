条码和移动仓库用于提升仓库收货、上架、拣货、出库、盘点和追溯效率。金蝶项目中，条码不是简单打印一个标签，而是把物料、批号、序列号、库位、包装、单据和移动端作业连成闭环。

## 一、适用场景

- 仓库物料多，人工录入容易出错。
- 批号、序列号、保质期需要追溯。
- 收货、拣货、盘点需要移动端作业。
- 生产领料和完工入库需要扫码。
- WMS 或 MES 与金蝶集成。
- 需要按箱、托盘、包装单位管理。

## 二、条码对象

| 对象 | 说明 |
| --- | --- |
| 物料条码 | 标识物料编码、名称、规格 |
| 批号条码 | 标识批次、生产日期、有效期 |
| 序列号条码 | 标识单件唯一编号 |
| 箱码 | 标识一箱或一包物料 |
| 托盘码 | 标识一托盘多个包装 |
| 库位码 | 标识仓库中的货位 |
| 单据条码 | 标识采购订单、出库单、生产订单等 |

## 三、条码规则设计

条码规则要回答：

- 谁生成条码。
- 在哪个环节生成。
- 条码包含哪些信息。
- 条码是否唯一。
- 条码是否允许重复打印。
- 条码作废后如何处理。
- 条码与批号、序列号、包装如何关联。

### 常见规则

| 场景 | 条码建议 |
| --- | --- |
| 普通物料 | 物料编码或内部条码 |
| 批次物料 | 物料 + 批号 + 数量 |
| 序列号物料 | 一物一码 |
| 外箱 | 箱码 + 物料 + 数量 + 批号 |
| 托盘 | 托盘码 + 多箱明细 |

## 四、移动收货流程

```text
采购订单
  -> 到货或收料通知
  -> 扫供应商送货单或采购单
  -> 扫物料或箱码
  -> 录入实收数量、批号、库位
  -> 提交收料
  -> 生成采购入库
```

检查点：

- 扫码物料是否属于该采购订单。
- 实收数量是否允许超过订单数量。
- 批号和生产日期是否必填。
- 收货仓库和库位是否正确。
- 不合格品是否进入待检或不良品仓。

## 五、移动上架流程

```text
收货暂存区
  -> 扫箱码
  -> 扫目标库位
  -> 确认数量
  -> 完成上架
  -> 库存进入正式库位
```

适合启用库位管理的仓库。不上架直接入库存，容易出现“系统有库存但找不到实物”。

## 六、移动拣货和出库

### 销售出库

```text
销售出库任务
  -> 系统推荐库位和批号
  -> 仓管员扫码拣货
  -> 扫箱码或物料
  -> 复核数量
  -> 打包发货
  -> 审核销售出库
```

### 生产领料

```text
生产订单
  -> 领料任务
  -> 扫生产订单或工单
  -> 扫物料、批号、库位
  -> 确认领料数量
  -> 生成生产领料单
```

关键控制：

- 是否允许超领。
- 是否按先进先出或批号规则拣货。
- 是否必须扫描库位。
- 是否允许手工改数量。

## 七、移动盘点

```text
生成盘点方案
  -> 下发盘点任务
  -> 扫库位
  -> 扫物料或箱码
  -> 录入实盘数量
  -> 生成盘点差异
  -> 审核盘盈盘亏
```

盘点注意：

- 盘点期间是否冻结库存。
- 盘点范围是全仓、库位还是物料。
- 差异是否需要复盘。
- 盘盈盘亏是否生成财务凭证。

## 八、和金蝶库存的关系

条码移动作业最终要落到金蝶库存单据：

| 移动作业 | 金蝶结果 |
| --- | --- |
| 扫码收货 | 收料通知或采购入库 |
| 扫码上架 | 库位调整或入库确认 |
| 扫码拣货 | 销售出库或生产领料 |
| 扫码盘点 | 盘点单和盘盈盘亏 |
| 扫码退货 | 采购退料或销售退货 |

如果移动端只记录了扫码日志，没有形成正式库存单据，库存账不会真正变化。

## 九、常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 扫码识别不到物料 | 条码规则不一致或基础资料无条码 | 统一条码规则并维护物料条码 |
| 扫码后数量不对 | 箱码数量、单位换算错误 | 检查包装单位和换算率 |
| 批号没带出 | 条码未包含批号或批号规则未启用 | 条码生成时写入批号 |
| 库位库存不准 | 上架、移库未扫码或未审核 | 严格库位扫码和单据审核 |
| 重复打印标签 | 标签状态未管控 | 建打印记录和作废机制 |

## 十、实际案例

### 案例：采购收货启用箱码

背景：

- 供应商每箱 100 个物料。
- 仓库原来手工录入数量，常出现少收或多收。

方案：

1. 供应商送货时贴箱码。
2. 箱码包含物料、批号、数量、供应商、送货单。
3. 仓管员扫码收货。
4. 系统自动带出采购订单和物料。
5. 异常数量进入差异处理。
6. 质检通过后扫码上架。

效果：

- 收货数量由箱码带出。
- 批号自动进入库存。
- 后续销售出库和质量追溯能查到来源。
