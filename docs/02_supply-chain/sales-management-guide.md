销售管理的目标，是把客户需求转成订单、发货、出库、应收、收款和收入成本闭环。销售模块最容易和财务、库存、信用、价格、发票、退货发生交叉。

## 一、销售业务总流程

```text
销售线索/报价
  -> 销售订单
  -> 信用检查
  -> 发货通知
  -> 销售出库
  -> 应收确认
  -> 收款核销
  -> 收入和成本凭证
```

常见变体：

- 先款后货：先收款，再发货。
- 先货后款：先出库应收，再收款。
- 按发票确认收入。
- 按出库或签收确认收入。
- 退货、换货、折让、红冲。

## 二、销售基础资料

### 客户

客户资料影响销售订单、应收、收款、账龄和信用。

| 字段 | 作用 | 检查点 |
| --- | --- | --- |
| 客户编码 | 唯一识别 | 不允许重复客户 |
| 客户名称 | 展示和单据打印 | 名称变更要留痕 |
| 结算客户 | 应收对象 | 收货客户和结算客户可能不同 |
| 信用额度 | 控制超额销售 | 是否启用信用控制 |
| 收款条件 | 应收到期日 | 影响账龄 |
| 税率 | 应收税额 | 和开票口径一致 |
| 客户分类 | 科目和分析 | 可影响收入或应收科目 |

### 销售价格

价格来源：

- 销售价目表。
- 客户协议价。
- 促销价。
- 手工改价。
- 外部系统同步价格。

价格控制：

- 是否允许低于最低价。
- 折扣是否审批。
- 是否按客户、物料、区域、渠道定价。
- 含税价和不含税价是否区分。
- 价格有效期是否维护。

## 三、销售订单

销售订单是客户需求和公司履约承诺。

### 操作步骤

1. 选择销售组织。
2. 选择客户、收货客户、结算客户。
3. 录入物料、数量、单位、价格、税率。
4. 录入交货日期和收货地址。
5. 确认发货仓库或发货组织。
6. 检查信用额度和价格。
7. 保存、提交、审核。

### 检查点

| 检查项 | 说明 |
| --- | --- |
| 客户 | 是否启用，是否可交易 |
| 结算客户 | 后续应收和收款对象 |
| 信用额度 | 是否超信用 |
| 价格 | 是否有效，是否低价 |
| 交期 | 是否可履约 |
| 库存 | 是否有可用库存 |
| 税率 | 应收税额是否正确 |

## 四、信用控制

信用控制用于防止客户欠款过多仍继续发货。

常见控制口径：

| 控制项 | 说明 |
| --- | --- |
| 信用额度 | 客户最大欠款额度 |
| 信用期限 | 逾期天数控制 |
| 订单占用 | 未出库订单是否占用信用 |
| 应收占用 | 已确认应收未收款 |
| 逾期控制 | 有逾期即禁止发货或需审批 |

信用异常处理：

- 临时放行要审批。
- 放行原因要留痕。
- 财务和销售要共同确认。
- 不要长期绕过信用控制。

## 五、发货和销售出库

### 发货通知

发货通知用于安排仓库发货。它可以和销售订单分开，适合有物流、拣货、复核流程的企业。

### 销售出库操作步骤

1. 从销售订单或发货通知下推销售出库单。
2. 选择库存组织、仓库、库位。
3. 检查可用库存。
4. 录入实际出库数量。
5. 选择批号、序列号。
6. 审核销售出库单。
7. 查询库存减少。

### 出库检查点

- 是否超订单出库。
- 是否低于订单数量部分出库。
- 批号和库位是否正确。
- 是否允许负库存。
- 是否影响信用额度。
- 是否触发应收确认。

## 六、应收确认

应收确认口径要在蓝图中明确。

| 口径 | 说明 |
| --- | --- |
| 按销售出库确认 | 发货即确认收入和应收 |
| 按签收确认 | 客户签收后确认 |
| 按开票确认 | 发票开具后确认 |
| 按项目节点确认 | 项目履约节点确认 |

应收单检查：

- 客户和结算客户。
- 销售组织和结算组织。
- 业务日期。
- 到期日。
- 含税金额、未税金额、税额。
- 币别和汇率。
- 是否生成凭证。

## 七、收款和核销

收款流程：

```text
银行到账
  -> 收款单
  -> 收款凭证
  -> 收款核销
  -> 客户余额和账龄更新
```

收款核销检查：

- 收款客户和应收客户一致。
- 结算组织一致。
- 币别一致。
- 金额方向一致。
- 单据状态已审核。
- 未被其他收款核销。

账龄不准，多数不是凭证问题，而是核销不及时或核销错对象。

## 八、销售退货、退款和折让

### 销售退货

```text
退货申请
  -> 销售退货单
  -> 退货入库
  -> 红字应收
  -> 成本冲回
```

检查点：

- 是否关联原销售出库。
- 退货数量是否超过原出库。
- 退货入库仓库是否正确。
- 是否影响已开票。
- 是否需要退款。
- 成本是否冲回。

### 销售折让

折让不一定退货，可能只调整应收金额。

处理方式：

- 红字应收。
- 费用或折扣单据。
- 开票红冲。
- 应收调整。

财务要确认收入、税额和应收是否调整正确。

## 九、销售与成本

销售出库减少库存，存货核算后形成销售成本。

```text
销售出库
  -> 出库成本计算
  -> 生成成本凭证
  -> 主营业务成本
```

常见分录：

收入：

```text
借：应收账款
贷：主营业务收入
贷：应交税费-销项税额
```

成本：

```text
借：主营业务成本
贷：库存商品
```

检查点：

- 销售出库是否已审核。
- 出库成本是否计算。
- 成本凭证是否生成并过账。
- 收入和成本期间是否匹配。

## 十、销售月结

月末要清理：

- 未审核销售订单。
- 已订单未发货。
- 已发货未出库。
- 已出库未应收。
- 已应收未生成凭证。
- 已收款未核销。
- 销售退货未处理。
- 成本未结转。

输出：

- 销售订单执行表。
- 销售出库明细。
- 应收明细和账龄。
- 收款核销表。
- 销售收入和成本对比。
- 毛利分析表。

## 十一、常见问题

### 销售订单不能审核

可能原因：

- 客户被禁用。
- 价格低于控制线。
- 超信用额度。
- 物料不可销售。
- 审批流未配置。
- 必填字段缺失。

### 销售出库失败

可能原因：

- 库存不足。
- 批号或序列号未指定。
- 仓库不属于当前组织。
- 订单未审核。
- 信用控制未通过。
- 不允许负库存。

### 应收和销售出库对不上

排查：

1. 查看应收确认口径。
2. 查销售出库是否生成应收。
3. 查是否存在部分出库。
4. 查退货和折让。
5. 查应收单状态和凭证。

## 十二、销售实施验收清单

- [ ] 客户资料、结算客户和信用资料完整。
- [ ] 销售价目表或价格审批可用。
- [ ] 销售订单到销售出库流程跑通。
- [ ] 信用控制按规则生效。
- [ ] 销售出库能减少库存。
- [ ] 应收单能按口径生成。
- [ ] 收款和应收核销跑通。
- [ ] 销售退货和退款流程跑通。
- [ ] 收入和成本凭证能生成。
- [ ] 销售、库存、应收、总账能对账。
