## 二、供应商协同的官方能力怎么理解

官方资料把供应商协同定位为企业和供应商之间的在线业务协同平台，核心功能包括：

- 询报价协同。
- 订单协同。
- 交货协同。
- 库存协同。
- 信息协同。

项目里可以理解为：

```text
企业采购需求
  -> 供应商在线报价
  -> 采购订单发布
  -> 供应商确认交期
  -> 供应商送货
  -> 企业收货质检
  -> 入库和应付
  -> 对账和付款
```

## 三、启用供应商协同的关键前提

官方供应商资料提到供应商商务信息中可设置是否启用供应商协同。项目落地时要注意：

- 不是所有供应商都必须启用协同。
- 可按供应商重要程度、合作频率、信息化能力分批启用。
- 启用前要确认供应商账号、联系人、邮箱、权限和可见数据范围。

### 启用前检查

- 供应商编码是否唯一。
- 供应商名称、税号、银行账号是否准确。
- 是否维护联系人。
- 是否启用供应商协同。
- 是否配置供应商门户账号。
- 是否限制供应商只能看自己的订单。

## 四、采购申请单中的执行状态值得重点学习

官方采购申请单资料中有很多字段和状态，项目现场尤其要理解：

| 字段或状态 | 项目解释 |
| --- | --- |
| 关联数量 | 下游单据已经关联申请的数量 |
| 订单数量 | 采购订单已关联的数量 |
| 剩余数量 | 还能继续下推或关联的数量 |
| 收料数量 | 已收料数量 |
| 入库数量 | 已入库数量 |
| 业务关闭 | 分录已经执行完成 |
| 业务终止 | 人工决定不再执行 |
| 计划跟踪号 | 用于跟踪计划需求来源 |
| 需求跟踪号 | 用于跟踪采购需求来源 |

### 为什么重要

采购申请不是“审批完就结束”，它要继续追踪：

- 是否已转采购订单。
- 是否已收料。
- 是否已入库。
- 是否还有剩余未执行。
- 是否被业务终止。

如果这些状态没看懂，就容易误判采购需求是否完成。

## 五、基础资料控制和自动分配

多组织供应链项目中，物料、供应商、客户、部门等基础资料经常需要跨组织共享或隔离。官方资料里的基础资料控制策略和自动分配计划，对多组织项目非常重要。

### 项目理解

```text
创建组织维护基础资料
  -> 按控制策略分配到目标组织
  -> 目标组织可引用或使用
  -> 分配结果影响录单、权限和报表
```

### 常见控制方式

- 全局共享。
- 按组织创建。
- 从集团或主组织分配。
- 自动分配。
- 按需分配。

### 自动分配适用场景

- 集团统一维护物料。
- 新供应商需要自动分配到多个采购组织。
- 客户资料要分配到多个销售组织。
- 部门资料需要下发到多个组织。

### 风险

- 分配顺序不对，导致引用资料缺失。
- 目标组织未审核，导致业务单据不能使用。
- 私有资料和分配资料混用，导致同名不同码。
- 供应商或物料分配了，但价格、税率、库存属性未同步。

## 六、业务监控在供应链中的用法

官方业务监控资料区分事后提醒和风险预警。供应链项目可以用它做：

- 采购订单超期未到货提醒。
- 采购申请长期未下单提醒。
- 销售订单超期未出库提醒。
- 库存低于安全库存提醒。
- 临期批号提醒。
- 未审核出入库单提醒。
- 供应商交期异常提醒。

### 监控设计模板

| 项目 | 示例 |
| --- | --- |
| 监控对象 | 采购订单 |
| 监控条件 | 交货日期小于今天且未入库数量大于 0 |
| 提醒对象 | 采购员、采购经理 |
| 频率 | 每天 9 点 |
| 处理动作 | 联系供应商或调整计划 |

## 七、供应链实施加深清单

- [ ] 供应商是否分级启用协同。
- [ ] 采购申请执行状态是否用于追踪。
- [ ] 计划跟踪号和需求跟踪号是否传递到采购订单。
- [ ] 基础资料控制策略是否区分集团共享和组织私有。
- [ ] 自动分配计划是否测试过分配顺序。
- [ ] 采购超期、库存低量、临期批号是否配置业务监控。
- [ ] 供应商协同数据权限是否严格隔离。
