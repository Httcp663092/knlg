来源：金蝶开发者官网 `productLineId=29`、`productId=87` 下的“2.项目创建”和相关项目创建资料。

## 适用范围

- 产品线：金蝶云苍穹。
- 产品页：金蝶 AI 苍穹。
- 场景：进入开发者工作台、创建团队/开发商标识、创建项目、本地 Gradle debug 工程。
- 入口：[金蝶开发者官网产品页](https://developer.kingdee.com/?productId=87&productLineId=29&lang=zh-CN)。

## 开发者工作台入口

官方资料给出的工作台访问地址是：

```text
https://dev.kingdee.com
```

进入方式：

1. 访问开发者工作台地址。
2. 登录账号。
3. 从首页右上角进入“工作台”。
4. 首次登录时，需要创建或选择团队。
5. 后续可以在工作台内切换不同团队或项目。

## 团队、开发商标识、项目的关系

官方资料中提到：团队创建过程也就是开发商标识申请过程，一个团队对应一个开发商标识。

| 对象 | 作用 | 注意点 |
| --- | --- | --- |
| 团队 | 开发协作的组织单元 | 首次进入工作台时需要创建或选择 |
| 开发商标识 | 项目开发身份标识 | 一个团队一个开发商标识 |
| 项目 | 具体开发交付对象 | 项目归属团队和开发商标识 |

## 开发商标识选择

客户定制化开发项目，官方资料建议使用类型为“金蝶客户”的开发商标识，申请开发商标识时组织类型选择“金蝶客户”。

POC 阶段可以由开发团队先代客户申请开发标识，待客户商务手续完成后，再把“开发商标识管理员”权限移交给客户负责人。

## 本地 Gradle debug 工程准备

官方资料中“苍穹 Gradle 工程项目创建”提到，本地 debug 工程搭建完成后，可以在本地进行 Java 插件开发，也可以在本地 IDEA 启动一个苍穹微服务节点进行插件功能调试。

创建工程前需要准备四类关键信息：

| 信息 | 用途 | 获取位置 |
| --- | --- | --- |
| 苍穹资源地址 | 下载工程依赖 jar 包和静态资源 | 轻量级环境从 Cosmic Studio 环境详情查看；安装器环境从 MC 集群管理同步开发资源处查看 |
| MC 访问地址 | 访问开发服务器管理中心 | 轻量级环境复制 Cosmic Studio 环境详情地址；容器版从 MC 系统管理查看 |
| Zookeeper 地址 | 本地微服务连接注册中心 | 一般形如 `服务器IP:2181?user=zookeeper&password=xxx`，可在 MC 中查看 |
| 开发商标识 | 标识当前项目的开发商 | 四位英文数字字符，向项目负责人获取 |

## IDEA 创建 Gradle 工程

官方资料给出的操作路径：

```text
File -> New -> Project -> 金蝶AI苍穹Gradle工程 -> 填写工程信息 -> 创建
```

常见字段：

| 字段 | 说明 |
| --- | --- |
| 项目目录 | 本地磁盘上的空目录 |
| 项目名称 | 通常自动取项目目录文件夹名称 |
| JDK 版本 | 与苍穹开发环境要求保持一致 |
| 苍穹资源地址 | 用于自动下载并引入工程依赖 |
| MC 访问地址 | 本地 debug 连接管理中心 |
| Zookeeper 地址 | 微服务注册和发现 |
| 开发商标识 | 与开发者工作台项目保持一致 |

## 本地调试检查清单

- [ ] IDEA 已安装苍穹开发助手或相关插件。
- [ ] JDK 版本与项目环境一致。
- [ ] 苍穹资源地址可访问。
- [ ] MC 地址可访问。
- [ ] Zookeeper 地址、账号和密码正确。
- [ ] 开发商标识与团队/项目一致。
- [ ] 本地 Gradle 依赖下载成功。
- [ ] 本地微服务节点启动后能在环境中被识别。

## 常见问题

### 资源依赖下载失败

优先检查苍穹资源地址是否能访问、网络是否能连到开发服务器、地址是否来自当前环境。

### 本地服务启动后注册不上

优先检查 Zookeeper 地址、账号密码、端口、防火墙和当前环境是否一致。

### 插件代码没有生效

排查顺序：

1. 本地微服务是否启动。
2. 服务是否注册到正确环境。
3. 开发商标识是否一致。
4. 表单、操作或插件配置是否发布。
5. 是否存在缓存或旧包影响。
