金蝶 BOS/苍穹二开中，插件常用于处理标准配置无法覆盖的校验、字段联动、单据转换、接口调用、批量处理和后台任务。插件不是万能工具，写插件之前必须先确认是否可以通过标准参数、表单规则、工作流、单据转换、报表配置解决。

## 一、插件适用场景

| 场景 | 是否适合插件 | 说明 |
| --- | --- | --- |
| 字段必填和简单校验 | 不一定 | 优先用表单规则或校验规则 |
| 金额、数量复杂校验 | 适合 | 需要跨字段或跨单据判断 |
| 审核后生成下游数据 | 适合 | 但要保证幂等和事务 |
| 调用外部接口 | 适合 | 不建议阻塞用户操作太久 |
| 批量处理数据 | 适合 | 建议后台任务或服务插件 |
| 修改已过账财务数据 | 不适合 | 破坏审计和账务一致性 |

## 二、插件类型

| 插件类型 | 常见用途 |
| --- | --- |
| 表单插件 | 页面加载、字段联动、按钮控制、提示信息 |
| 操作插件 | 保存、提交、审核、反审核前后校验 |
| 列表插件 | 列表按钮、批量操作、过滤条件 |
| 服务插件 | 后台逻辑、复杂计算、数据处理 |
| 单据转换插件 | 下推、字段映射、特殊反写 |
| 定时任务 | 定时同步、检查、补偿 |

## 三、开发前设计

插件开发前必须写清：

```text
需求编号：
业务场景：
触发对象：
触发时机：
输入字段：
输出字段：
涉及组织：
涉及单据状态：
是否影响库存：
是否影响财务：
异常提示：
日志字段：
测试用例：
回滚方案：
```

如果写不清触发时机和异常处理，不要急着开发。

## 四、触发时机选择

| 时机 | 适合做 | 不适合做 |
| --- | --- | --- |
| 页面加载 | 初始化默认值、控制字段显示 | 大量查库 |
| 字段变更 | 联动带出、简单计算 | 复杂跨系统调用 |
| 保存前 | 必填、格式、基础业务校验 | 耗时接口 |
| 保存后 | 写日志、补充辅助数据 | 破坏主单据事务 |
| 提交前 | 是否满足提交条件 | 修改核心金额 |
| 审核前 | 关键业务规则校验 | 调用不稳定外部接口 |
| 审核后 | 生成下游、通知、状态回写 | 无幂等地重复创建 |
| 反审核前 | 检查下游引用 | 强行删除下游数据 |

## 五、插件设计原则

### 幂等

审核后生成下游或调用接口时，必须避免重复执行。

幂等键可以由：

- 单据内码。
- 单据编号。
- 单据分录内码。
- 外部系统编号。
- 业务类型。

组成。

### 日志

关键插件必须写日志：

- 执行时间。
- 单据编号。
- 用户。
- 输入摘要。
- 输出结果。
- 错误信息。
- 重试次数。

### 异常提示

错误提示要让业务用户看得懂。

不好：

```text
NullPointerException
```

更好：

```text
供应商银行账号为空，无法提交付款申请。请先维护供应商结算资料。
```

## 六、开发环境和分支

建议环境：

```text
本地开发
  -> DEV 单元测试
  -> TEST 集成测试
  -> UAT 用户验证
  -> PROD 发布
```

代码管理要求：

- 每个需求一个分支或明确提交。
- 发布包和代码版本对应。
- 配置变更和代码变更一起记录。
- 生产问题修复要回合并到主干。

## 七、调试方法

### 调试前

- 确认插件已注册到正确对象。
- 确认触发时机正确。
- 确认环境版本和发布包正确。
- 准备可重复测试数据。
- 使用普通用户账号测试权限影响。

### 日志排查

日志要看：

- 插件是否进入。
- 输入字段是否为空。
- 查询条件是否正确。
- 外部接口是否返回错误。
- 事务是否回滚。
- 异常是否被吞掉。

### 常见错误

| 错误 | 可能原因 |
| --- | --- |
| 插件不执行 | 注册对象或事件错误、发布未生效 |
| 字段取不到值 | 字段标识错误、数据还未加载 |
| 保存时报错 | 校验逻辑、必填字段、事务异常 |
| 审核后重复生成 | 缺少幂等控制 |
| 发布后旧逻辑仍生效 | 缓存、版本、环境不一致 |

## 八、财务相关插件注意

涉及财务时必须回归：

- 凭证模板。
- 应收应付生成。
- 核销。
- 资产折旧。
- 存货核算。
- 总账过账。
- 月结。
- 报表公式。

不要通过插件直接修改已过账凭证、核销结果、结账状态和库存余额。

## 九、性能注意

插件常见性能风险：

- 保存时循环查库。
- 列表加载时查大量数据。
- 审核时同步调用慢接口。
- 批量操作逐条提交事务。
- 没有过滤组织和期间。

优化建议：

- 批量查询，减少循环查库。
- 只查必要字段。
- 加组织、期间、状态过滤。
- 慢接口异步处理或后台补偿。
- 对批量任务分批执行。

## 十一、实际案例

### 案例：审核后重复推送 WMS

现象：

- 销售出库单审核后推送 WMS。
- 用户反审核再审核后，WMS 收到两次同一出库单。

原因：

- 插件只在审核后调用接口，没有检查是否已推送。
- 外部系统也没有用外部单号做幂等。

处理：

1. 金蝶侧增加推送状态和推送批次。
2. 以金蝶单号作为幂等键。
3. 重复审核时，如果内容未变化，返回已推送结果。
4. 内容变化时生成变更消息，而不是重复创建。
5. 补充接口对账表。
