来源：金蝶开发者官网产品页中的“单据建模（开发平台）”和“基础资料建模”。

## 适用范围

- 单据：用于保存业务流水数据的业务对象。
- 基础资料：用于被其他表单引用的业务对象，例如物料、币别、科目。
- 场景：BOS/开发平台页面创建、业务对象设计、基础资料字段引用、系统数据建模。

## 单据是什么

官方资料中对单据的定位是：单据是最常见的页面类型，用于保存业务流水数据。创建单据时，系统会自动生成表单、列表、移动表单、移动列表四个页面。

单据相比动态表单，多了这些能力：

- 数据存储能力。
- 数据查询能力。
- 预置编码、组织等常用字段。
- 预置新增、保存、删除等操作。
- 自带表单、列表、移动端表单、移动端列表框架逻辑。

## 什么时候建单据

适合建单据的场景：

| 场景 | 示例 |
| --- | --- |
| 业务流水 | 采购申请、销售订单、费用申请 |
| 需要保存和查询 | 审批单、登记单、处理单 |
| 有单据状态 | 暂存、提交、审核、关闭 |
| 需要下推或转换 | 申请单下推订单、订单下推出库 |
| 需要移动端录入 | 移动审批、移动填报 |

不适合直接建单据的场景：

- 只展示数据，没有持久化要求。
- 临时查询页面。
- 纯报表或分析页面。
- 更适合用基础资料表达的主数据。

## 单据设计要点

| 设计点 | 建议 |
| --- | --- |
| 编码规则 | 先确定编码是否自动生成，是否按组织、日期或业务类型分段 |
| 组织字段 | 多组织环境下必须明确创建组织、使用组织、业务组织 |
| 状态字段 | 明确暂存、提交、审核、关闭、作废等状态 |
| 分录体 | 明细行数据用分录体承载，例如物料明细、费用明细 |
| 操作按钮 | 新增、保存、提交、审核、反审核、删除、关闭要有权限控制 |
| 列表查询 | 常用过滤条件提前设计，例如日期、组织、状态、创建人 |
| 移动端 | 如果要移动审批或填报，移动表单字段要单独简化 |

## 基础资料是什么

官方资料中提到：基础资料是一种特殊的单据，可以在基础资料字段中引用。物料、币别、科目都是典型基础资料。

金蝶 AI 苍穹基础资料特点：

- 基于单据，继承动态表单和单据特性。
- 可以用一个编码代表某个业务对象。
- 可以被其他表单作为基础资料字段引用。
- 预置编码、名称、组织等常用字段。
- 预置新增、保存、删除等操作。
- 自带表单、列表、移动端表单、移动端列表逻辑。
- 带组织模板支持按组织进行基础资料管控。
- 不带组织模板不需要按组织隔离。

## 基础资料模板选择

| 模板类型 | 适用场景 |
| --- | --- |
| 带组织基础资料 | 物料、客户、供应商等需要组织隔离的数据 |
| 不带组织基础资料 | 币别、地区、公共分类等全局共享数据 |
| 树形基础资料 | 行政组织、分类、目录、科目类层级数据 |

## 树形基础资料关键字段

官方“基础资料建模”中提到树形基础资料常见属性：

| 字段 | 作用 |
| --- | --- |
| 级次 | 表示节点层级，通常由上级级次 + 1 得出 |
| 长编码 | 由上级长编码和当前编码组合生成 |
| 长名称 | 由上级长名称和当前名称组合生成 |
| 是否叶子 | 被其他数据引用为上级节点后，会自动变为非叶子 |
| 上级 | 当前节点的父节点，也是一个基础资料字段 |
| 长编码界定符 | 用于拼接长编码，默认常见为 `.` |

## 长编码界定符注意事项

树形基础资料依赖长编码界定符生成长编码。如果业务编码本身包含默认界定符，例如编码中也有 `.`，会和默认界定符冲突，可能导致搜索或层级识别异常。

处理建议：

1. 先确定业务编码是否可能包含默认界定符。
2. 如果会冲突，修改长编码界定符为业务编码中不会出现的字符。
3. 启用或补充长编码界定符校验。
4. 上线前导入一批真实编码测试搜索、引用和层级展开。

## 建模落地步骤

### 建单据

1. 进入开发平台对应业务云和应用。
2. 创建页面。
3. 选择 PC 端。
4. 选择单据模板。
5. 录入编码和名称。
6. 维护单据头字段、分录字段、操作、列表。
7. 配置权限、发布、预览。
8. 用真实业务数据测试新增、保存、提交、审核、查询。

### 建基础资料

1. 进入开发平台对应业务云和应用。
2. 创建页面。
3. 选择 PC 端。
4. 选择基础资料模板。
5. 根据是否需要组织隔离选择带组织或不带组织模板。
6. 如果是层级数据，选择树形基础资料模板。
7. 维护编码、名称、组织、上级、状态等字段。
8. 发布后在业务单据中用基础资料字段引用。

## 建模检查清单

- [ ] 对象应该建成单据、动态表单、基础资料还是报表已经判断清楚。
- [ ] 编码规则明确。
- [ ] 组织隔离规则明确。
- [ ] 状态流转明确。
- [ ] 权限范围明确。
- [ ] 列表常用过滤条件明确。
- [ ] 移动端是否需要简化表单已确认。
- [ ] 是否需要被其他单据引用已确认。
- [ ] 树形基础资料界定符没有和业务编码冲突。

## 常见错误

### 把主数据做成普通单据

后果：其他单据难以通过基础资料字段引用，后续扩展和权限控制都麻烦。

处理：如果数据具有编码、名称、启用状态、被多处引用等特征，优先建基础资料。

### 树形基础资料编码含默认界定符

后果：长编码拼接冲突，搜索、层级展开或引用可能异常。

处理：修改长编码界定符，增加校验，重新验证历史数据。

### 忽略组织隔离

后果：多组织环境下数据串用，权限和报表口径混乱。

处理：上线前确认数据是全局共享、创建组织隔离，还是使用组织分配。
