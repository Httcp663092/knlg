金蝶项目中的报表问题，往往不是“SQL 不会写”，而是指标口径没有定义清楚。老板看经营报表、财务看法定报表、业务看执行报表，三类报表的口径不一定相同。做报表前必须先定义指标、维度、过滤条件和数据来源。

## 一、报表分类

| 类型 | 示例 | 特点 |
| --- | --- | --- |
| 法定财务报表 | 资产负债表、利润表、现金流量表 | 口径严格，以总账为准 |
| 管理报表 | 部门利润、项目利润、产品毛利 | 口径由管理需要定义 |
| 业务执行报表 | 采购未到货、销售未出库、库存呆滞 | 以业务单据状态为主 |
| 对账报表 | 应收对总账、库存对存货 | 关注差异原因 |
| 运营看板 | 销售额、回款、库存周转 | 关注及时性和趋势 |

## 二、指标定义模板

每个指标都要写清：

| 项目 | 示例 |
| --- | --- |
| 指标名称 | 销售收入 |
| 业务含义 | 已确认收入的销售金额 |
| 数据来源 | 销售出库、应收单、总账收入科目 |
| 时间口径 | 出库日期、开票日期、记账期间 |
| 金额口径 | 含税、不含税、本位币 |
| 状态口径 | 已审核、已过账 |
| 组织口径 | 销售组织、核算组织 |
| 维度 | 客户、产品、部门、业务员 |
| 排除项 | 内部交易、样品、赠品 |

## 三、常见指标口径

### 销售额

常见口径：

- 按销售订单：反映接单。
- 按销售出库：反映发货。
- 按应收单或发票：反映结算。
- 按总账收入科目：反映财务确认收入。

四个口径都可能叫“销售额”，但数值不同。报表标题必须写明。

### 回款

常见口径：

- 收款单金额。
- 已核销收款金额。
- 银行到账金额。
- 总账银行科目借方发生额。

如果看客户回款完成率，通常要用已核销口径；如果看资金到账，通常看银行流水或收款单。

### 库存金额

常见口径：

- 即时库存数量乘参考成本。
- 存货核算余额。
- 总账存货科目余额。

管理看板可以用估算成本，但财务对账必须用存货核算和总账口径。

### 毛利

```text
毛利 = 销售收入 - 销售成本
毛利率 = 毛利 / 销售收入
```

注意：

- 收入和成本必须用同一期间。
- 退货要冲减收入和成本。
- 赠品、样品、内部交易要明确是否剔除。
- 销售成本必须来自存货核算或财务口径。

## 四、维度设计

### 常用维度

| 维度 | 用途 |
| --- | --- |
| 组织 | 集团、公司、工厂、销售组织 |
| 部门 | 费用归属、绩效分析 |
| 客户 | 销售、应收、回款 |
| 供应商 | 采购、应付、付款 |
| 物料 | 销售、库存、成本 |
| 产品分类 | 产品线分析 |
| 项目 | 项目收入、成本、费用 |
| 业务员 | 销售绩效 |
| 仓库 | 库存分析 |

维度过多会让报表难维护。先确定管理动作，再确定维度。

## 五、报表取数路径

### 从业务到财务

```text
业务单据
  -> 业务报表
  -> 核算结果
  -> 凭证
  -> 总账
  -> 财务报表
```

选择数据源时要问：

- 报表是要及时性还是准确性。
- 是否允许未审核数据。
- 是否需要和总账一致。
- 是否需要追溯到原始单据。

### 业务执行报表

可以从业务单据取数：

- 采购订单执行表。
- 销售订单执行表。
- 库存明细账。
- 生产订单执行情况。

### 财务报表

优先从总账和核算结果取数：

- 科目余额表。
- 凭证分录。
- 应收应付余额。
- 存货核算结果。
- 固定资产折旧。

## 六、报表开发流程

```text
确认报表目的
  -> 定义指标口径
  -> 确认数据源
  -> 设计维度和过滤条件
  -> 写 SQL 或配置报表
  -> 抽样验证
  -> 和前台报表对账
  -> 用户确认
  -> 固化维护说明
```

## 七、报表需求文档

报表需求必须包含：

- 报表名称。
- 使用对象。
- 使用场景。
- 刷新频率。
- 权限范围。
- 字段清单。
- 指标公式。
- 过滤条件。
- 排序和分组。
- 导出要求。
- 数据源。
- 验证方式。

## 八、数据验证

### 抽样验证

抽样方式：

- 按单据编号抽样。
- 按客户或供应商抽样。
- 按物料抽样。
- 按期间抽样。
- 按异常金额抽样。

### 汇总验证

汇总验证：

- 报表总数和前台标准报表比较。
- 分组织汇总和集团汇总比较。
- 分客户汇总和应收余额比较。
- 分物料库存和库存台账比较。
- 利润报表和总账发生额比较。

## 九、常见报表问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 用户说数不对 | 没定义口径 | 先确认时间、状态、组织、币别 |
| 报表和总账不一致 | 使用业务口径 | 标注口径或改用总账取数 |
| 报表很慢 | 大表无过滤、维度过多 | 加期间组织过滤，优化 SQL |
| 同一客户重复 | 客户资料重复或分组织不同 | 建主数据清洗和映射 |
| 毛利为负 | 成本异常、退货、收入成本期间错配 | 回查销售出库和存货核算 |

## 十、实际案例

### 案例：老板报表销售额和财务收入不一致

现象：

- 老板报表显示 6 月销售额 500 万。
- 财务利润表收入 460 万。
- 差异 40 万。

排查：

1. 老板报表取销售出库已审核金额。
2. 财务收入取总账主营业务收入过账凭证。
3. 6 月有 30 万出库未开票，未生成收入凭证。
4. 还有 10 万样品出库被老板报表计入销售额，但财务未确认收入。

处理：

- 老板报表改名为“销售出库额”。
- 新增“财务确认收入”指标。
- 样品出库单据类型单独排除。
- 报表说明中写清口径。

结论：

- 两个数字都不是错，是口径不同。

## 十一、报表维护

报表上线后要维护：

- 指标口径变更记录。
- 数据源变更记录。
- 权限变更记录。
- 性能优化记录。
- 用户反馈和修复记录。

不要让报表变成没人敢改、没人知道口径的黑盒。

## 十二、检查清单

- [ ] 报表目的已确认。
- [ ] 指标口径已写明。
- [ ] 时间、组织、状态、币别已定义。
- [ ] 维度和过滤条件已确认。
- [ ] 数据源已确认。
- [ ] 权限范围已确认。
- [ ] 抽样和汇总验证通过。
- [ ] 用户签字或确认记录完整。
- [ ] 报表维护责任人明确。
