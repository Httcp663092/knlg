金蝶系统的数据模型通常围绕“基础资料、业务单据、单据关联、账务结果、日志状态”展开。不同产品、版本、补丁和二开会影响表结构，所以学习数据模型的重点不是死记表名，而是能根据前台单据找到后台数据、根据后台数据解释前台现象。

## 一、数据模型阅读原则

- 先找前台单据，再查后台数据。
- 先确定业务口径，再写 SQL。
- 先查头表和分录，再查关联和账务结果。
- 先查询，不直接修改。
- 先在测试环境验证，再用于生产排查。
- 结果必须回前台报表或单据验证。

## 二、常见数据层级

### 基础资料

基础资料通常被业务单据引用。

| 对象 | 示例 | 关注点 |
| --- | --- | --- |
| 组织 | 核算组织、业务组织、库存组织 | 多组织隔离和汇总 |
| 人员 | 员工、用户、业务员、采购员 | 用户和人员绑定 |
| 客商 | 客户、供应商 | 名称、税号、状态、结算资料 |
| 物料 | 原材料、半成品、成品、服务 | 属性、单位、库存、采购、销售、生产 |
| 仓库 | 仓库、库位 | 库存组织、库存状态 |
| 科目 | 会计科目 | 余额方向、辅助核算 |

### 业务单据

业务单据一般至少有：

```text
单据头
  -> 单据体分录
  -> 子分录或明细
  -> 单据状态
  -> 上下游关联
  -> 操作日志
```

单据头常见信息：

- 单据编号。
- 单据类型。
- 业务日期。
- 组织。
- 客户或供应商。
- 部门和业务员。
- 币别和汇率。
- 单据状态。
- 创建人、审核人、审核日期。

单据体常见信息：

- 物料。
- 数量。
- 单价。
- 金额。
- 税率和税额。
- 仓库和库位。
- 批号。
- 源单编号。
- 关闭状态。

## 三、从前台定位后台数据

### 第一步：前台拿证据

前台先记录：

- 菜单路径。
- 单据编号。
- 单据日期。
- 组织。
- 单据状态。
- 关键字段截图。
- 报错原文。

### 第二步：查单据头

SQL 思路：

```sql
-- 示例为结构思路，表名和字段必须以实际环境为准
select *
from 业务单据头表
where 单据编号 = '单据号';
```

要确认：

- 是否查到唯一记录。
- 单据类型是否正确。
- 组织是否正确。
- 状态是否和前台一致。
- 日期和期间是否正确。

### 第三步：查单据体

```sql
select *
from 业务单据分录表
where 单据头内码 = '头表内码';
```

要确认：

- 分录行数是否和前台一致。
- 物料、数量、金额是否一致。
- 仓库、批号、单位是否正确。
- 是否有关闭或锁定状态。

### 第四步：查关联关系

很多问题不是单据本身错，而是上下游断链。

```text
销售订单
  -> 发货通知
  -> 销售出库
  -> 应收单
  -> 收款单
  -> 凭证
```

查关联时关注：

- 源单 ID。
- 源单分录 ID。
- 下游单据 ID。
- 转换数量。
- 关闭数量。
- 反写状态。

## 四、常见状态字段

不同系统字段名不一定相同，但状态含义通常包括：

| 状态 | 业务意义 | 排查意义 |
| --- | --- | --- |
| 暂存 | 草稿未提交 | 通常不参与后续正式业务 |
| 提交 | 已提交待审核 | 可能卡在审批 |
| 审核 | 正式生效 | 通常允许下推或入账 |
| 反审核 | 撤回生效状态 | 可能影响下游 |
| 关闭 | 不再继续执行 | 未关闭会影响未结报表 |
| 作废 | 业务无效 | 需要看是否仍影响账表 |
| 过账 | 凭证已入账 | 总账余额受影响 |

排查时不要只看一个状态。例如业务单据已审核，但凭证未过账，总账余额仍可能不体现。

## 五、金蝶常见命名线索

不同版本不完全一致，但常见命名有一些规律：

- `BD` 常用于基础资料。
- `GL` 常用于总账。
- `AR` 常用于应收。
- `AP` 常用于应付。
- `STK` 常用于库存。
- `SAL` 常用于销售。
- `PUR` 常用于采购。
- `PRD` 或 `SFC` 常用于生产或车间。
- `CB` 或成本相关前缀常用于成本核算。

这些只能作为找表线索，不能直接当作固定标准。正式 SQL 必须通过元数据、数据库字典、开发平台或实际表结构确认。

## 六、基础资料引用

业务单据中看到的客户名称、物料名称、科目名称，后台往往存的是内码。

```text
业务单据分录.物料内码
  -> 物料基础资料.内码
  -> 物料名称、规格、属性
```

查询报表时要注意：

- 名称字段可能来自多语言表。
- 编码和名称可能不在同一张表。
- 被禁用的基础资料仍可能被历史单据引用。
- 修改基础资料名称可能影响展示，不一定影响历史业务实质。

## 七、单据关联阅读

上下游关联是金蝶排查的重点。

### 常见关联问题

| 现象 | 可能原因 |
| --- | --- |
| 订单已下推但下游查不到 | 下游单据被删除、作废或权限不可见 |
| 入库数量不回写订单 | 关联关系断裂或转换规则异常 |
| 应收生成后不能反审核出库 | 下游存在正式单据 |
| 生产订单领料数量异常 | BOM、用料清单、超领规则或反写异常 |
| 未结报表数量不对 | 关闭状态或累计执行数量不正确 |

### 排查顺序

1. 查源单状态。
2. 查下游单据是否存在。
3. 查关联表或关联字段。
4. 查累计执行数量。
5. 查关闭状态。
6. 回前台单据联查验证。

## 八、账务结果阅读

业务单据影响财务通常经过：

```text
业务单据
  -> 应收应付或存货核算
  -> 凭证生成
  -> 凭证审核
  -> 凭证过账
  -> 总账报表
```

排查财务结果时确认：

- 源单是否满足生成凭证条件。
- 凭证模板是否匹配。
- 凭证是否生成。
- 凭证是否审核和过账。
- 科目、辅助核算、币别、金额是否正确。
- 是否存在手工凭证影响同一科目。

## 九、日志和操作记录

出现“谁改了数据”“为什么状态变了”时，要查日志。

常见日志：

- 创建记录。
- 修改记录。
- 审核记录。
- 反审核记录。
- 删除或作废记录。
- 接口调用日志。
- 后台任务日志。
- 异常堆栈日志。

日志排查要记录：

- 操作时间。
- 操作用户。
- 操作对象。
- 操作前后状态。
- 报错原文。

## 十、实际案例

### 案例：应收账龄和总账应收余额不一致

排查路径：

1. 确认账龄报表的组织、客户、期间、币别。
2. 查询应收单、收款单、核销状态。
3. 查询应收单是否全部生成凭证。
4. 查询凭证是否过账。
5. 查询总账应收科目是否存在手工凭证。
6. 按客户辅助核算汇总总账余额。
7. 输出差异分类。

常见差异：

- 应收模块有单据但未生成凭证。
- 凭证生成了但未过账。
- 总账有人手工录了应收科目。
- 查询账龄时包含未审核单据，总账不包含。
- 币别或汇率口径不同。

## 十一、数据模型笔记模板

```markdown
# 对象名称

## 适用产品和版本

## 前台菜单

## 业务用途

## 主表

## 分录表

## 关键字段

## 上下游关系

## 常用查询

## 验证方法

## 风险
```

## 十二、安全提醒

- 不要把生产库账号写进笔记。
- 不要记录客户敏感数据明细。
- 导出数据要脱敏。
- 修改类 SQL 必须审批、备份、测试。
- 对账结果要保留口径说明。
