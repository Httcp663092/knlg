财务数据排查的目标不是写一条 SQL 查出金额，而是解释“为什么前台报表、业务模块、总账和管理口径出现差异”。金蝶财务对账要围绕账簿、组织、期间、币别、凭证状态、业务来源和辅助核算来做。

## 一、财务取数前置口径

写 SQL 前必须确认：

| 口径 | 必问问题 |
| --- | --- |
| 组织 | 查单个核算组织还是集团汇总 |
| 账簿 | 主账簿、副账簿、管理账簿是否一致 |
| 期间 | 业务日期、记账日期、会计期间是否一致 |
| 币别 | 原币、本位币、综合本位币 |
| 状态 | 是否只含已审核、已过账 |
| 来源 | 手工凭证、业务生成凭证、期初 |
| 维度 | 客户、供应商、部门、项目、物料 |

如果这些口径没有写清，SQL 结果无法作为对账证据。

## 二、总账排查

### 总账数据链路

```text
凭证
  -> 凭证分录
  -> 科目
  -> 辅助核算
  -> 过账
  -> 科目余额表
  -> 财务报表
```

### 常见问题

| 现象 | 排查方向 |
| --- | --- |
| 科目余额不对 | 期初、凭证分录、过账状态、手工调整 |
| 辅助余额不等于科目余额 | 凭证辅助维度缺失或维度口径不同 |
| 报表取数不对 | 报表公式、科目范围、期间、币别 |
| 凭证无法过账 | 期间未打开、科目禁用、辅助核算缺失 |
| 凭证来源无法追溯 | 业务凭证关联丢失或手工凭证未备注 |

### 查询思路

```sql
-- 结构模板：查某期间、某组织、某科目的凭证分录
select
  凭证号,
  凭证日期,
  科目编码,
  科目名称,
  借方金额,
  贷方金额,
  辅助核算,
  凭证状态,
  过账状态
from 凭证分录视图或凭证分录表
where 组织 = '目标组织'
  and 会计期间 = '目标期间'
  and 科目编码 like '1122%';
```

验证方法：

- 汇总借贷发生额，与科目余额表比较。
- 抽样凭证回前台打开。
- 检查辅助核算是否完整。
- 检查是否存在未过账凭证。

## 三、应收对总账

### 对账链路

```text
销售出库或销售发票
  -> 应收单
  -> 收款单
  -> 核销
  -> 应收凭证
  -> 总账应收科目
```

### 对账步骤

1. 固定客户、组织、期间、币别。
2. 查询应收单余额。
3. 查询收款和核销。
4. 查询应收单对应凭证。
5. 汇总总账应收科目辅助余额。
6. 按客户输出差异。

### 差异分类

| 差异 | 判断方式 | 处理 |
| --- | --- | --- |
| 应收单未生成凭证 | 应收有余额，总账无凭证 | 生成或补生成凭证 |
| 凭证未过账 | 凭证存在但总账余额不体现 | 审核过账 |
| 手工凭证影响应收 | 总账有余额，应收模块无来源 | 要求财务说明并规范 |
| 核销未完成 | 应收余额和收款余额未抵消 | 做收款核销 |
| 查询期间不同 | 应收按业务日期，总账按记账日期 | 统一期间口径 |

### SQL 模板

```sql
-- 应收余额查询结构模板
select
  客户编码,
  客户名称,
  币别,
  sum(应收金额) as 应收金额,
  sum(已核销金额) as 已核销金额,
  sum(未核销金额) as 未核销金额
from 应收明细表或视图
where 组织 = '目标组织'
  and 业务日期 <= '截止日期'
group by 客户编码, 客户名称, 币别;
```

```sql
-- 总账应收辅助余额结构模板
select
  客户编码,
  客户名称,
  科目编码,
  sum(借方金额 - 贷方金额) as 余额
from 总账凭证明细表或余额表
where 科目编码 like '1122%'
  and 会计期间 <= '截止期间'
  and 过账状态 = '已过账'
group by 客户编码, 客户名称, 科目编码;
```

## 四、应付对总账

### 对账链路

```text
采购入库或采购发票
  -> 应付单
  -> 付款单
  -> 核销
  -> 应付凭证
  -> 总账应付科目
```

### 常见差异

- 采购入库暂估未生成凭证。
- 到票后暂估冲回未完成。
- 应付单未审核。
- 付款单未核销。
- 总账有手工应付凭证。
- 供应商辅助核算缺失。

### 对账输出格式

| 供应商 | 应付模块余额 | 总账辅助余额 | 差异 | 原因 | 处理建议 |
| --- | --- | --- | --- | --- | --- |
| A 供应商 | 100,000 | 100,000 | 0 | 一致 | 无 |
| B 供应商 | 80,000 | 75,000 | 5,000 | 凭证未过账 | 过账后复核 |

## 五、现金银行对账

### 对账对象

- 银行日记账。
- 收款单。
- 付款单。
- 银行流水。
- 总账银行科目。

### 排查方向

| 现象 | 可能原因 |
| --- | --- |
| 银行日记账和总账不一致 | 收付款未生成凭证、凭证未过账 |
| 银行流水和收付款不一致 | 未认领流水、重复认领、日期差异 |
| 现金科目余额异常 | 手工凭证或出纳单据口径不同 |
| 未达账项过多 | 银企对账未及时处理 |

## 六、固定资产对账

### 对账链路

```text
资产卡片
  -> 折旧计提
  -> 资产变动或清理
  -> 资产凭证
  -> 总账固定资产和累计折旧科目
```

### 检查点

- 卡片原值汇总等于固定资产科目余额。
- 卡片累计折旧汇总等于累计折旧科目余额。
- 本期折旧额等于费用科目发生额。
- 已清理资产不应继续折旧。
- 资产部门和费用归属正确。

## 七、存货和总账对账

存货对账跨供应链和财务，不能只看总账。

### 对账链路

```text
库存单据
  -> 库存数量
  -> 存货核算金额
  -> 存货凭证
  -> 总账存货科目
```

### 差异分类

| 差异 | 可能原因 |
| --- | --- |
| 库存有数量无金额 | 期初单价缺失或核算未完成 |
| 存货核算和总账不一致 | 凭证未生成或未过账 |
| 总账有存货手工凭证 | 财务手工调整未同步业务 |
| 暂估差异 | 采购入库和发票跨期 |
| 负库存影响成本 | 出库早于入库或成本重算异常 |

## 八、财务报表取数排查

### 资产负债表

检查：

- 报表期间是否正确。
- 取数账簿是否正确。
- 科目范围是否完整。
- 重分类公式是否正确。
- 未过账凭证是否包含。

### 利润表

检查：

- 损益科目发生额。
- 结转损益是否完成。
- 管理费用、销售费用、财务费用科目是否完整。
- 多组织汇总是否抵消内部交易。

## 九、实际案例

### 案例：客户应收账龄比总账多 30,000

现象：

- 应收账龄显示客户 A 未收 130,000。
- 总账 1122 客户 A 辅助余额为 100,000。
- 差异 30,000。

排查：

1. 查询客户 A 应收明细，发现一张 30,000 应收单已审核。
2. 查询凭证生成记录，发现未生成凭证。
3. 检查凭证模板，发现该应收单单据类型未匹配凭证模板。
4. 补充凭证模板后生成凭证。
5. 审核过账后总账余额变为 130,000。

结论：

- 差异不是应收金额错，而是业务凭证未生成。

## 十、财务 SQL 排查清单

- [ ] 组织、账簿、期间已确认。
- [ ] 查询口径是否只包含已审核或已过账已确认。
- [ ] 业务模块余额已导出。
- [ ] 总账科目余额已导出。
- [ ] 辅助核算维度一致。
- [ ] 是否存在手工凭证。
- [ ] 是否存在未生成凭证业务单据。
- [ ] 是否存在未审核或未过账凭证。
- [ ] 是否存在期初差异。
- [ ] 差异表已按原因分类。
