这篇用于快速判断金蝶项目日常问题的方向。它不是替代详细排查，而是帮助你在 5-10 分钟内判断问题大概率属于权限、配置、数据、流程、接口、二开、环境还是业务口径。

## 一、通用判断顺序

遇到任何问题，先问：

1. 是单个用户还是所有用户。
2. 是单张单据还是同类单据。
3. 是生产环境还是测试环境也一样。
4. 是今天突然出现还是上线后一直存在。
5. 最近是否改过权限、参数、审批流、插件、接口、补丁或基础资料。
6. 是否有完整报错原文和单据编号。

## 二、快速定位表

| 现象 | 优先判断 | 常见方向 |
| --- | --- | --- |
| 看不到菜单 | 用户、角色、组织、许可 | 权限 |
| 能看菜单但不能保存 | 按钮权限、必填字段、插件校验 | 权限或配置 |
| 单据不能审核 | 审批流、必填、下游状态、插件 | 流程或二开 |
| 下推不了下游单据 | 单据状态、转换规则、未执行数量 | 单据转换 |
| 报表数字不对 | 查询条件、状态、期间、组织、口径 | 数据口径 |
| 接口失败 | 认证、主数据、字段、业务规则 | 集成 |
| 库存不对 | 单据审核、批号库位、盘点、核算 | 库存 |
| 成本不对 | 入库成本、出库核算、负库存、费用分配 | 存货或制造成本 |
| 凭证不对 | 凭证模板、科目映射、辅助核算 | 财务 |
| 系统突然变慢 | 发布、任务、数据库、接口、资源 | 环境或性能 |

## 三、权限类问题

### 1. 用户看不到菜单

快速判断：

- 新用户通常是角色未分配或组织范围未给。
- 老用户突然看不到，先查最近权限变更或角色调整。

排查顺序：

1. 用户是否启用。
2. 是否绑定人员。
3. 是否分配角色。
4. 角色是否有菜单权限。
5. 是否有组织范围。
6. 是否有模块许可。
7. 用同岗位用户对比。

处理建议：

- 按岗位模板授权。
- 不要直接给管理员权限。
- 记录权限变更原因。

### 2. 能打开单据但不能新增、保存或审核

快速判断：

- 能进菜单不代表有按钮权限。
- 保存失败不一定是权限，也可能是字段必填或插件校验。

排查顺序：

1. 查新增、保存、提交、审核按钮权限。
2. 查单据状态。
3. 查必填字段。
4. 查审批任务是否到当前用户。
5. 查插件报错或校验规则。
6. 查组织和数据范围。

## 四、供应链类问题

### 3. 采购订单为什么不能下推入库

常见原因：

- 采购订单未审核。
- 订单已关闭。
- 订单未入库数量为 0。
- 单据转换规则不适用。
- 当前用户无下推或入库权限。
- 供应商、物料、仓库状态异常。

排查顺序：

1. 看订单状态。
2. 看分录未入库数量。
3. 看是否已被下游单据占用。
4. 看转换路线。
5. 看用户权限。
6. 用另一张正常订单对比。

### 4. 销售出库后库存没有减少

常见原因：

- 出库单未审核。
- 出库仓库不是查询仓库。
- 批号、库位、库存状态查询条件不一致。
- 查询的是历史库存报表，不是即时库存。
- 出库单被反审核或作废。

处理建议：

- 先以前台即时库存和库存明细账交叉验证。
- 再查销售出库单状态和分录仓库。

### 5. 库存账实不符怎么办

排查顺序：

1. 固定物料、仓库、批号、库位。
2. 查即时库存。
3. 查库存明细账。
4. 查未审核出入库单。
5. 查盘点单和调整单。
6. 查是否有线下实物移动未入系统。

处理建议：

- 不能直接改库存余额。
- 通过盘点、调整、补录单据形成可追溯处理。

## 五、生产制造类问题

### 6. MRP 计划结果不符合预期

常见原因：

- 需求来源没纳入。
- BOM 未审核或版本不生效。
- 库存可用量口径不一致。
- 安全库存、提前期、批量规则设置不合理。
- 已下达订单没有正确抵扣需求。

排查顺序：

1. 确认计划范围。
2. 确认需求来源。
3. 检查 BOM 版本。
4. 检查物料计划属性。
5. 检查库存和在途。
6. 查看 MRP 运算日志或结果明细。

### 7. 生产领料数量不对

常见原因：

- BOM 用量或损耗设置错误。
- 生产订单数量变更。
- 替代料启用。
- 超领或补料规则。
- 已领料数量没有正确反写。

处理建议：

- 从生产订单用料清单开始查，不要直接查库存。
- 对比 BOM 标准用量、订单用量、已领数量、可领数量。

### 8. 完工入库后成本异常

常见原因：

- 材料未完整领用。
- 入库或出库成本未核算。
- 人工和制造费用未分配。
- 完工数量或在制品数量错误。
- 负库存影响出库成本。

处理建议：

- 按“领料 -> 完工 -> 存货核算 -> 成本计算 -> 凭证”顺序排查。

## 六、财务类问题

### 9. 业务单据生成不了凭证

优先看：

- 单据是否审核。
- 期间是否开放。
- 凭证模板是否命中。
- 科目和辅助核算是否能取到。
- 金额是否为 0。
- 生成日志是否有明确报错。

详细处理见 [财务高频问题](finance-faq.md)。

### 10. 应收应付和总账不一致

常见原因：

- 单据未生成凭证。
- 凭证未过账。
- 手工凭证影响往来科目。
- 查询条件的期间、币别、组织不同。
- 核销状态不同。

处理建议：

- 按客户或供应商输出差异表。
- 分成未生成凭证、未过账、手工凭证、期初差异、查询口径五类。

## 七、数据和报表类问题

### 11. 报表数字和用户认为的不一致

快速判断：

- 先不要说报表错，先问口径。

必须确认：

- 是业务日期还是记账期间。
- 是已审核还是全部单据。
- 是含税还是不含税。
- 是原币还是本位币。
- 是销售订单、销售出库、应收还是总账收入。

处理建议：

- 给报表标题加口径说明。
- 指标定义写到报表需求文档。

### 12. SQL 查出来和前台不一致

常见原因：

- SQL 过滤条件少了组织、期间、状态。
- 前台报表包含权限或业务口径。
- 后台查的是明细，前台看的是汇总或余额。
- 多语言名称、基础资料引用或历史版本影响展示。

处理建议：

- 用同一张单据编号抽样验证。
- SQL 结果必须回前台单据和标准报表核对。

## 八、接口类问题

### 13. 外部系统推单失败

常见原因：

- 接口账号认证失败。
- 接口账号无组织或单据权限。
- 客户、供应商、物料、仓库不存在。
- 必填字段为空。
- 金额、税额、单位、币别格式错误。
- 业务规则不允许，例如库存不足或期间关闭。

排查顺序：

1. 查接口日志。
2. 查请求报文。
3. 查响应报错。
4. 查主数据映射。
5. 查接口账号权限。
6. 查金蝶前台是否能手工录入同样数据。

### 14. 接口重复生成单据

常见原因：

- 没有幂等键。
- 外部系统超时后重试。
- 金蝶已创建成功但响应失败。
- 接口没有先查后补偿。

处理建议：

- 用外部单号加业务类型加组织做幂等。
- 重试前先查询金蝶是否已存在。
- 建立重复数据处理流程。

## 九、开发和配置类问题

### 15. 插件发布后没有生效

常见原因：

- 发布到错误环境。
- 插件注册对象不对。
- 事件时机不对。
- 缓存未刷新。
- 测试单据不满足触发条件。

处理建议：

- 在日志中打出插件版本和触发对象。
- 用最小测试单据验证。
- 检查发布包和环境版本。

### 16. 新增字段后报表或接口取不到

原因：

- 字段只加在表单，没有加入列表或数据源。
- 接口字段映射未更新。
- 报表数据集未更新。
- 权限或字段可见性限制。

处理建议：

- 新增字段要同步检查：表单、列表、接口、报表、权限、凭证模板。

## 十、环境和性能类问题

### 17. 系统突然变慢

快速判断：

- 是否所有用户都慢。
- 是否某个菜单慢。
- 是否刚发布补丁或二开。
- 是否有大报表、批量导入、成本计算、月结任务。

排查顺序：

1. 查服务器 CPU、内存、磁盘。
2. 查数据库连接和慢查询。
3. 查后台任务。
4. 查接口失败和重试积压。
5. 查最近发布。
6. 用测试环境对比。

### 18. 发布后出现异常

处理：

1. 先确认影响范围。
2. 查看发布内容和变更清单。
3. 用发布验证用例复测。
4. 查日志。
5. 判断是否回滚或热修复。
6. 记录发布问题和修复过程。

## 十一、问题升级标准

满足以下条件要升级处理：

- 影响生产大面积使用。
- 阻塞月结、付款、发货、生产。
- 涉及数据错误或财务报表。
- 涉及重复付款、重复出库、重复入库。
- 涉及安全、权限、敏感数据泄露。
- 临时方案超过一天仍未根因修复。

## 十二、沉淀规则

- 高频短问题留在本页。
- 财务专项问题补到财务高频问题。
- 完整处理过程补到运维与问题库。
- 可复用 SQL 补到数据模型与 SQL。
- 涉及配置或实施风险的，反补到实施交付模块。
