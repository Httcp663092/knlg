车间执行用于把生产订单落到工序、班组、人员、设备、报工和完工。生产订单只是告诉工厂“做什么、做多少、什么时候做”，车间执行要回答“谁做、在哪做、做到哪一步、用了多少工时、合格多少、不良多少”。

## 一、适用范围

- 有工序管理要求的制造企业。
- 需要按班组、产线、工序统计产量。
- 需要采集人工工时、机器工时。
- 需要报工后触发质检、完工入库或成本计算。
- 需要和 MES、条码、设备采集集成。

## 二、核心对象

| 对象 | 说明 |
| --- | --- |
| 生产订单 | 生产任务来源 |
| 工艺路线 | 产品加工步骤 |
| 工序 | 具体加工环节 |
| 工作中心 | 产线、班组、设备或加工单元 |
| 派工单 | 把任务分派给班组或人员 |
| 报工单 | 记录工序完成数量和工时 |
| 完工入库 | 产成品或半成品入库 |
| 不良记录 | 报工或质检发现的不合格 |

## 三、业务流程

```text
BOM 和工艺路线
  -> 生产订单
  -> 工序计划
  -> 派工
  -> 领料
  -> 工序报工
  -> 工序检验
  -> 返工或让步
  -> 完工入库
  -> 成本计算
```

不同企业可以简化，例如不启用工序，只做生产订单领料和完工；但如果要精细统计产能、工时和质量，就需要工序报工。

## 四、上线前配置

### 工艺路线

工艺路线要明确：

- 工序编码和名称。
- 工序顺序。
- 工作中心。
- 标准工时。
- 是否关键工序。
- 是否需要质检。
- 是否允许跳工序。
- 是否允许超报工。

### 工作中心

工作中心可以按：

- 产线。
- 班组。
- 设备。
- 工段。
- 外协加工点。

设计时要考虑后续统计口径。如果管理层要看“产线效率”，工作中心就不能只按大车间维护。

### 报工规则

| 规则 | 需要确认 |
| --- | --- |
| 报工时点 | 每道工序、关键工序、完工时 |
| 报工数量 | 合格数、不良数、返工数 |
| 工时采集 | 标准工时、实际人工工时、机器工时 |
| 领料关系 | 先领料后报工，还是倒冲领料 |
| 质检触发 | 报工后自动生成检验任务 |
| 完工入库 | 最后工序报工后是否自动入库 |

## 五、操作步骤

### 派工

1. 根据生产订单生成工序任务。
2. 按工作中心、班组、日期筛选待派工任务。
3. 分配人员、设备和计划开始结束时间。
4. 下发派工单。
5. 车间确认接收任务。

检查点：

- 生产订单是否审核或下达。
- 工艺路线是否生效。
- 工作中心是否可用。
- 计划数量和派工数量是否一致。

### 工序报工

1. 选择生产订单或派工单。
2. 选择工序。
3. 录入合格数量。
4. 录入不良数量和不良原因。
5. 录入人工工时和机器工时。
6. 上传或关联质检记录。
7. 提交并审核报工。

检查点：

- 报工数量不能超过可报数量，除非允许超报。
- 前工序未完成时，后工序是否允许报工。
- 不良数量是否触发质量处理。
- 报工工时是否用于成本分配。

### 完工入库

最后工序完成后，可以：

- 手工生成完工入库。
- 根据报工自动生成完工入库。
- 质检合格后再入库。

检查点：

- 完工数量和最后工序合格数量一致。
- 入库仓库和库存状态正确。
- 批号、序列号、生产日期正确。
- 完工入库影响存货核算和制造成本。

## 六、条码报工

条码报工常见方式：

```text
扫生产订单
  -> 扫工序或工位
  -> 扫员工或设备
  -> 录入数量和工时
  -> 提交报工
```

好处：

- 减少手工找单。
- 降低工序错报。
- 方便现场实时采集产量。
- 可和看板、MES、设备联网结合。

## 七、和质量管理的关系

工序报工可能触发：

- 首检。
- 巡检。
- 完工检验。
- 不良品处理。
- 返工返修。

如果质量数据不进系统，报工只有产量，没有质量口径，后续良率和成本分析会失真。

## 八、和成本的关系

报工数据可用于：

- 人工费用分配。
- 制造费用分配。
- 产量统计。
- 工时效率分析。
- 不良成本分析。

成本会关注：

- 合格数量。
- 完工数量。
- 在制数量。
- 工时。
- 报废数量。
- 返工耗用。

## 九、常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 生产订单看不到工序 | 工艺路线未维护或未生效 | 检查物料工艺路线和版本 |
| 报工数量不能保存 | 超过可报数量或前工序未完成 | 检查报工规则 |
| 报工后没有质检 | 工序未设置检验触发 | 检查质检规则 |
| 完工入库数量不对 | 最后工序合格数、入库规则不一致 | 对比报工和入库 |
| 成本工时不准 | 报工工时漏填或使用标准工时 | 确认工时采集口径 |

## 十、实际案例

### 案例：关键工序报工管理

背景：

- 企业有下料、焊接、喷涂、装配四道工序。
- 只想对焊接和装配做详细报工。

方案：

1. 工艺路线维护四道工序。
2. 焊接和装配设置为关键工序。
3. 生产订单下达后生成关键工序任务。
4. 班组扫码报工。
5. 焊接报工触发巡检。
6. 装配合格后生成完工入库。
7. 工时用于人工费用分配。

效果：

- 不增加所有工序操作负担。
- 关键产量和质量可追踪。
- 月末成本有工时分配依据。

- [ ] 工艺路线已维护并审核。
- [ ] 工作中心和班组已维护。
- [ ] 报工规则已确认。
- [ ] 前后工序控制已测试。
- [ ] 报工触发质检已测试。
- [ ] 完工入库规则已测试。
- [ ] 报工工时和成本分配口径已确认。
- [ ] 条码报工设备和权限已测试。
