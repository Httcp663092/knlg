计划管理的目标，是把销售订单、预测、安全库存和生产需求转成可执行的采购、生产、委外和调拨建议。金蝶计划管理官方资料强调多组织计划、按需求优先级占用资源、物料预留网络和计划员工作台，这些都是 MRP 落地时要理解的核心。

## 一、MRP 是什么

MRP 可以理解为“根据需求和库存，算出未来应该买什么、做什么、委外什么、调拨什么”。

```text
需求
  - 库存
  - 在途供应
  + 安全库存
  -> 净需求
  -> 计划订单
  -> 采购/生产/委外/调拨
```

## 二、MRP 输入

| 输入 | 说明 |
| --- | --- |
| 销售订单 | 客户真实需求 |
| 预测单 | 备货或销售预测 |
| 安全库存 | 低于安全线补货 |
| 生产订单下级需求 | 半成品和材料需求 |
| 库存余额 | 当前可用库存 |
| 采购在途 | 已下采购未入库 |
| 生产在制 | 已下生产未完工 |
| 委外在途 | 委外加工未入库 |
| BOM | 需求展开 |
| 提前期 | 采购、生产、委外时间 |
| 批量规则 | 最小批量、固定批量、批量倍数 |

## 三、MRP 输出

| 输出 | 适用 |
| --- | --- |
| 采购申请 | 外购物料 |
| 生产订单或计划订单 | 自制物料 |
| 委外申请或委外订单 | 委外物料 |
| 调拨建议 | 其他组织或仓库有货 |
| 例外信息 | 缺基础资料、缺库存、日期异常 |

## 四、计划方案

计划方案决定 MRP 怎么算。

关键参数：

| 参数 | 说明 |
| --- | --- |
| 计划范围 | 哪些组织、仓库、物料参与 |
| 需求来源 | 销售订单、预测单、生产需求 |
| 供应来源 | 库存、采购在途、生产在制 |
| 运算日期 | 计划开始日期 |
| 计划周期 | 算多远 |
| 需求优先级 | 急单优先还是按日期 |
| 是否自动投放 | 是否直接生成生产订单或采购申请 |
| 默认部门 | 投放单据默认部门 |
| 默认单据类型 | 投放生产、采购、委外单据类型 |

自动投放要谨慎。官方计划方案资料里提到可在 MRP 运算后直接把满足条件的计划订单投放成生产订单、委外申请单和采购申请单。项目上建议先人工审核计划结果，稳定后再考虑自动投放。

## 五、MRP 运行前检查

### 需求检查

- 销售订单是否审核。
- 预测单是否审核。
- 需求日期是否合理。
- 是否重复录入预测和销售订单。
- 是否有插单需要优先。

### 供应检查

- 即时库存是否准确。
- 采购订单在途是否准确。
- 生产订单在制是否准确。
- 委外订单在途是否准确。
- 调拨在途是否准确。

### 工程数据检查

- BOM 是否审核和生效。
- 子项物料是否参与 MRP。
- 物料计划属性是否正确。
- 提前期是否维护。
- 批量规则是否维护。
- 替代料是否维护。

## 六、MRP 运行步骤

1. 确认计划方案。
2. 确认计划组织和范围。
3. 清理异常需求。
4. 检查库存和在途。
5. 执行 MRP 运算。
6. 查看运算日志和例外信息。
7. 查看计划订单。
8. 调整计划订单数量、日期、供应方式。
9. 审核计划订单。
10. 投放为采购申请、生产订单、委外申请或调拨。

## 七、计划订单处理

计划订单是 MRP 输出的建议，不等于正式业务。

计划员要检查：

- 建议数量是否合理。
- 建议开工/到货日期是否可执行。
- 供应方式是否正确。
- 是否已有库存替代。
- 是否可合并订单。
- 是否要拆分订单。
- 是否要改供应商或生产部门。

## 八、需求优先级和预留

金蝶计划管理强调按需求优先级占用物料供给资源，并通过物料预留网络追踪供需关系。

业务意义：

- 急单可以优先占用库存或供应。
- 计划员能看到某个供应被哪个需求占用。
- 插单时可以重新分配资源。
- 预留关系能帮助追踪缺料原因。

## 九、多组织计划

集团企业常见多组织计划：

| 模式 | 说明 |
| --- | --- |
| 集中计划 | 集团统一看供需，统一安排 |
| 协同计划 | 各组织边界清楚，只传递需求 |
| 调拨计划 | 一个组织需求由其他组织供货 |
| 跨组织生产 | A 公司销售，B 工厂生产 |

多组织计划检查：

- 需求组织、供应组织是否明确。
- 组织间是否允许调拨。
- 跨组织是否需要结算。
- 库存是否能跨组织共享。
- 凭证和内部交易如何处理。

## 十、MRP 常见异常

### 需求重复

原因：

- 预测单和销售订单重复。
- 销售订单已经转生产，但仍参与计划。
- 手工计划未关闭。

处理：

- 确认需求抵消规则。
- 清理重复需求。
- 调整计划方案。

### 跑不出计划订单

排查：

1. 物料是否参与计划。
2. 需求单据是否审核。
3. BOM 是否有效。
4. 计划范围是否包含该物料和组织。
5. 库存和在途是否已经满足需求。

### 建议日期不合理

原因：

- 提前期错误。
- 工厂日历错误。
- 需求日期错误。
- 采购提前期未维护。
- 生产工期未维护。

### 计划数量异常

原因：

- 单位换算错误。
- BOM 用量错误。
- 批量规则错误。
- 安全库存过高。
- 在途供应没有纳入。

## 十一、计划到执行的衔接

```text
计划订单
  -> 采购申请
  -> 采购订单
  -> 采购入库
```

```text
计划订单
  -> 生产订单
  -> 领料
  -> 完工入库
```

```text
计划订单
  -> 委外申请/委外订单
  -> 委外发料
  -> 委外入库
```

投放后要跟踪计划执行：

- 采购是否按期到货。
- 生产是否按期开工完工。
- 委外是否按期交付。
- 缺料是否解除。
- 计划是否需要重算。

## 十二、计划管理验收清单

- [ ] 计划方案已配置。
- [ ] 物料计划属性维护完整。
- [ ] BOM 能正常展开。
- [ ] 库存和在途能参与运算。
- [ ] 销售订单和预测单能形成需求。
- [ ] MRP 能输出采购、生产、委外建议。
- [ ] 计划订单能审核和投放。
- [ ] 急单优先级能体现。
- [ ] 多组织供需规则清楚。
- [ ] 计划结果能被采购、生产、委外执行。
