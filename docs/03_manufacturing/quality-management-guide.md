质量管理用于控制来料、生产过程、完工、发货和售后环节的质量风险。金蝶项目中的质量管理不能只写“检验合格或不合格”，还要和采购、库存、生产、销售、供应商绩效和成本核算联动。

## 一、质量管理范围

| 环节 | 质量动作 |
| --- | --- |
| 采购来料 | 收料检验、退货、让步接收 |
| 生产过程 | 首检、巡检、工序检验 |
| 完工入库 | 完工检验、入库判定 |
| 库存 | 库存状态、复检、冻结 |
| 销售发货 | 发货检验、客户退货检验 |
| 售后 | 客诉、返修、质量追溯 |

## 二、核心对象

| 对象 | 说明 |
| --- | --- |
| 检验项目 | 尺寸、外观、性能、重量等 |
| 检验标准 | 合格范围、抽样规则、判定规则 |
| 检验单 | 实际检验记录 |
| 不合格品处理 | 退货、返工、让步、报废 |
| 质量状态 | 合格、待检、不合格、冻结 |
| 质量追溯 | 批号、序列号、供应商、生产订单 |

## 三、来料检验

```text
采购收料
  -> 生成来料检验任务
  -> 质检录入检验结果
  -> 合格入库 / 不合格处理 / 让步接收
```

检查点：

- 哪些物料需要检验。
- 是否按供应商、物料、批号触发检验。
- 抽样规则是什么。
- 检验不合格后库存状态如何变化。
- 让步接收是否需要审批。

## 四、过程检验

过程检验适合生产过程中发现问题。

常见类型：

- 首件检验。
- 巡回检验。
- 工序完工检验。
- 关键工序检验。

流程：

```text
工序报工
  -> 触发检验
  -> 质检判定
  -> 合格流转下工序
  -> 不合格进入返工或报废
```

## 五、完工检验

完工检验用于产成品或半成品入库前质量确认。

```text
生产完工报工
  -> 完工检验
  -> 合格入库
  -> 不合格处理
```

检查：

- 是否允许未检验直接入库。
- 检验合格数量是否等于入库数量。
- 不合格数量是否进入不良品仓。
- 返工后是否重新检验。

## 六、不合格品处理

| 处理方式 | 说明 |
| --- | --- |
| 退货 | 采购来料不合格退给供应商 |
| 返工 | 生产或供应商重新加工 |
| 返修 | 修复后继续使用 |
| 让步接收 | 不完全合格但允许使用 |
| 降级使用 | 改为低等级用途 |
| 报废 | 无法使用，进入报废流程 |

每种处理都要明确：

- 审批人。
- 库存状态。
- 是否影响成本。
- 是否影响供应商评分。
- 是否需要质量报告。

## 七、质量和库存

质量结果会影响库存状态：

```text
待检库存
  -> 合格库存
  -> 不合格库存
  -> 冻结库存
```

常见控制：

- 待检物料不能领用。
- 不合格物料不能销售出库。
- 冻结库存需要解冻审批。
- 让步接收需要留原因和审批记录。

## 八、质量追溯

质量追溯通常按：

- 批号。
- 序列号。
- 供应商。
- 采购订单。
- 生产订单。
- 工序。
- 客户发货单。

追溯问题要能回答：

- 这个批次来自哪个供应商。
- 用在哪些生产订单。
- 发给了哪些客户。
- 同批次还有多少库存。
- 是否需要召回或冻结。

## 九、质量成本

质量问题可能影响成本：

- 报废材料成本。
- 返工人工和制造费用。
- 供应商扣款。
- 客户退货损失。
- 质量索赔。

系统要保留质量问题和成本处理之间的追溯关系。

## 十、常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 收料后没生成检验 | 检验规则未命中 | 检查物料、供应商、单据类型 |
| 待检库存被领用 | 库存状态控制不足 | 收紧待检状态可用性 |
| 让步接收无审批 | 流程缺失 | 设置让步审批 |
| 不合格无法追供应商 | 批号或采购来源缺失 | 强制批号和源单关联 |
| 质量报表口径混乱 | 检验类型和不良原因不统一 | 维护统一质量分类 |

## 十一、实际案例

### 案例：来料不合格让步接收

背景：

- 供应商送来 1000 件原料。
- 抽检发现 50 件外观轻微瑕疵，但生产可以使用。

处理：

1. 收料后生成来料检验单。
2. 质检记录不合格数量 50。
3. 发起让步接收审批。
4. 审批通过后 1000 件入合格或指定库存状态。
5. 系统记录让步原因。
6. 供应商绩效扣分或价格扣减。

检查：

- 让步记录能追溯到供应商和批号。
- 后续如客户投诉，能找到同批次去向。

- [ ] 库存状态控制已测试。

- [ ] 质量报表口径已确认。
