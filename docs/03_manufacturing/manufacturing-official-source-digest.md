## 二、官方资料能补充什么

制造模块不是只有生产订单、领料、完工。官方资料把制造能力拆成更完整的链条：

```text
产品数据
  -> 计划管理
  -> 排产
  -> 生产执行
  -> 车间管理
  -> 质量管理
  -> 委外
  -> 成本
  -> 现场文档和设备数据
```

项目现场要重点看四个问题：

- 计划怎么生成和调整。
- 现场怎么排产、报工、追踪进度。
- 质量怎么拦截不合格。
- 技术文档和工艺要求怎么下达到现场。

## 三、计划管理：不只是跑 MRP

官方计划管理强调多组织计划、物料预留、计划员平台和计划模拟。落到项目里，计划管理要解决：

- 哪些需求进入计划。
- 哪些供给可以抵扣需求。
- 急单如何抢占资源。
- 插单后如何重算。
- 计划订单如何投放采购、生产或委外。

### 关键能力理解

| 能力 | 项目解释 |
| --- | --- |
| 集中计划 | 集团或计划中心统一平衡需求和供给 |
| 协同计划 | 多组织各自计划但相互协同 |
| 物料预留 | 把某个供给锁定给某个需求，减少计划结果漂移 |
| 净改变计划 | 需求或供给变化后，只重算受影响部分 |
| 计划员平台 | 计划员集中处理建议订单、异常和投放 |

### 项目配置重点

- 计划组织和计划范围。
- 物料计划属性。
- 提前期、最小批量、固定批量。
- 安全库存。
- 需求来源优先级。
- 供应来源。
- 预留策略。
- 计划订单投放规则。

### 排查 MRP 结果

如果计划结果不符合预期，按下面顺序查：

1. 需求是否进入计划。
2. 需求日期和数量是否正确。
3. 可用库存是否被纳入。
4. 在途采购、在制生产是否被纳入。
5. BOM 是否有效。
6. 提前期和批量规则是否正确。
7. 是否存在预留。
8. 是否有急单优先级影响分配。

## 四、生产订单排产：把订单落到产线和日期

官方生产订单排产资料强调产线日产量、产线日排产、按计划开工日期或指定开工日期顺排、锁定和顺延。项目里可理解为：

```text
生产订单
  -> 选择车间和产线
  -> 根据产线能力排日期
  -> 锁定关键订单
  -> 插单时顺延普通订单
```

### 排产前提

- 生产订单已审核或下达。
- 产线日产量已维护。
- 生产车间和产线清晰。
- 计划开工日期和完工日期可靠。
- 插单和锁定规则已确认。

### 排产策略

| 策略 | 适用场景 |
| --- | --- |
| 按计划开工日期顺排 | 计划稳定，按订单计划执行 |
| 按指定开工日期顺排 | 现场需要提前或延后生产 |
| 锁定已排产订单 | 关键客户或关键订单不能被插单挤掉 |
| 后续订单顺延 | 插单后自动调整后面订单 |

### 现场管理建议

- 关键订单排产后锁定。
- 插单必须记录原因。
- 排产变化要同步给采购、仓库和销售。
- 排产结果要能反馈生产进度。

## 五、质量管理：从来料到发货的质量闭环

官方质量管理覆盖来料检验、产品检验、库存检验、发货检验、退货检验、工序检验、巡检、退料检验和质量报表。项目中要把质量管理设计成“拦截机制”，而不是事后登记。

### 质量场景地图

| 场景 | 触发点 | 系统结果 |
| --- | --- | --- |
| 来料检验 | 收料或采购入库前 | 合格入库、不合格退货、让步接收 |
| 工序检验 | 工序报工后 | 合格流转、不合格返工 |
| 完工检验 | 完工入库前 | 合格入库、不合格处理 |
| 库存复检 | 库存到期或质量变更 | 冻结、解冻、报废 |
| 发货检验 | 销售出库前 | 合格发货、不合格拦截 |
| 退货检验 | 客户退货后 | 可用、返修、报废 |

### 质量基础资料

- 检验项目。
- 检验标准。
- 抽样方案。
- 缺陷类型。
- 不良原因。
- 处理方式。
- 库存状态。

### 质量和库存的衔接

质量结果必须影响库存状态：

```text
待检
  -> 合格
  -> 不合格
  -> 冻结
  -> 报废
```

如果质量只做记录，不控制库存状态，仓库仍可能把待检或不合格物料发出去。

## 六、技术文档：现场作业不应只靠口头经验

官方技术文档资料强调在工程数据、工艺路线、智慧车间 MES 中维护技术文件，并支持现场智能终端读取。项目里应理解为：

- 工艺文件要和物料、BOM、工艺路线、生产订单关联。
- 现场人员在报工或作业时能看到最新版本文件。
- PLM、ERP、MES 可以通过接口传递 PDF 等文档。

### 技术文档适用范围

常见适用范围：

- 通用。
- 物料。
- 物料 + 作业。
- 工艺路线。
- 物料 + BOM 版本。
- 生产订单。

现场读取时可以按更具体的范围优先匹配，例如先找工艺路线、再找物料和作业、再找物料、最后找通用文档。

### 项目价值

- 减少现场使用过期图纸。
- 把临时工艺调整留痕。
- 支撑 PLM 到 MES 的文档传递。
- 审计质量问题时可以追溯当时执行的技术文件。

## 七、制造模块实施清单

- [ ] 是否启用计划管理、生产管理、车间、质量、委外、成本。
- [ ] 物料计划属性是否完整。
- [ ] BOM 和工艺路线是否有版本、生效日期。
- [ ] 计划需求来源和优先级是否确认。
- [ ] 产线日产量和排产策略是否确认。
- [ ] 工序报工和质检触发点是否确认。
- [ ] 不合格处理和库存状态是否联动。
- [ ] 技术文档是否能按物料、工艺或生产订单查看。
- [ ] 生产数据是否影响成本和总账。
