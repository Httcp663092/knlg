设备管理用于维护生产设备台账、保养计划、维修工单、备件消耗和设备停机记录。设备数据如果不进入系统，生产效率、停机损失、维修成本和产能分析就很难落地。

## 一、适用场景

- 关键设备影响生产交付。
- 需要定期保养和点检。
- 设备故障造成停线。
- 维修备件需要库存管理。
- 需要统计设备稼动率、停机时间、维修成本。
- 设备数据需要和生产订单、工序、工作中心关联。

## 二、核心对象

| 对象 | 说明 |
| --- | --- |
| 设备台账 | 设备编码、名称、型号、位置、状态 |
| 设备分类 | 产线设备、检测设备、动力设备等 |
| 点检标准 | 每日或每班检查项目 |
| 保养计划 | 定期保养、按运行小时保养 |
| 维修工单 | 故障维修或计划维修 |
| 备件 | 维修消耗物料 |
| 停机记录 | 故障时间、恢复时间、原因 |

## 三、设备台账

设备台账字段建议：

- 设备编码。
- 设备名称。
- 型号规格。
- 所属组织。
- 使用部门。
- 所在产线或工作中心。
- 安装位置。
- 供应商。
- 购置日期。
- 启用日期。
- 状态。
- 维护责任人。
- 关联固定资产卡片。

设备如果同时作为固定资产管理，要和固定资产卡片建立关联，但设备管理关注运行维护，固定资产关注价值和折旧。

## 四、点检

```text
生成点检任务
  -> 操作工或维修人员点检
  -> 记录结果
  -> 异常生成维修工单
```

点检项目：

- 温度。
- 压力。
- 噪音。
- 润滑。
- 安全防护。
- 清洁。
- 关键参数。

点检结果：

- 正常。
- 异常。
- 未点检。
- 待维修。

## 五、保养计划

保养可以按：

- 日期周期：每周、每月、每季度。
- 运行小时：运行 500 小时保养。
- 产量：生产 10,000 件后保养。
- 状态触发：异常点检后保养。

保养计划要明确：

- 设备。
- 保养项目。
- 周期。
- 责任人。
- 备件和耗材。
- 停机窗口。
- 验收标准。

## 六、维修流程

```text
设备故障
  -> 报修
  -> 维修派工
  -> 备件领用
  -> 维修处理
  -> 试机验收
  -> 关闭维修工单
  -> 统计停机和成本
```

维修工单字段：

- 设备。
- 故障时间。
- 故障描述。
- 故障等级。
- 影响产线。
- 维修人员。
- 开始维修时间。
- 完成时间。
- 停机时长。
- 备件消耗。
- 维修结果。
- 根因和预防措施。

## 七、备件管理

备件通常和库存模块联动。

```text
维修工单
  -> 备件领用
  -> 库存出库
  -> 维修成本归集
```

备件管理关注：

- 安全库存。
- 替代件。
- 关键备件。
- 采购周期。
- 维修消耗统计。
- 呆滞备件。

## 八、设备和生产的关系

设备可以关联：

- 工作中心。
- 工艺路线。
- 工序。
- 生产订单。
- 报工。
- 停机记录。

这样可以分析：

- 某设备影响了哪些生产订单。
- 某产品主要在哪些设备加工。
- 设备故障导致多少产量损失。
- 维修成本是否计入制造费用。

## 九、财务和成本关系

设备管理和财务关联：

- 设备购置形成固定资产。
- 设备折旧进入制造费用或管理费用。
- 维修备件出库形成维修费用。
- 大修可能资本化或费用化。
- 停机损失用于管理分析。

会计处理要由财务确认，不要由维修人员直接判断资本化或费用化。

## 十、常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 设备台账和固定资产不一致 | 两套编码未关联 | 建设备和资产映射 |
| 保养任务没人处理 | 责任人和提醒机制缺失 | 设置任务和逾期提醒 |
| 维修备件库存不准 | 线下领用未入系统 | 维修工单关联备件出库 |
| 停机时间不准 | 故障开始和恢复未记录 | 强制记录停机时间 |
| 维修成本无法分析 | 备件、人工、外协费用未归集 | 建维修成本口径 |

## 十一、实际案例

### 案例：关键设备预防性维护

背景：

- 一条产线的关键设备故障会导致整线停产。
- 过去只在故障后维修。

方案：

1. 建设备台账并关联工作中心。
2. 维护每周点检项目。
3. 设置每 500 小时保养一次。
4. 关键备件设置安全库存。
5. 点检异常自动生成维修工单。
6. 维修完成记录停机时间和备件消耗。
7. 月度分析故障次数和维修成本。

效果：

- 从事后维修转向预防维护。
- 停机原因和成本可追踪。
- 生产计划能提前避开保养窗口。
