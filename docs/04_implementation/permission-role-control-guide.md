权限实施不是简单给用户“能看能点”。在金蝶项目中，权限会影响组织隔离、单据录入、审批流、字段可见性、报表查询、数据安全和内控审计。权限做得太松，数据和审批失控；权限做得太紧，业务无法推进。

## 一、权限设计目标

权限方案要同时满足四个目标：

- 用户能完成岗位工作。
- 用户不能越权查看或处理无关数据。
- 关键业务动作有审批和日志。
- 财务、仓库、采购、销售、生产之间职责分离。

## 二、权限对象

金蝶项目中常见权限对象：

| 对象 | 说明 | 示例 |
| --- | --- | --- |
| 用户 | 登录系统的人 | 张三、李四 |
| 角色 | 权限集合 | 采购员、仓管员、总账会计 |
| 岗位 | 企业职责位置 | 华东销售经理 |
| 组织范围 | 用户能操作哪些组织数据 | A 公司、B 工厂 |
| 功能权限 | 能进入哪些菜单和按钮 | 新增、保存、提交、审核 |
| 数据权限 | 能看哪些组织、部门、客户、仓库数据 | 只看本部门客户 |
| 字段权限 | 字段是否可见、可编辑 | 价格、成本、毛利 |
| 报表权限 | 能看哪些报表和维度 | 成本报表、利润分析 |

## 三、权限设计流程

```text
收集岗位
  -> 梳理职责
  -> 设计角色
  -> 分配功能权限
  -> 分配组织和数据范围
  -> 配置字段和报表权限
  -> 测试关键流程
  -> 用户确认
  -> 上线冻结
```

## 四、岗位权限矩阵

权限矩阵是实施交付必备文件。

| 岗位 | 主要职责 | 关键权限 | 禁止事项 |
| --- | --- | --- | --- |
| 采购员 | 建采购申请、采购订单、跟踪到货 | 采购订单新增、提交、查询本人订单 | 不能审核付款，不能改供应商银行 |
| 采购经理 | 审批采购订单、管理价格 | 采购订单审核、价格查询 | 不能做付款凭证 |
| 仓管员 | 收料、入库、出库、盘点 | 入库单、出库单、盘点单 | 不能改物料成本 |
| 销售员 | 销售订单、发货申请、收款跟踪 | 销售订单新增、查询本人客户 | 不能改信用政策 |
| 财务应收 | 应收确认、开票、收款核销 | 应收单、收款单、核销 | 不能改销售订单数量 |
| 财务应付 | 应付确认、付款申请、核销 | 应付单、付款单、核销 | 不能改采购入库数量 |
| 总账会计 | 凭证、结账、报表 | 凭证、账簿、科目余额表 | 不能做业务审批替代 |
| 系统管理员 | 用户、角色、参数 | 管理权限 | 日常不应直接处理业务单据 |

## 五、职责分离

内控项目要重点检查职责分离。

### 采购到付款

| 环节 | 建议职责 |
| --- | --- |
| 采购申请 | 需求部门 |
| 采购订单 | 采购员 |
| 采购订单审批 | 采购经理或授权人 |
| 收料入库 | 仓库 |
| 应付确认 | 应付会计 |
| 付款审批 | 财务经理或资金负责人 |
| 付款执行 | 出纳 |

风险点：

- 同一人既建供应商又做付款。
- 同一人既做采购订单又审核付款。
- 仓库人员能修改采购价格。

### 销售到收款

| 环节 | 建议职责 |
| --- | --- |
| 销售订单 | 销售员 |
| 价格或信用审批 | 销售经理或财务 |
| 发货 | 仓库 |
| 应收确认 | 应收会计 |
| 收款登记 | 出纳 |
| 核销 | 应收会计 |

风险点：

- 销售员能修改客户信用和收款条件。
- 出纳既收款又核销且无人复核。
- 仓库可以无订单发货。

### 总账和月结

| 环节 | 建议职责 |
| --- | --- |
| 凭证录入 | 会计 |
| 凭证审核 | 主管会计 |
| 凭证过账 | 总账会计或财务主管 |
| 结账 | 总账会计 |
| 反结账 | 严格授权 |
| 报表确认 | 财务负责人 |

风险点：

- 制单人和审核人为同一人。
- 反审核、反过账、反结账权限过宽。
- 已结账期间仍允许业务补录。

## 六、字段和敏感数据权限

敏感字段要特别控制：

- 采购价格。
- 销售价格和折扣。
- 客户信用额度。
- 供应商银行账号。
- 员工工资或费用。
- 成本单价。
- 毛利率。
- 凭证调整说明。

字段权限设计方法：

1. 先列敏感字段清单。
2. 确认哪些岗位必须查看。
3. 确认哪些岗位可以编辑。
4. 对报表和导出权限同步控制。
5. 测试列表、单据、报表、移动端是否都生效。

## 七、组织和数据范围

多组织项目要明确：

- 用户能登录哪些组织。
- 能处理哪些业务组织单据。
- 能查看哪些库存组织数据。
- 能查询哪些账簿和核算组织。
- 跨组织单据谁可以发起，谁可以审核。

示例：

| 用户 | 组织范围 | 数据范围 |
| --- | --- | --- |
| 华东销售员 | 华东销售组织 | 本人客户、本部门订单 |
| 集团财务 | 集团和下属公司 | 全集团财务报表 |
| A 工厂仓管 | A 工厂库存组织 | A 工厂仓库和库位 |
| 采购共享中心 | 多个采购组织 | 授权供应商和采购订单 |

## 八、审批流和权限的关系

审批流不能替代权限，权限也不能替代审批。

- 权限决定能不能发起或处理。
- 审批流决定关键动作是否需要批准。
- 操作日志记录谁做了什么。
- 参数控制业务规则能否突破。

审批流设计关注：

- 金额阈值。
- 部门或组织。
- 物料类别。
- 客户信用。
- 供应商类型。
- 紧急采购。
- 价格例外。

## 九、权限测试

### 测试方法

不要只用管理员测试。必须用真实岗位账号测试。

| 测试类型 | 示例 |
| --- | --- |
| 正向测试 | 采购员能新增采购订单并提交 |
| 反向测试 | 采购员不能审核付款单 |
| 组织测试 | A 工厂仓管不能看 B 工厂库存 |
| 字段测试 | 销售员不能看成本价 |
| 报表测试 | 部门经理只能看本部门数据 |
| 审批测试 | 超限金额自动走上级审批 |

### 测试记录

| 处理 | 调整角色权限 |

## 十、实际案例

### 案例：上线后仓管员看不到入库单

现象：

- 仓管员可以登录系统。
- 菜单里看得到采购入库单。
- 查询不到采购组织为 A 公司的入库单。

排查：

1. 检查菜单权限，发现有采购入库单查询权限。
2. 检查按钮权限，新增、审核权限按设计未给。
3. 检查组织范围，用户只有库存组织权限，没有采购组织数据权限。
4. 检查单据数据权限，过滤条件限制了仓库范围。

处理：

- 保留功能权限不变。
- 给该用户对应采购组织的查询数据权限。
- 仓库范围只开放本人负责仓库。
- 用真实账号重新查询采购入库单。

复盘：

- 只检查菜单权限不够。
- 组织、单据、仓库、字段权限要一起测。

## 十一、上线前权限冻结

上线前一周建议冻结权限方案：

- 不再随意新增角色。
- 权限变更走申请和审批。
- 保留权限矩阵版本。
- 导出用户角色清单。
- 关键岗位做最终登录测试。
- 管理员账号单独登记。

## 十二、权限运维

上线后要建立权限维护机制：

- 新员工按岗位模板授权。
- 离职员工及时禁用。
- 调岗员工收回原权限再授权。
- 临时权限设置有效期。
- 高危权限定期审计。
- 每月导出用户角色清单复核。

## 十三、常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 能进菜单但看不到数据 | 数据权限或组织范围不足 | 检查组织、部门、仓库、单据过滤 |
| 能看但不能保存 | 按钮或操作权限不足 | 检查新增、保存、提交权限 |
| 审批人收不到任务 | 审批流条件或人员映射错误 | 检查岗位、部门、组织和审批规则 |
| 用户看到成本价 | 字段或报表权限过宽 | 收紧字段、列表、报表和导出权限 |
| 管理员被用于日常业务 | 权限管理不规范 | 建普通业务账号，管理员只做维护 |
