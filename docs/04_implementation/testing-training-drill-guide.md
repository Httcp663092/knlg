测试、培训和上线演练是把“配置完成”变成“用户真的会用、系统真的能跑”的阶段。金蝶项目常见问题不是没有配置，而是关键流程没测全、用户不会处理异常、上线当天才发现权限或数据问题。

## 一、测试总体策略

测试要覆盖四类内容：

- 功能能不能用。
- 流程能不能串起来。
- 数据结果是否正确。
- 用户是否能独立操作。

```text
配置单元测试
  -> 端到端流程测试
  -> 接口和二开测试
  -> 权限测试
  -> 月结演练
  -> UAT 签字
  -> 上线演练
```

## 二、测试环境准备

### 环境要求

- 测试环境版本和生产计划版本一致。
- 基础资料和参数接近生产。
- 权限使用真实岗位账号。
- 测试数据包含正常业务和异常业务。
- 接口、打印、附件、审批流都要可用。

### 数据准备

| 数据 | 示例 |
| --- | --- |
| 主数据 | 物料、客户、供应商、仓库、部门、人员 |
| 业务数据 | 采购订单、销售订单、生产订单 |
| 财务数据 | 科目、凭证模板、期初余额 |
| 异常数据 | 退货、红冲、关闭、超限审批 |
| 接口数据 | 外部系统推送和回写样例 |

## 三、测试用例设计

测试用例不能只写“新增一张单”。要覆盖完整业务链。

### 用例字段

| 字段 | 说明 |
| --- | --- |
| 用例编号 | FIN-GL-001、SCM-PUR-001 |
| 模块 | 财务、采购、销售、库存、生产 |
| 场景 | 采购入库到应付暂估 |
| 前置条件 | 主数据、权限、参数、期间 |
| 操作账号 | 真实岗位账号 |
| 操作步骤 | 每一步菜单和关键字段 |
| 预期结果 | 单据状态、库存、应收应付、凭证 |
| 实际结果 | 通过、失败、阻塞 |
| 问题编号 | 关联缺陷台账 |

### 采购到付款用例

```text
采购申请
  -> 采购订单
  -> 收料通知
  -> 采购入库
  -> 应付单
  -> 采购发票
  -> 付款申请
  -> 付款单
  -> 核销
  -> 生成凭证
```

检查点：

- 订单数量和入库数量是否一致。
- 部分入库是否正确。
- 暂估应付是否正确。
- 到票后暂估冲回是否正确。
- 付款是否按供应商、币别、账期控制。
- 凭证科目和辅助核算是否正确。

### 销售到收款用例

```text
销售订单
  -> 发货通知
  -> 销售出库
  -> 应收单
  -> 销售发票
  -> 收款单
  -> 核销
  -> 生成凭证
```

检查点：

- 价格、折扣、税率是否正确。
- 信用控制是否生效。
- 部分发货、退货、红字应收是否正确。
- 应收账龄和客户余额是否正确。
- 收款核销后余额是否正确。

### 生产到成本用例

```text
BOM
  -> MRP
  -> 生产订单
  -> 领料
  -> 补料或退料
  -> 完工入库
  -> 存货核算
  -> 成本计算
  -> 生成凭证
```

检查点：

- BOM 用料、损耗、替代料是否正确。
- MRP 建议是否符合计划参数。
- 领料数量和生产订单关联是否正确。
- 完工入库是否影响库存。
- 材料成本、人工、制造费用分配是否符合方案。

## 四、缺陷管理

### 缺陷字段

| 字段 | 说明 |
| --- | --- |
| 编号 | BUG-001 |
| 模块 | 采购、总账、库存 |
| 严重级别 | 阻塞、高、中、低 |
| 现象 | 用户看到的具体问题 |
| 复现步骤 | 能让别人复现 |
| 期望结果 | 应该是什么 |
| 责任人 | 顾问、开发、客户、运维 |
| 状态 | 新建、处理中、待验证、关闭 |
| 关闭依据 | 截图、测试结果、签字 |

### 缺陷分级

| 级别 | 定义 | 示例 |
| --- | --- | --- |
| 阻塞 | 关键流程无法继续 | 采购入库无法审核 |
| 高 | 影响上线核心目标 | 应收凭证科目错误 |
| 中 | 有替代方案但需修复 | 报表字段展示不完整 |
| 低 | 不影响上线 | 文案、排序、样式问题 |

缺陷必须闭环，不能只在微信群里说一声。

## 五、UAT 用户验收测试

UAT 的核心是让关键用户按真实业务操作。

### UAT 准备

- 按岗位分配测试账号。
- 发放测试用例和操作手册。
- 准备真实业务样例。
- 明确测试时间和签到。
- 现场记录问题。

### UAT 通过标准

- 关键流程测试通过。
- 阻塞和高优先级问题清零。
- 中低优先级问题有计划。
- 关键用户签字确认。
- 测试报告归档。

## 六、月结演练

金蝶项目必须做月结演练，尤其是财务供应链制造一体化项目。

### 演练顺序

1. 采购入库和应付确认完成。
2. 销售出库和应收确认完成。
3. 库存单据审核完成。
4. 存货核算完成。
5. 生产成本计算完成。
6. 固定资产计提折旧。
7. 应收应付核销和汇兑处理。
8. 业务单据生成凭证。
9. 总账审核、过账、结转损益。
10. 出具资产负债表、利润表、科目余额表。

### 演练检查点

- 业务单据是否全部审核。
- 暂估、发票、核销是否完整。
- 存货核算是否有异常单价。
- 生产成本是否存在未分配费用。
- 凭证模板是否能覆盖主要业务。
- 科目余额、辅助余额、报表是否一致。

## 七、培训设计

培训不是讲功能清单，而是让用户完成岗位任务。

### 培训对象

| 对象 | 培训重点 |
| --- | --- |
| 高层和负责人 | 项目目标、流程变化、报表查看 |
| 关键用户 | 端到端流程、异常处理、问题反馈 |
| 普通操作用户 | 本岗位日常操作 |
| 财务用户 | 凭证、对账、月结、报表 |
| 系统管理员 | 用户、权限、基础资料、环境维护 |

### 培训材料

- 岗位操作手册。
- 流程图。
- 常见问题。
- 练习数据。
- 截图步骤。
- 异常处理说明。

## 八、培训方法

每次培训建议采用：

```text
讲流程
  -> 演示系统
  -> 用户练习
  -> 现场答疑
  -> 留作业
  -> 收集问题
```

培训后要记录：

- 培训主题。
- 参训人员。
- 签到表。
- 练习完成情况。
- 问题清单。
- 后续补训对象。

## 九、上线演练

上线演练要模拟正式切换。

### 演练内容

- 数据冻结。
- 最终模板导入。
- 期初对账。
- 用户登录。
- 权限验证。
- 首批业务单据录入。
- 接口任务执行。
- 打印和附件验证。
- 问题反馈通道。
- 回退预案演练。

### 演练结果

上线演练后输出：

- 演练报告。
- 问题清单。
- 修复计划。
- 上线风险评估。
- 是否允许上线的结论。

## 十、实际案例

### 案例：测试只测新增，没测反审核

现象：

- 上线后采购入库单发现价格错误。
- 用户想反审核修改。
- 系统提示下游应付单已生成，无法直接反审核。

原因：

- 测试阶段只测了新增、审核、生成应付。
- 没有测试错误单据如何处理。
- 用户不知道需要先处理下游单据。

补救：

1. 梳理采购入库错误处理流程。
2. 补充反审核、红冲、差异调整用例。
3. 培训仓库和应付会计。
4. 在操作手册中加入异常处理章节。

教训：

- 每个核心流程都要测“正常、异常、冲销、关闭”。
