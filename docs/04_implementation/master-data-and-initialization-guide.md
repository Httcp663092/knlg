主数据和初始化是金蝶项目最容易低估、也最容易导致上线失败的工作。系统配置再完整，如果物料、客户、供应商、科目、仓库、期初余额和库存期初不准确，上线后就会出现业务跑不动、对账对不上、凭证生成异常、成本不可信等问题。

## 一、主数据和初始化的关系

```text
主数据
  -> 支撑业务单据录入
  -> 支撑权限、审批、报表和凭证
  -> 支撑期初数据导入

期初数据
  -> 依赖主数据
  -> 承接旧系统余额
  -> 决定新系统上线起点
```

主数据先不干净，期初导入就会反复失败。期初不准确，后续业务即使录入正确，也很难和旧系统、财务账、库存账对齐。

## 二、主数据范围

### 组织和基础架构

| 数据 | 说明 | 关键字段 |
| --- | --- | --- |
| 组织 | 法人、业务组织、库存组织、工厂、利润中心 | 编码、名称、组织类型、上下级 |
| 部门 | 管理和费用归属 | 编码、名称、所属组织、状态 |
| 员工 | 业务员、采购员、仓管员、审批人 | 员工编码、部门、岗位、用户绑定 |
| 用户 | 系统登录和权限 | 用户名、人员、角色、组织范围 |

### 财务主数据

| 数据 | 说明 | 常见风险 |
| --- | --- | --- |
| 科目 | 总账核算基础 | 科目级次、辅助核算、余额方向不一致 |
| 核算维度 | 部门、客户、供应商、项目等 | 维度过多导致录入和对账复杂 |
| 币别 | 本位币、外币 | 汇率类型、精度、期初外币余额 |
| 账簿 | 核算账套或准则 | 期间、币别、科目表、组织关系 |
| 凭证字 | 记、收、付、转等 | 和客户习惯、打印格式相关 |

### 供应链主数据

| 数据 | 说明 | 常见风险 |
| --- | --- | --- |
| 物料 | 采购、销售、库存、生产、成本核心资料 | 编码重复、计量单位混乱、属性错误 |
| 客户 | 销售、应收、开票、信用 | 名称重复、税号不准、停用状态混乱 |
| 供应商 | 采购、应付、发票、付款 | 付款条件、税率、结算币别不一致 |
| 仓库 | 收发存地点 | 仓库和组织关系不清 |
| 库位 | 精细库存位置 | 启用后必须规范收发存 |
| 批号规则 | 批次管理 | 历史批号不规范 |
| 计量单位 | 基本单位、采购单位、销售单位 | 换算率错误导致数量异常 |

### 制造主数据

| 数据 | 说明 | 常见风险 |
| --- | --- | --- |
| BOM | 产品结构 | 版本、生效日期、损耗、替代料不清 |
| 工艺路线 | 工序和工时 | 不用车间模块时也要明确成本口径 |
| 工作中心 | 产能和费用归集 | 和部门、产线、设备混淆 |
| 成本项目 | 材料、人工、制造费用 | 分配规则不明确 |

## 三、编码设计

### 编码原则

- 唯一：同一业务对象不能多个编码表示。
- 稳定：编码不要频繁变更。
- 可扩展：预留未来组织、品类、区域。
- 少含义：不要把过多业务含义塞进编码。
- 易识别：关键对象可以保留分类前缀。

### 物料编码建议

| 方案 | 示例 | 适用场景 | 注意 |
| --- | --- | --- | --- |
| 分类流水 | RM-010001 | 物料数量多、分类清晰 | 分类变更时不要随意改编码 |
| 大类小类流水 | RM-EL-0001 | 需要区分原材料、电子料等 | 分类层级不要过深 |
| 无含义流水 | M0000001 | 数据量大、系统自动编码 | 依赖名称、规格和属性识别 |

物料编码不要把供应商、价格、仓库、颜色等经常变化的信息写死进去，变化信息应放字段或维度。

### 客商编码建议

- 客户可按区域或渠道分类。
- 供应商可按类别或合作类型分类。
- 同一公司既是客户又是供应商时，要明确是否维护两份资料，还是通过客商统一资料管理。

## 四、数据清洗流程

```text
导出旧数据
  -> 去重
  -> 补必填字段
  -> 统一编码和名称
  -> 补业务属性
  -> 校验引用关系
  -> 关键用户确认
  -> 试导入
  -> 修正错误
  -> 正式导入
```

### 清洗规则

| 问题 | 处理方式 |
| --- | --- |
| 重复物料 | 按规格、单位、用途、库存余额确认保留编码 |
| 客户名称不规范 | 统一工商全称，简称放简称字段 |
| 停用资料仍有余额 | 上线前保留，余额结清后停用 |
| 缺少税率或付款条件 | 由采购、销售、财务共同确认 |
| 仓库和组织不匹配 | 先确认库存组织，再建仓库 |
| 科目辅助核算不一致 | 先冻结科目方案，再填期初 |

## 五、导入模板管理

### 模板控制

- 模板必须带版本号。
- 不允许多人各自改模板结构。
- 必填字段用颜色或说明标记。
- 字段说明写清取值范围。
- 每次导入保留原始文件、修正文件和导入结果。

### 常用字段说明

| 字段类型 | 示例 | 检查点 |
| --- | --- | --- |
| 编码 | 物料编码、客户编码 | 不重复、不为空 |
| 名称 | 物料名称、客户名称 | 不含多余空格和特殊字符 |
| 引用字段 | 物料分类、仓库、部门 | 引用资料必须存在 |
| 状态 | 创建、审核、禁用 | 上线时可用数据必须审核 |
| 金额数量 | 期初数量、余额 | 精度、方向、币别 |
| 日期 | 启用日期、生效日期 | 不能晚于上线期间 |

## 六、财务期初初始化

### 总账期初

总账期初至少包含：

- 科目期初余额。
- 辅助核算期初余额。
- 外币余额。
- 数量金额核算余额。

检查规则：

- 借贷必须平衡。
- 辅助核算余额汇总等于科目余额。
- 外币原币、本位币、汇率关系合理。
- 期初期间和启用期间一致。
- 科目方向和余额方向异常要说明原因。

### 应收期初

应收期初建议按单据明细导入，而不是只导总额。

字段要素：

- 客户。
- 业务组织。
- 销售部门和业务员。
- 单据日期。
- 到期日。
- 币别和汇率。
- 应收金额。
- 已收金额。
- 未收余额。
- 发票信息或合同订单信息。

应收期初必须和总账应收科目对账。

### 应付期初

应付期初字段：

- 供应商。
- 采购组织。
- 采购部门和采购员。
- 单据日期。
- 到期日。
- 币别和汇率。
- 应付金额。
- 已付金额。
- 未付余额。
- 发票或采购入库关联信息。

应付期初必须和总账应付科目对账。

### 固定资产期初

资产卡片要能支撑后续折旧。

关键字段：

- 资产编码和名称。
- 资产类别。
- 使用部门。
- 原值。
- 累计折旧。
- 净值。
- 折旧方法。
- 使用年限。
- 已使用期间。
- 开始使用日期。
- 残值率。

检查规则：

- 卡片原值汇总等于固定资产科目余额。
- 卡片累计折旧汇总等于累计折旧科目余额。
- 折旧方法和剩余年限正确。

## 七、库存期初初始化

库存期初不是只导数量，还要关注金额、批号、库位、库存状态和成本。

### 期初字段

- 物料。
- 仓库。
- 库位。
- 批号。
- 库存状态。
- 基本单位数量。
- 辅助单位数量。
- 单价。
- 金额。
- 生产日期或有效期。

### 库存期初对账

```text
仓库盘点数量
  = 金蝶库存期初数量

库存期初金额
  = 存货核算期初金额
  = 总账存货科目余额
```

如果库存有数量无金额，后续成本会失真；如果有金额无数量，则存货核算也会异常。

## 八、制造期初

制造项目常见期初：

- 未完工生产订单。
- 已领未完工材料。
- 委外在制。
- 在产品成本。
- BOM 当前版本。
- 未结采购订单和销售订单。

处理原则：

- 能在旧系统结清的尽量结清。
- 无法结清的要迁移未完业务。
- 在制品数量和金额要和财务、生产确认。
- 上线首月成本计算前必须演练。

## 九、初始化验证

### 验证顺序

1. 主数据导入成功。
2. 主数据审核状态正确。
3. 财务期初借贷平衡。
4. 应收应付期初与总账一致。
5. 固定资产卡片与总账一致。
6. 库存期初数量与盘点一致。
7. 存货期初金额与总账一致。
8. 未结业务可以继续流转。
9. 业务单据可以生成凭证。
10. 报表能按期初数据出数。

### 验证报表

| 对账对象 | 报表或查询 |
| --- | --- |
| 总账 | 科目余额表、辅助余额表 |
| 应收 | 应收明细表、应收账龄表 |
| 应付 | 应付明细表、应付账龄表 |
| 库存 | 即时库存、库存台账 |
| 存货 | 存货收发存汇总、存货核算报表 |
| 固定资产 | 固定资产清单、折旧明细 |

## 十、实际案例

### 案例：库存期初和总账存货对不上

现象：

- 库存期初金额为 1,200,000。
- 总账存货科目余额为 1,260,000。
- 差异 60,000。

排查：

1. 按仓库汇总库存期初金额。
2. 按物料分类汇总库存金额。
3. 检查是否有已入库未开票暂估。
4. 检查旧系统是否把在途物资也放在存货科目。
5. 检查盘点调整是否已入账。

处理：

- 如果差异属于在途物资，不应导入库存期初，要在总账或采购未结业务体现。
- 如果差异属于盘点差异，要让客户在旧系统或新系统期初调整前确认。
- 最终输出库存期初和总账差异说明，由财务和仓库签字。

## 十一、签字确认

上线前必须让客户确认：

- 主数据版本。
- 财务期初余额。
- 应收应付期初明细。
- 库存盘点和期初金额。
- 固定资产卡片。
- 未结业务清单。

签字不是形式，它是上线后判断“系统问题”还是“源数据问题”的依据。

## 十二、常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| 导入失败 | 引用资料不存在、字段为空、格式错误 | 先校验模板，再分批导入 |
| 期初不平 | 科目方向、辅助核算、外币数据错误 | 按科目和辅助维度逐级核对 |
| 物料重复 | 历史编码不统一 | 建立合并映射表 |
| 期初库存无单价 | 旧系统没有成本或成本异常 | 财务确认估价或重算成本 |
| 应收应付与总账不一致 | 明细口径和总账口径不同 | 先定义口径，再出差异说明 |
| 上线后不能录单 | 主数据未审核或权限不足 | 检查状态、组织范围和角色 |
