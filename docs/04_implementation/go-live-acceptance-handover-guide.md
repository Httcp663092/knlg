上线不是把系统打开给用户使用那么简单。金蝶项目上线切换要处理旧系统冻结、最终数据导入、期初验证、权限开放、首批业务跟踪、问题响应、验收归档和运维移交。上线阶段最怕职责不清、时间不清、口径不清。

## 一、上线判断

满足以下条件才建议进入正式上线：

- 蓝图范围内核心流程测试通过。
- 主数据和期初数据完成最终确认。
- 关键用户培训完成。
- 权限矩阵测试通过。
- 接口和二开通过联调。
- 月结演练至少完成一次。
- 阻塞问题清零，高优先级问题有关闭记录。
- 上线切换方案已评审。
- 应急和回退方案已确认。

## 二、上线切换计划

### 切换计划结构

```text
1 上线范围
2 上线时间
3 冻结安排
4 数据导入安排
5 权限开放安排
6 首批业务验证
7 问题响应机制
8 应急预案
9 责任人和联系方式
10 上线后支持计划
```

### 时间安排示例

| 时间 | 工作 | 责任人 | 输出 |
| --- | --- | --- | --- |
| T-7 | 冻结权限方案和主数据模板 | 项目经理 | 冻结通知 |
| T-5 | 最后一轮主数据校验 | 顾问、关键用户 | 校验报告 |
| T-3 | 旧系统未结业务清单确认 | 业务负责人 | 未结清单 |
| T-2 | 库存盘点和财务余额确认 | 仓库、财务 | 盘点表、余额表 |
| T-1 | 旧系统停止录入或限定录入 | 客户项目经理 | 冻结确认 |
| T | 正式导入和验证 | 顾问、IT | 导入结果 |
| T+1 | 开放用户和首批单据 | 业务、顾问 | 首批业务记录 |
| T+3 | 问题复盘和修复计划 | 项目组 | 问题台账 |
| T+30 | 首月月结支持 | 财务、顾问 | 月结报告 |

## 三、冻结管理

### 需要冻结的内容

- 组织和账簿。
- 科目和辅助核算。
- 物料、客户、供应商、仓库。
- BOM 和关键生产资料。
- 期初余额和盘点结果。
- 未结采购、销售、生产、委外业务清单。
- 权限矩阵。
- 二开和接口版本。

冻结不是不能改，而是任何修改都必须登记、审批和同步影响范围。

### 冻结通知内容

- 冻结开始时间。
- 冻结对象。
- 允许例外的场景。
- 例外审批人。
- 变更登记方式。
- 联系人。

## 四、最终数据导入

### 导入顺序

```text
组织和基础配置
  -> 用户和权限
  -> 主数据
  -> BOM 和业务基础资料
  -> 财务期初
  -> 应收应付期初
  -> 固定资产期初
  -> 库存和存货期初
  -> 未结业务
  -> 验证报表
```

### 导入记录

每次正式导入必须记录：

- 文件名。
- 模板版本。
- 导入人。
- 导入时间。
- 成功条数。
- 失败条数。
- 失败原因。
- 修正方式。
- 最终确认人。

## 五、上线验证

### 基础验证

- 用户可以登录。
- 菜单和角色正确。
- 组织范围正确。
- 主数据可查询且状态正确。
- 单据编号规则正常。
- 审批流正常。
- 打印模板正常。

### 数据验证

| 数据 | 验证方式 |
| --- | --- |
| 总账期初 | 科目余额表借贷平衡 |
| 应收期初 | 应收明细汇总等于总账 |
| 应付期初 | 应付明细汇总等于总账 |
| 库存期初 | 即时库存和盘点表一致 |
| 存货金额 | 存货报表和总账存货科目一致 |
| 固定资产 | 卡片原值和累计折旧与总账一致 |

### 首批业务验证

上线后不要让所有业务一拥而上。先选首批典型业务：

- 一笔采购订单到入库。
- 一笔销售订单到出库。
- 一笔生产订单到完工。
- 一笔收款和核销。
- 一笔付款和核销。
- 一张业务凭证生成。
- 一张关键报表查询。

首批业务要截图和记录，作为上线成功证据。

## 六、应急预案

### 常见上线风险

| 风险 | 应急处理 |
| --- | --- |
| 用户无法登录 | 准备管理员和 IT 联系人，检查账号和权限 |
| 单据无法保存 | 检查必填字段、编码规则、基础资料状态 |
| 审批流卡住 | 临时授权审批管理员，记录后补审批 |
| 接口失败 | 启用手工导入或重传机制 |
| 期初差异 | 暂停业务开放，定位数据差异 |
| 打印异常 | 准备临时打印模板或导出方案 |
| 性能异常 | 限制批量任务，检查服务器和后台任务 |

### 回退原则

回退不是随口说“回旧系统”。必须明确：

- 什么条件触发回退。
- 回退由谁决策。
- 已录入新系统的数据怎么处理。
- 旧系统是否还能继续录入。
- 回退后如何重新上线。

多数项目更适合“局部应急处理”，而不是整体回退。

## 七、上线支持

### 支持机制

- 建问题反馈群，但问题必须进入台账。
- 设置现场或远程值守时间。
- 明确问题分级和响应时限。
- 每天固定时间复盘问题。
- 对重复问题补充 FAQ 和操作手册。

### 问题台账字段

| 字段 | 说明 |
| --- | --- |
| 编号 | GO-001 |
| 提出人 | 用户姓名 |
| 模块 | 财务、采购、库存 |
| 现象 | 报错原文或截图描述 |
| 影响 | 阻塞、高、中、低 |
| 责任人 | 顾问、开发、客户 IT |
| 处理方案 | 临时方案和最终方案 |
| 状态 | 新建、处理中、待验证、关闭 |
| 关闭确认 | 用户确认或截图 |

## 八、首月月结支持

首月月结是上线后最关键的验收场景。

### 月结前

- 检查所有业务单据状态。
- 清理未审核、未生成下游、未核销单据。
- 检查库存负数和异常成本。
- 检查应收应付未核销余额。
- 检查固定资产折旧。
- 检查业务凭证生成状态。

### 月结中

- 按模块顺序关账。
- 先业务模块，后总账。
- 遇到差异记录原因。
- 不要无记录地直接改数据。

### 月结后

- 输出科目余额表。
- 输出应收应付账龄。
- 输出库存和存货核算报表。
- 输出资产负债表和利润表。
- 与旧口径或管理口径做差异说明。

## 九、验收

### 验收条件

- 项目范围内功能上线。
- 关键业务流程已运行。
- 首月或约定周期月结完成。
- 关键报表可以出具。
- 阻塞问题关闭。
- 交付物完整。
- 运维移交完成。

### 验收材料

| 材料 | 说明 |
| --- | --- |
| 蓝图确认版 | 作为项目范围依据 |
| 配置清单 | 参数、流程、权限、凭证模板 |
| 数据导入报告 | 主数据和期初结果 |
| 测试报告 | 用例、缺陷、UAT 签字 |
| 培训记录 | 签到、材料、问题 |
| 上线报告 | 切换过程、首批业务、问题 |
| 月结报告 | 首月结账和报表验证 |
| 问题清单 | 已关闭和遗留问题 |
| 操作手册 | 岗位操作说明 |
| 运维移交 | 账号、环境、备份、发布流程 |

## 十、运维移交

### 移交对象

- 客户 IT。
- 系统管理员。
- 财务关键用户。
- 业务关键用户。
- 后续运维顾问。

### 移交内容

| 类别 | 内容 |
| --- | --- |
| 系统环境 | 访问地址、服务器、数据库、中间件、版本 |
| 用户权限 | 管理员、角色矩阵、权限维护流程 |
| 配置资料 | 参数清单、编码规则、审批流、打印模板 |
| 接口二开 | 接口清单、日志位置、失败重传、代码版本 |
| 数据维护 | 主数据新增、停用、变更流程 |
| 运维操作 | 备份、补丁、发布、重启、监控 |
| 问题处理 | 常见问题、升级渠道、服务联系人 |

## 十一、实际案例

### 案例：上线当天发现销售员不能提交订单

现象：

- 销售员可以新增销售订单。
- 保存成功。
- 点击提交提示无权限。

原因：

- 权限测试只测了新增和保存。
- 提交按钮权限漏配。

处理：

1. 临时给销售员角色补充提交权限。
2. 使用销售员账号重新测试新增、保存、提交、撤销提交。
3. 检查其他角色是否也漏配按钮权限。
4. 更新权限矩阵和上线问题台账。

复盘：

- 上线前权限测试必须按完整动作测，不只测菜单。
